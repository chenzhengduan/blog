# 提示词工程基础

## 🌟 写在前面

如果你很久没做前端开发，或者刚接触AI编程，不用担心！这份指南会从最基础的概念开始，帮你一步步掌握如何与AI有效沟通来解决编程问题。

## 🎯 什么是提示词工程

### 简单理解
想象你在和一个非常聪明的助手对话，这个助手懂很多编程知识，但需要你清楚地告诉它你想要什么。**提示词工程**就是学会如何更好地"说话"，让AI助手准确理解你的需求并给出有用的代码。

### 为什么重要？
- **节省时间**：好的提示词能让AI一次性给出你想要的代码
- **减少错误**：清晰的描述能避免AI生成错误的代码
- **提高效率**：学会提示词技巧，编程效率能提升数倍

### 基础概念解释

**前端开发**：就是制作网页和手机App界面的技术，包括：
- **HTML**：网页的骨架结构（就像房子的框架）
- **CSS**：网页的样式美化（就像房子的装修）
- **JavaScript**：网页的交互功能（就像房子的电器设备）

**常见前端框架**：
- **Vue**：一个让网页开发更简单的工具包
- **React**：另一个流行的网页开发工具包
- **TypeScript**：JavaScript的增强版，能帮助发现代码错误

## 🏗️ 如何和AI对话

### 🚀 从最简单开始

刚开始时，你可以这样和AI对话：

```
"帮我写一个简单的网页，显示'欢迎来到我的网站'"
```

AI会给你一个基本的HTML代码。这就是最简单的提示词！

### 📝 让提示词更有效的5个要素

想要AI给出更好的代码，你需要告诉它这5个信息：

#### 1. 🎭 角色设定（告诉AI它是谁）
```
"你是一个前端开发专家"
```

#### 2. 🎯 任务描述（你想要什么）
```
"帮我创建一个登录页面"
```

#### 3. 📋 具体要求（详细说明）
```
"需要用户名和密码输入框，还要一个登录按钮"
```

#### 4. 📤 输出格式（你想要什么样的结果）
```
"请给我完整的HTML和CSS代码"
```

#### 5. 🔍 背景信息（项目相关信息）
```
"这是一个简单的个人网站项目"
```

### 🌟 完整示例

把这5个要素组合起来：

```
你是一个前端开发专家。

请帮我创建一个登录页面，需要：
1. 用户名输入框
2. 密码输入框  
3. 登录按钮
4. 简洁美观的样式

请提供完整的HTML和CSS代码，要求代码简单易懂。

这是一个个人学习项目，不需要复杂的功能。
```

这样的提示词会让AI给出更准确、更符合你需求的代码！

## 🎨 写好提示词的3个技巧

### 1. 🎯 要具体，不要模糊

**❌ 这样说AI会困惑：**
```
"帮我写一个组件"
```
*AI心想：什么组件？做什么用的？什么样子的？*

**✅ 这样说AI就明白了：**
```
"帮我写一个显示用户头像的组件，要求：
1. 如果有头像图片就显示图片
2. 如果没有图片就显示用户名的第一个字母
3. 头像是圆形的
4. 点击头像时可以触发事件
5. 用简单的HTML和CSS实现"
```
*AI心想：明白了！这样我就知道该怎么写了。*

### 2. 📋 分条列出，有条理

**❌ 一大段文字，AI容易漏掉：**
```
"帮我做一个搜索框，要能实时搜索，还要有建议列表，用户可以用键盘选择，也可以点击选择，还要有清空按钮，用React写"
```

**✅ 分条列出，清晰明了：**
```
"帮我做一个搜索框组件，需要这些功能：

功能要求：
1. 用户输入时显示搜索建议
2. 可以用上下箭头键选择建议
3. 点击建议项可以选中
4. 有清空按钮

技术要求：
1. 用HTML、CSS、JavaScript实现
2. 代码要简单易懂
3. 提供使用示例

请给我完整的代码。"
```

### 3. 🔍 告诉AI你的情况

**❌ 没有背景信息：**
```
"帮我写一个商品列表页面"
```
*AI不知道你的项目情况，可能给出不合适的代码*

**✅ 提供背景信息：**
```
"我正在做一个网店的管理后台，需要一个商品列表页面。

我的项目情况：
- 这是我第一次做网站，希望代码简单一些
- 主要显示商品名称、价格、库存数量
- 需要能够编辑和删除商品
- 用普通的HTML、CSS、JavaScript就行

请给我一个简单易懂的实现方案。"
```
*AI知道你是初学者，会给出更适合的简单代码*

## 🛠️ 4种常见的AI对话场景

### 1. 🆕 我要创建新功能

**什么时候用：** 你想从零开始做一个新的功能

**怎么说：**
```
"我想做一个[功能名称]，需要：
1. [功能1]
2. [功能2]
3. [功能3]

请给我完整的代码。"
```

**真实例子：**
```
"我想做一个图片轮播功能，需要：
1. 图片可以自动切换，每3秒换一张
2. 用户可以点击左右箭头手动切换
3. 下面有小圆点显示当前是第几张
4. 可以无限循环播放

请用简单的HTML、CSS、JavaScript给我完整的代码。"
```

### 2. 🔧 我的代码有问题

**什么时候用：** 你的代码不工作，或者有错误

**怎么说：**
```
"我的代码有问题：

问题：[描述什么不对]

错误信息：[复制错误信息]

我的代码：
[粘贴你的代码]

请帮我修复。"
```

**真实例子：**
```
"我的代码有问题：

问题：点击按钮后页面没有反应

错误信息：Uncaught TypeError: Cannot read property 'addEventListener' of null

我的代码：
<button id='myBtn'>点击我</button>
<script>
document.getElementById('myBtn').addEventListener('click', function() {
  alert('Hello!');
});
</script>

请帮我修复。"
```

### 3. ⚡ 我想优化代码

**什么时候用：** 代码能工作，但你觉得可以写得更好

**怎么说：**
```
"请帮我优化这段代码：

[粘贴代码]

我希望：
1. [优化目标1，比如：运行更快]
2. [优化目标2，比如：代码更简洁]
3. [优化目标3，比如：更容易理解]

请给我优化后的代码。"
```

**真实例子：**
```
"请帮我优化这段代码：

for (let i = 0; i < users.length; i++) {
  if (users[i].age > 18) {
    document.getElementById('list').innerHTML += '<div>' + users[i].name + '</div>';
  }
}

我希望：
1. 运行更快
2. 代码更简洁
3. 更容易理解

请给我优化后的代码。"
```

### 4. 🤔 我想学习理解

**什么时候用：** 你看到一段代码，想知道它是怎么工作的

**怎么说：**
```
"请解释这段代码是怎么工作的：

[粘贴代码]

请用简单的话解释，我是[你的水平，比如：初学者]。"
```

**真实例子：**
```
"请解释这段代码是怎么工作的：

const users = data.filter(user => user.age > 18)
                  .map(user => ({ name: user.name, email: user.email }))
                  .sort((a, b) => a.name.localeCompare(b.name));

请用简单的话解释，我是JavaScript初学者。"
```

## 🎯 前端开发的特殊注意事项

### 1. 🏷️ 说清楚你用什么技术

**❌ 技术不明确：**
```
"帮我做一个网页"
```
*AI不知道用什么技术，可能给出过时的代码*

**✅ 技术明确：**
```
"用HTML5、CSS3和现代JavaScript帮我做一个网页"
```
*或者*
```
"用Vue 3帮我做一个网页组件"
```

### 2. 📱 别忘了手机端

**❌ 只考虑电脑：**
```
"做一个导航菜单"
```

**✅ 考虑手机和电脑：**
```
"做一个导航菜单，要求：
- 在电脑上显示完整的菜单项
- 在手机上显示汉堡菜单（三条横线的按钮）
- 点击汉堡菜单可以展开/收起"
```

### 3. ⚡ 考虑网页速度

**❌ 不考虑性能：**
```
"做一个图片展示页面"
```

**✅ 考虑加载速度：**
```
"做一个图片展示页面，要求：
- 图片要懒加载（滚动到才加载）
- 先显示模糊的占位图
- 加载失败时显示默认图片
- 图片要压缩优化"
```

### 4. ♿ 让所有人都能用

**❌ 只考虑正常用户：**
```
"做一个弹窗"
```

**✅ 考虑特殊需求用户：**
```
"做一个弹窗，要求：
- 可以用键盘操作（Tab键切换，Esc键关闭）
- 颜色对比度要高，方便视力不好的人看清
- 支持屏幕阅读器
- 字体大小要合适"
```

## 🔧 让提示词越来越好用

### 1. 🔄 一步步改进

**第一次（太简单）：**
```
"做一个表单"
```
*AI给出的可能很基础*

**第二次（加点细节）：**
```
"做一个注册表单，要有用户名、邮箱、密码"
```
*好一些，但还不够具体*

**第三次（更具体）：**
```
"做一个用户注册表单，需要：
1. 用户名输入框（3-20个字符）
2. 邮箱输入框（要验证格式）
3. 密码输入框（至少8位）
4. 确认密码输入框
5. 提交按钮
6. 输入错误时要有提示"
```
*这样AI就知道具体要做什么了！*

**💡 小贴士：** 如果AI给出的结果不满意，不要重新开始，而是说"请改进一下，我还需要..."

### 2. 🧩 复杂功能分步做

**❌ 一次要求太多：**
```
"做一个购物车，要能显示商品、修改数量、删除商品、计算总价、选择规格、用优惠券、算运费、检查库存、有动画效果、适配手机"
```
*AI可能会漏掉一些功能，或者代码太复杂*

**✅ 分步骤来：**

**第一步：**
```
"先做一个简单的购物车，能显示商品列表、修改数量、删除商品、计算总价就行"
```

**第二步：**
```
"在刚才的购物车基础上，添加商品规格选择功能（颜色、尺寸）"
```

**第三步：**
```
"再添加优惠券功能和运费计算"
```

**💡 好处：** 每一步都能得到可以工作的代码，出问题也容易找到原因

### 3. 🎯 给AI看例子

**❌ 只说要什么：**
```
"做一个数据表格组件，要能排序、分页、选择行"
```
*AI不知道你想要什么样的接口*

**✅ 给出具体例子：**
```
"做一个数据表格组件，我希望这样使用：

<DataTable 
  data={用户列表}
  columns={[
    {name: 'name', title: '姓名', sortable: true},
    {name: 'age', title: '年龄', sortable: true},
    {name: 'email', title: '邮箱'}
  ]}
  pageSize={10}
  showCheckbox={true}
/>

请实现这个组件。"
```
*AI看到例子就知道要做成什么样了*

**💡 小贴士：** 如果你见过类似的组件（比如Element UI的表格），可以说"做一个类似Element UI表格的组件"

    result.push(users[i].name);
