### 1. JavaScript运行时错误

#### TypeError: Cannot read property 'xxx' of undefined
```markdown
# 调试提示词
我遇到了以下JavaScript错误，请帮我分析原因并提供解决方案：

**错误信息：**
```
TypeError: Cannot read property 'name' of undefined
    at UserProfile.jsx:15:23
    at renderWithHooks (react-dom.development.js:14803:18)
```

**相关代码：**
```vue
<template>
  <div>
    <h1>{{ user.name }}</h1>  <!-- 第15行 - 错误发生位置 -->
    <p>{{ user.email }}</p>
  </div>
</template>

<script setup>
import { ref, watchEffect } from 'vue'

const props = defineProps({
  userId: String
})

const user = ref(null)

watchEffect(() => {
  if (props.userId) {
    fetchUser(props.userId).then(userData => {
      user.value = userData
    })
  }
})
</script>
```

**环境信息：**
- Vue 3.3.0
- 浏览器：Chrome 120
- 发生时机：组件首次渲染时
```

**AI分析和解决方案：**

**问题分析：**
1. **根本原因**: 组件首次渲染时，`user` 状态为 `null`，尝试访问 `null.name` 导致错误
2. **时序问题**: `fetchUser` 是异步操作，在数据返回前组件已经开始渲染
3. **防御性编程缺失**: 没有对 `user` 为空的情况进行处理

**解决方案：**

```vue
<!-- 方案1：条件渲染 -->
<template>
  <div v-if="loading">Loading...</div>
  <div v-else-if="error">Error: {{ error }}</div>
  <div v-else-if="!user">User not found</div>
  <div v-else>
    <h1>{{ user.name }}</h1>
    <p>{{ user.email }}</p>
  </div>
</template>

<script setup>
import { ref, watchEffect } from 'vue'

const props = defineProps({
  userId: String
})

const user = ref(null)
const loading = ref(true)
const error = ref(null)

watchEffect(async () => {
  if (props.userId) {
    loading.value = true
    error.value = null
    
    try {
      const userData = await fetchUser(props.userId)
      user.value = userData
    } catch (err) {
      error.value = err.message
    } finally {
      loading.value = false
    }
  }
})
</script>

<!-- 方案2：可选链操作符 -->
<template>
  <div>
    <h1>{{ user?.name || 'Loading...' }}</h1>
    <p>{{ user?.email || 'Loading...' }}</p>
  </div>
</template>

<script setup>
import { ref, watchEffect } from 'vue'

const props = defineProps({
  userId: String
})

const user = ref(null)

watchEffect(() => {
  if (props.userId) {
    fetchUser(props.userId).then(userData => {
      user.value = userData
    })
  }
})
</script>

<!-- 方案3：默认值 -->
<template>
  <div>
    <h1>{{ user.name || 'Loading...' }}</h1>
    <p>{{ user.email || 'Loading...' }}</p>
  </div>
</template>

<script setup>
import { ref, watchEffect } from 'vue'

const props = defineProps({
  userId: String
})

const user = ref({
  name: '',
  email: ''
})

watchEffect(() => {
  if (props.userId) {
    fetchUser(props.userId).then(userData => {
      user.value = userData || { name: '', email: '' }
    })
  }
})
</script>
```

**预防措施：**
1. 使用TypeScript进行类型检查
2. 实现统一的加载状态管理
3. 使用ESLint规则检查潜在的空值访问

### 2. Vue 3 组合式API错误

#### 响应式数据丢失错误
```markdown
# 调试提示词
我的Vue 3组件出现了响应式相关的错误：

**错误信息：**
```
[Vue warn]: Invalid prop: type check failed for prop "data". Expected Object, got undefined.
    at <AdvancedComponent data=undefined >
    at <ConditionalComponent showAdvanced=true >
```

**问题代码：**
```vue
<template>
  <div>
    <BasicComponent :data="basicData" />
    <AdvancedComponent v-if="showAdvanced" :data="advancedData" />
  </div>
</template>

<script setup>
import { ref, watchEffect } from 'vue'

const props = defineProps({
  showAdvanced: Boolean
})

const basicData = ref(null)
let advancedData = null  // 错误：非响应式数据

watchEffect(() => {
  fetchBasicData().then(data => {
    basicData.value = data
  })
  
  if (props.showAdvanced) {
    fetchAdvancedData().then(data => {
      advancedData = data  // 错误：直接赋值给非响应式变量
    })
  }
})
</script>
```
```

**AI分析和解决方案：**

**问题分析：**
1. **响应式丢失**: `advancedData` 不是响应式数据，组件无法追踪其变化
2. **数据传递问题**: 非响应式数据传递给子组件导致渲染问题
3. **Vue 3机制**: Vue 3依赖响应式系统来触发重新渲染

**解决方案：**

```vue
<!-- 方案1：始终使用响应式数据 -->
<template>
  <div>
    <BasicComponent :data="basicData" />
    <AdvancedComponent v-if="showAdvanced && advancedData" :data="advancedData" />
  </div>
</template>

<script setup>
import { ref, watchEffect } from 'vue'

const props = defineProps({
  showAdvanced: Boolean
})

const basicData = ref(null)
const advancedData = ref(null)  // 正确：使用响应式数据

watchEffect(() => {
  fetchBasicData().then(data => {
    basicData.value = data
  })
  
  if (props.showAdvanced) {
    fetchAdvancedData().then(data => {
      advancedData.value = data  // 正确：更新响应式数据
    })
  } else {
    advancedData.value = null  // 重置状态
  }
})
</script>

<!-- 方案2：组件拆分 -->
<!-- BasicComponent.vue -->
<template>
  <div><!-- 基础组件渲染 --></div>
</template>

<script setup>
defineProps({
  basicData: Object
})
</script>

<!-- AdvancedComponent.vue -->
<template>
  <div><!-- 高级组件渲染 --></div>
</template>

<script setup>
import { ref, onMounted } from 'vue'

const advancedData = ref(null)

onMounted(() => {
  fetchAdvancedData().then(data => {
    advancedData.value = data
  })
})
</script>

<!-- ConditionalComponent.vue -->
<template>
  <div>
    <BasicComponent :data="basicData" />
    <AdvancedComponent v-if="showAdvanced" />
  </div>
</template>

<script setup>
import { ref } from 'vue'
import BasicComponent from './BasicComponent.vue'
import AdvancedComponent from './AdvancedComponent.vue'

defineProps({
  showAdvanced: Boolean
})

const basicData = ref(null)
</script>

<!-- 方案3：组合式函数 -->
<script>
// composables/useConditionalData.js
import { ref, watchEffect } from 'vue'

export function useConditionalData(condition) {
  const data = ref(null)
  
  watchEffect(() => {
    if (condition.value) {
      fetchAdvancedData().then(result => {
        data.value = result
      })
    } else {
      data.value = null
    }
  })
  
  return data
}
</script>

<!-- ConditionalComponent.vue -->
<template>
  <div>
    <BasicComponent :data="basicData" />
    <AdvancedComponent v-if="showAdvanced" :data="advancedData" />
  </div>
</template>

<script setup>
import { ref, toRef } from 'vue'
import { useConditionalData } from '@/composables/useConditionalData'
import BasicComponent from './BasicComponent.vue'
import AdvancedComponent from './AdvancedComponent.vue'

const props = defineProps({
  showAdvanced: Boolean
})

const basicData = ref(null)
const advancedData = useConditionalData(toRef(props, 'showAdvanced'))
</script>
```

- 输入框输入时整个页面卡顿
- Vue DevTools显示大量组件重新渲染
- 页面响应延迟明显

**相关代码：**
```vue
<template>
  <div>
    <SearchInput

import { ref, computed } from 'vue'

import ResultList from './ResultList.vue'

const searchTerm = ref('')
const results = ref([])
const filters = ref({})

const filteredResults = computed(() => {
  return results.value.filter(item => {
    return item.name.includes(searchTerm.value) && 
           Object.keys(filters.value).every(key => 
             !filters.value[key] || item[key] === filters.value[key]
           )
  })
})

const setSearchTerm = (value) => {
  searchTerm.value = value
}

const setFilters = (value) => {
  filters.value = value
}
</script>

<!-- ResultList.vue -->
<template>
  <div>
    <ResultItem 
      v-for="item in results" 
      :key="item.id" 
      :item="item" 
    />
  </div>

<script setup>
defineProps({
  results: Array
})
</script>
```

请分析性能问题并提供优化方案。
```

**AI分析和优化方案：**

**性能问题分析：**
1. **计算密集**: `filteredResults` 在每次输入时重新计算大量数据
2. **缺少防抖**: 搜索词每次变化都触发过滤计算
3. **子组件重渲染**: `ResultList` 因为 `results` 变化而重渲染所有子项
4. **缺少优化**: 没有使用 `shallowRef`、`markRaw` 等优化API

**优化方案：**

```vue
<!-- 优化后的主组件 -->
<template>
  <div>
    <SearchInput 
      :value="searchTerm" 
      @update:value="handleSearchChange"
      @filter-change="handleFilterChange"
    />
    <ResultList :results="filteredResults" />
  </div>
</template>

<script setup>
import { ref, computed, shallowRef, useDebounceFn } from 'vue'
import SearchInput from './SearchInput.vue'
import ResultList from './ResultList.vue'

const searchTerm = ref('')
const debouncedSearchTerm = ref('')
const results = shallowRef([])  // 使用 shallowRef 优化大数组
const filters = ref({})

const handleSearchChange = useDebounceFn((term) => {
  debouncedSearchTerm.value = term
}, 300)

// 处理搜索词变化
const handleSearchTermChange = (term) => {
  searchTerm.value = term
  handleSearchChange(term)
}

// 优化过滤逻辑
const filteredResults = computed(() => {
  if (!debouncedSearchTerm.value && Object.keys(filters.value).length === 0) {
    return results.value
  }
  
  return results.value.filter(item => {
    // 搜索词匹配
    const matchesSearch = !debouncedSearchTerm.value || 
      item.name.toLowerCase().includes(debouncedSearchTerm.value.toLowerCase())
    
    // 过滤器匹配
    const matchesFilters = Object.keys(filters.value).every(key => 
      !filters.value[key] || item[key] === filters.value[key]
    )
    
    return matchesSearch && matchesFilters
  })
})

// 优化过滤器更新
const handleFilterChange = (newFilters) => {
  filters.value = { ...filters.value, ...newFilters }
}
</script>

<!-- 优化的结果列表组件 -->
<!-- ResultList.vue -->
<template>
  <div class="result-list">
    <ResultItem 
      v-for="item in results" 
      :key="item.id" 
      :item="item" 
    />
  </div>
</template>

<script setup>
import { onUpdated } from 'vue'
import ResultItem from './ResultItem.vue'

const props = defineProps({
  results: Array
})

onUpdated(() => {
  console.log('ResultList rendered with', props.results.length, 'items')
})
</script>

<!-- 优化的结果项组件 -->
<!-- ResultItem.vue -->
<template>
  <div class="result-item" @click="handleClick">
    <h3>{{ item.name }}</h3>
    <p>{{ item.description }}</p>
  </div>
</template>

<script setup>
const props = defineProps({
  item: Object
})

const handleClick = () => {
  // 处理点击事件
  console.log('Item clicked:', props.item.id)
}
</script>

<!-- 虚拟滚动版本（适用于大量数据） -->
<!-- VirtualizedResultList.vue -->

  <RecycleScroller
    class="scroller"

    :item-size="100"
    key-field="id"

// composables/usePerformanceMonitor.js
import { ref, onUpdated } from 'vue'

    startTime = performance.now()
  })
  
  return { renderCount }
}

**AI分析和解决方案：**

**问题分析：**
1. **视口单位问题**: `100vh` 在移动设备上可能包含地址栏高度
2. **Flexbox收缩**: 移动端可能出现意外的收缩行为
3. **滚动容器**: 嵌套滚动容器可能导致滚动问题
4. **媒体查询优先级**: 可能存在样式覆盖问题

**解决方案：**

```css
/* 修复后的CSS */

  display: flex;
  flex-direction: row;
  width: 100%;
  /* 使用动态视口高度，兼容移动设备 */
  min-height: 100vh;
  min-height: 100dvh; /* 动态视口高度 */
}

.sidebar {
  width: 300px;
  background: #f5f5f5;
  flex-shrink: 0;

// 内存泄漏检测工具
class MemoryLeakDetector {
  constructor() {
    this.components = new Set()
    this.timers = new Set()
    this.listeners = new Set()
  }
  
  trackComponent(name) {
    this.components.add(name)
    console.log(`Component ${name} mounted. Total: ${this.components.size}`)
    
    return () => {
      this.components.delete(name)
      console.log(`Component ${name} unmounted. Total: ${this.components.size}`)
    }
  }
  
  trackTimer(id) {
    this.timers.add(id)
    return () => {
      this.timers.delete(id)
      clearTimeout(id)
    }
  }
  
  trackListener(element, event, handler) {
    const listener = { element, event, handler }
    this.listeners.add(listener)
    element.addEventListener(event, handler)
    
    return () => {
      this.listeners.delete(listener)
      element.removeEventListener(event, handler)
    }
  }
  
  getReport() {
    return {
      components: this.components.size,
      timers: this.timers.size,
      listeners: this.listeners.size
    }
  }
}

// 使用示例
const detector = new MemoryLeakDetector()

function MyComponent() {
  useEffect(() => {
    const cleanup = detector.trackComponent('MyComponent')
    return cleanup
  }, [])
  
  useEffect(() => {
    const timer = setTimeout(() => {}, 1000)
    const cleanupTimer = detector.trackTimer(timer)
    return cleanupTimer
  }, [])
  
  return <div>Component content</div>
}

// 高级错误边界组件
class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props)
    this.state = { 
      hasError: false, 
      error: null, 
      errorInfo: null 
    }
  }
  
  static getDerivedStateFromError(error) {
    return { hasError: true }
  }
  
  componentDidCatch(error, errorInfo) {
    this.setState({
      error,
      errorInfo
    })
    
    // 错误上报
    this.reportError(error, errorInfo)

        stack: error.stack,
        name: error.name
      },
      errorInfo: {
        componentStack: errorInfo.componentStack
      },
      props: this.props
    }
    
    // 发送到错误监控服务
    if (window.Sentry) {
      window.Sentry.captureException(error, {
        contexts: { errorInfo }
      })
    }
    
    // 或发送到自定义API
    fetch('/api/errors', {
      method: 'POST',

      body: JSON.stringify(errorReport)

          </details>

          </button>
        </div>

class MicroAppEventBus {
  constructor() {
    this.events = new Map()
  }
  
  on(event, callback) {
    if (!this.events.has(event)) {

    }

  setupListeners() {
    window.addEventListener('online', () => {
      this.isOnline = true
      this.handleNetworkChange('online')
    })
    
    window.addEventListener('offline', () => {
      this.isOnline = false
      this.handleNetworkChange('offline')
    })
    
    if (this.connection) {
      this.connection.addEventListener('change', () => {
        this.handleConnectionChange()
      })
    }
  }
  
  handleNetworkChange(status) {
    // 显示网络状态提示
    const toast = document.createElement('div')
    toast.className = `network-toast ${status}`
    toast.textContent = status === 'online' ? '网络已连接' : '网络已断开'
    document.body.appendChild(toast)
    
    setTimeout(() => {
      document.body.removeChild(toast)
    }, 3000)
  }
  
  handleConnectionChange() {
    const { effectiveType, downlink } = this.connection
    
    // 根据网络状况调整策略
    if (effectiveType === '2g' || downlink < 0.5) {
      // 启用省流量模式
      this.enableDataSavingMode()
    } else {
      this.disableDataSavingMode()
    }
  }
  
  enableDataSavingMode() {
    document.body.classList.add('data-saving-mode')
    // 停止自动播放视频
    document.querySelectorAll('video[autoplay]').forEach(video => {
      video.pause()
    })
  }
  
  disableDataSavingMode() {
    document.body.classList.remove('data-saving-mode')
  }
}

   - 能说出具体的技术选型原因
   - 能分析问题的根本原因

   - 有团队协作和项目管理的经验
   - 能从业务角度思考技术决策

4. **持续学习能力**：
   - 能跟上技术发展趋势
   - 有自己的技术博客或开源项目
   - 能分享技术经验和最佳实践

- **实施过程**：具体怎么实现的，遇到了什么困难
- **效果评估**：最终达到了什么效果，有什么数据支撑
