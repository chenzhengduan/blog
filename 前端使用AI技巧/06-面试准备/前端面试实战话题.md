## 🎯 性能优化实战经验

### 话题："你是如何解决首屏加载慢的问题的？"

**回答思路：**
```markdown
我在之前的项目中遇到过首屏加载时间超过5秒的问题，主要通过以下几个方面进行了优化：

**问题分析：**
- 使用 Lighthouse 和 WebPageTest 分析，发现主要问题是 JavaScript 包过大（2.5MB）
- Network 面板显示有多个大图片资源未压缩
- 关键渲染路径被阻塞

**解决方案：**
1. **代码分割**：
   - 使用 React.lazy() 和 Suspense 实现路由级别的代码分割
   - 将第三方库（如 moment.js）替换为更轻量的 day.js
   - 使用 webpack-bundle-analyzer 分析包大小

2. **资源优化**：
   - 图片使用 WebP 格式，并提供 fallback
   - 实现图片懒加载，使用 Intersection Observer API
   - 启用 Gzip 压缩和 HTTP/2

3. **关键渲染路径优化**：
   - 内联关键 CSS，非关键 CSS 异步加载
   - 使用 preload 预加载关键资源
   - 实现骨架屏提升用户体验

**最终效果：**
- 首屏加载时间从 5.2s 降低到 1.8s
- Lighthouse 性能评分从 45 提升到 92
- 用户跳出率降低了 23%
```

```javascript
// 图片懒加载 Hook
function useImageLazyLoad() {
  const [isLoaded, setIsLoaded] = useState(false)
  const imgRef = useRef()
  
  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          setIsLoaded(true)
          observer.disconnect()
        }
      },
      { threshold: 0.1 }
    )
    
    if (imgRef.current) {
      observer.observe(imgRef.current)
    }
    
    return () => observer.disconnect()
  }, [])
  
  return { isLoaded, imgRef }
}

            metric: 'LCP',
            value: entry.startTime,

          })
        }
      })
```

## 🏗️ 微前端架构实践

### 话题："你在微前端项目中遇到过什么技术挑战？"

**回答思路：**
```markdown
我参与了公司从单体应用向微前端架构的迁移，主要负责基座应用和子应用的通信机制：

**项目背景：**
- 原有单体应用代码量超过 50 万行，多团队协作困难
- 不同业务模块技术栈不统一（React、Vue、Angular）
- 部署耦合严重，一个模块的问题影响整个系统

**技术方案：**
- 使用 qiankun 作为微前端框架
- 基座应用负责路由分发、用户认证、公共组件
- 子应用独立开发、测试、部署

**核心挑战和解决方案：**

1. **应用间通信**：
   - 实现了基于 EventBus 的通信机制
   - 设计了统一的数据格式和通信协议
   - 使用 SharedStore 共享关键状态

2. **样式隔离**：
   - 启用 qiankun 的样式沙箱
   - 制定 CSS 命名规范，使用 BEM 方法论
   - 公共样式通过 CSS 变量统一管理

3. **依赖共享**：
   - 基座应用提供 React、ReactDOM 等公共依赖
   - 使用 webpack externals 避免重复打包
   - 实现了动态依赖加载机制
```

**技术实现细节：**
```javascript
// 基座应用 - 主应用配置
import { registerMicroApps, start } from 'qiankun'

// 应用间通信 EventBus
class MicroAppEventBus {
  constructor() {
    this.events = new Map()
  }
  
  on(event, callback) {
    if (!this.events.has(event)) {
      this.events.set(event, [])
    }
    this.events.get(event).push(callback)
  }
  
  emit(event, data) {
    if (this.events.has(event)) {
      this.events.get(event).forEach(callback => callback(data))
    }
  }
  
  off(event, callback) {
    if (this.events.has(event)) {
      const callbacks = this.events.get(event)
      const index = callbacks.indexOf(callback)
      if (index > -1) {
        callbacks.splice(index, 1)
      }
    }
  }
}

const eventBus = new MicroAppEventBus()

// 注册微应用
registerMicroApps([
  {
    name: 'user-management',
    entry: '//localhost:3001',
    container: '#subapp-container',
    activeRule: '/user',
    props: {
      eventBus,
      getGlobalState: () => store.getState(),
      setGlobalState: (state) => store.dispatch(updateGlobalState(state))
    }
  },
  {
    name: 'order-management',
    entry: '//localhost:3002',
    container: '#subapp-container',
    activeRule: '/order',
    props: { eventBus }
  }
])

// 子应用 - 生命周期函数
export async function bootstrap() {
  console.log('子应用启动')
}

export async function mount(props) {
  const { container, eventBus } = props
  
  // 监听全局事件
  eventBus.on('user-logout', () => {
    // 清理子应用状态
    store.dispatch(clearUserData())
  })
  
  // 渲染子应用
  ReactDOM.render(
    <App eventBus={eventBus} />,
    container ? container.querySelector('#root') : document.getElementById('root')
  )
}

export async function unmount(props) {
  const { container } = props
  ReactDOM.unmountComponentAtNode(
    container ? container.querySelector('#root') : document.getElementById('root')
  )
}
```

## 📱 移动端适配和性能优化

### 话题："移动端开发中你遇到过哪些坑？"

**回答思路：**
```markdown
在开发移动端 H5 应用时，我遇到了很多兼容性和性能问题：

**主要挑战：**
1. **iOS Safari 的各种兼容性问题**
2. **Android 设备的性能差异**
3. **不同屏幕尺寸的适配**
4. **网络环境不稳定的处理**

**具体问题和解决方案：**

1. **iOS Safari 滚动问题**：
   - 问题：-webkit-overflow-scrolling: touch 导致的滚动卡顿
   - 解决：使用 better-scroll 库替代原生滚动

2. **软键盘弹起问题**：
   - 问题：iOS 软键盘弹起时页面布局错乱
   - 解决：监听 resize 事件，动态调整视口高度

3. **1px 边框问题**：
   - 问题：高 DPR 设备上 1px 边框显示过粗
   - 解决：使用伪元素 + transform: scale(0.5)

4. **图片加载优化**：
   - 问题：大图片在移动端加载慢
   - 解决：实现渐进式图片加载和 WebP 格式支持
```

**技术实现：**

```javascript
// 移动端适配工具类
class MobileAdapter {
  constructor() {
    this.setupViewport()
    this.setupSafeArea()
    this.handleKeyboard()
  }
  
  // 设置视口
  setupViewport() {
    const viewport = document.querySelector('meta[name="viewport"]')
    if (viewport) {
      viewport.content = 'width=device-width,initial-scale=1,maximum-scale=1,user-scalable=no'
    }
  }
  
  // 处理安全区域
  setupSafeArea() {
    // 检测是否支持安全区域
    if (CSS.supports('padding: env(safe-area-inset-top)')) {
      document.documentElement.style.setProperty(
        '--safe-area-top', 
        'env(safe-area-inset-top)'
      )
      document.documentElement.style.setProperty(
        '--safe-area-bottom', 
        'env(safe-area-inset-bottom)'
      )
    }
  }
  
  // 处理软键盘
  handleKeyboard() {
    let originalHeight = window.innerHeight
    
    window.addEventListener('resize', () => {
      const currentHeight = window.innerHeight
      const heightDiff = originalHeight - currentHeight
      
      // 判断是否是软键盘弹起
      if (heightDiff > 150) {
        document.body.classList.add('keyboard-open')
        // 调整页面布局
        this.adjustLayoutForKeyboard(heightDiff)
      } else {
        document.body.classList.remove('keyboard-open')
        this.resetLayout()
      }
    })
  }
  
  adjustLayoutForKeyboard(heightDiff) {
    const activeElement = document.activeElement
    if (activeElement && activeElement.tagName === 'INPUT') {
      // 将输入框滚动到可视区域
      setTimeout(() => {
        activeElement.scrollIntoView({ behavior: 'smooth', block: 'center' })
      }, 300)
    }
  }
  
  resetLayout() {
    // 重置布局
  }
}
```
