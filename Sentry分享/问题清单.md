# 异常场景问题清单

> 记录在没有 Sentry 工具时常见的异常场景和问题，用于团队内部培训
> 
> **注意**：本清单主要针对前端 Sentry 使用场景，数据库相关问题由后端团队负责处理

## 分类说明
- **优先级**：高/中/低（综合考虑影响范围、发生频率、解决难度）
- **技术栈**：按问题发生的主要技术领域分类
- **职责范围**：★ 表示前端重点关注，☆ 表示需要与后端协作

---

## 1. 网络/网关问题

### 1.1 参数过大导致网关拦截
- **优先级**：高
- **问题现象**：客户端请求后台接口，参数过大，网关拦截，阿里日志拿不到请求参数
- **影响范围**：影响接口调用，无法获取完整的请求信息进行问题排查
- **传统排查方式**：查看网关日志、检查参数大小限制、手动测试不同参数大小
- **使用 Sentry 后的改进效果**：可以捕获到被网关拦截前的请求信息，快速定位参数问题

---

## 2. 前端问题

### 2.1 JavaScript 运行时错误
- **优先级**：高
- **问题现象**：页面白屏、功能异常、控制台报错但无法定位具体原因
- **影响范围**：影响用户体验，可能导致功能完全不可用
- **传统排查方式**：查看浏览器控制台、添加 console.log、手动复现问题
- **使用 Sentry 后的改进效果**：自动捕获 JS 错误，提供完整的错误堆栈、用户环境信息、错误上下文

### 2.2 网络请求失败
- **优先级**：高
- **问题现象**：API 调用失败、接口超时、网络异常
- **影响范围**：影响数据展示和用户操作
- **传统排查方式**：查看 Network 面板、检查接口状态码、手动测试接口
- **使用 Sentry 后的改进效果**：自动记录失败的请求详情，包括请求参数、响应状态、错误信息

### 2.3 性能问题
- **优先级**：中
- **问题现象**：页面加载慢、操作卡顿、内存泄漏
- **影响范围**：影响用户体验，可能导致用户流失
- **传统排查方式**：使用 Chrome DevTools、Lighthouse 测试、手动观察
- **使用 Sentry 后的改进效果**：自动监控页面性能指标，识别性能瓶颈

---

## 3. 接口调用问题 ☆

### 3.1 接口异常（前端视角）
- **优先级**：高
- **问题现象**：接口返回 500 错误、请求超时、网络异常
- **影响范围**：影响前端功能，可能导致整个业务流程中断
- **前端职责**：捕获和记录接口调用异常，提供用户友好的错误提示
- **传统排查方式**：查看 Network 面板、检查请求参数、手动测试接口
- **使用 Sentry 后的改进效果**：自动捕获接口异常，记录请求参数、响应状态、用户操作上下文

### 3.2 接口参数问题
- **优先级**：高
- **问题现象**：参数格式错误、必填参数缺失、参数类型不匹配
- **影响范围**：导致接口调用失败，影响业务功能
- **前端职责**：参数校验、格式化、错误处理
- **传统排查方式**：手动检查参数、添加 console.log、查看接口文档
- **使用 Sentry 后的改进效果**：记录完整的请求参数和错误信息，快速定位参数问题

### 3.3 跨域和权限问题
- **优先级**：中
- **问题现象**：CORS 错误、401/403 权限异常、Token 过期
- **影响范围**：影响接口调用，可能导致用户无法正常使用功能
- **前端职责**：处理跨域配置、Token 刷新、权限验证
- **传统排查方式**：查看控制台错误、检查请求头、手动测试权限
- **使用 Sentry 后的改进效果**：自动记录权限相关错误，包含用户信息和请求上下文

---

## 4. 数据处理问题 ★

### 4.1 数据格式异常
- **优先级**：高
- **问题现象**：接口返回数据格式不符合预期、字段缺失、类型错误
- **影响范围**：导致页面渲染异常、功能错误、用户体验差
- **前端职责**：数据校验、容错处理、默认值设置
- **传统排查方式**：手动检查接口返回、添加数据校验、查看接口文档
- **使用 Sentry 后的改进效果**：自动捕获数据异常，记录完整的数据结构和错误上下文

### 4.2 大数据量渲染
- **优先级**：中
- **问题现象**：页面卡顿、渲染缓慢、浏览器假死
- **影响范围**：影响用户体验，可能导致页面无响应
- **前端职责**：虚拟滚动、分页加载、性能优化
- **传统排查方式**：使用 Performance 面板、手动测试大数据、代码审查
- **使用 Sentry 后的改进效果**：监控渲染性能，记录数据量和渲染时间

### 4.3 状态管理异常
- **优先级**：中
- **问题现象**：状态不一致、数据丢失、组件状态错误
- **影响范围**：影响应用状态，可能导致功能异常
- **前端职责**：状态管理、数据同步、状态持久化
- **传统排查方式**：使用 Redux DevTools、手动调试状态、代码审查
- **使用 Sentry 后的改进效果**：记录状态变化过程，便于分析状态异常原因

## 5. 部署和环境问题 ☆

### 5.1 静态资源加载失败
- **优先级**：高
- **问题现象**：JS/CSS 文件 404、CDN 异常、资源加载超时
- **影响范围**：影响页面正常显示和功能，可能导致白屏
- **前端职责**：资源路径配置、CDN 容灾、加载失败处理
- **传统排查方式**：检查 Network 面板、手动测试资源链接、查看构建配置
- **使用 Sentry 后的改进效果**：自动监控资源加载状态，记录失败的资源信息

### 5.2 环境配置错误
- **优先级**：中
- **问题现象**：API 地址错误、环境变量缺失、配置不匹配
- **影响范围**：影响功能正常使用，可能导致数据错乱
- **前端职责**：环境配置管理、配置校验、环境切换
- **传统排查方式**：检查配置文件、手动验证环境变量、查看构建日志
- **使用 Sentry 后的改进效果**：记录环境信息，便于快速定位配置问题

### 5.3 版本兼容性问题
- **优先级**：中
- **问题现象**：新版本功能异常、API 不兼容、浏览器兼容性问题
- **影响范围**：影响特定用户群体，可能导致功能不可用
- **前端职责**：版本管理、兼容性测试、渐进式升级
- **传统排查方式**：手动测试不同版本、查看兼容性文档、用户反馈
- **使用 Sentry 后的改进效果**：按版本统计错误，快速识别版本相关问题

---

## 6. 第三方集成问题 ★

### 6.1 第三方 SDK 异常
- **优先级**：高
- **问题现象**：第三方库报错、SDK 初始化失败、功能不可用
- **影响范围**：影响依赖第三方服务的功能，如支付、地图、统计等
- **前端职责**：SDK 集成、错误处理、降级方案
- **传统排查方式**：查看第三方文档、检查集成代码、联系技术支持
- **使用 Sentry 后的改进效果**：自动捕获第三方 SDK 错误，记录集成上下文和环境信息

### 6.2 跨域和安全策略
- **优先级**：中
- **问题现象**：CSP 策略阻止、iframe 嵌入失败、第三方资源被阻止
- **影响范围**：影响第三方功能集成，可能导致功能不完整
- **前端职责**：安全策略配置、跨域处理、权限申请
- **传统排查方式**：查看控制台安全警告、检查 CSP 配置、手动测试
- **使用 Sentry 后的改进效果**：记录安全策略相关错误，便于快速调整配置

### 6.3 外部服务依赖
- **优先级**：中
- **问题现象**：外部服务不可用、API 限流、服务降级
- **影响范围**：影响依赖外部服务的功能，需要降级处理
- **前端职责**：服务可用性检测、降级方案、用户提示
- **传统排查方式**：手动测试外部服务、查看服务状态页、联系服务商
- **使用 Sentry 后的改进效果**：监控外部服务调用状态，自动记录服务异常
