## 3. Sentry 核心概念（8 分钟）

### 3.1 什么是 Sentry
- 实时错误监控和性能监控平台
- 自动捕获、聚合、分析错误
- 提供完整的错误上下文和用户环境

### 3.2 核心功能
- **错误捕获**：自动捕获 JS 错误、Promise 异常
- **性能监控**：页面加载、接口调用性能
- **用户上下文**：错误发生时的用户信息和操作路径
- **告警通知**：实时错误告警，支持多种通知方式
- **错误聚合**：相同错误自动聚合，避免重复告警

### 3.3 前端职责范围
- ★ 前端重点关注：JS 错误、性能问题、第三方集成
- ☆ 需要协作：接口异常、部署问题
- 后端负责：数据库、服务器、基础设施监控

### 4.5.3 分版本升级指南

#### v6 → v7 升级（重点）
**主要变更：**
- 包名变更：`@sentry/browser` 替代 `raven-js`
- 初始化方式改变
- 错误捕获API更新

**迁移步骤：**
```typescript
// 旧版本 (v6)
import Raven from 'raven-js';
Raven.config('DSN').install();
Raven.captureException(error);

// 新版本 (v7+)
import * as Sentry from '@sentry/browser';
Sentry.init({ dsn: 'DSN' });
Sentry.captureException(error);
```

#### v7 → v8 升级
**主要变更：**
- 性能监控集成优化
- 配置选项调整
- 类型定义改进

```typescript
// v7 配置
Sentry.init({
  dsn: 'DSN',
  integrations: [
    new Integrations.BrowserTracing()
  ]
});

// v8 配置
Sentry.init({
  dsn: 'DSN',
  integrations: [
    new BrowserTracing()
  ]
});
```

```typescript
// 在 Axios 拦截器中捕获
axios.interceptors.response.use(
  res => res,
  err => {
    Sentry.captureException(err, {
      tags: { type: "API_ERROR", url: err.config?.url }
    });
    return Promise.reject(err);
  }
);
```

## 6. 最佳实践和规范（8 分钟）

### 6.1 团队规范
1. **统一配置**：使用团队统一的 Sentry 配置模板
2. **数据脱敏**：必须过滤敏感信息（token、密码、个人信息）
3. **标签规范**：统一的标签命名和分类规则
4. **告警处理**：建立错误处理和响应流程

### 6.2 工具类封装
```typescript
// 封装统一的错误上报工具
class Logger {
  static captureException(error: Error, context?: any) {
    Sentry.withScope(scope => {
      if (context) {
        scope.setExtras(context);
      }
      scope.setTag('source', 'frontend');
      Sentry.captureException(error);
    });
  }
}
```

### 6.3 成本控制
- **采样率设置**：生产环境 0.1-0.3，开发环境 1.0
- **错误过滤**：忽略无意义的错误（如第三方脚本错误）
- **数据量控制**：合理设置 maxValueLength 和 normalizeDepth

## 7. Sentry 平台实战演示（15 分钟）

### 7.1 平台界面介绍（配合截图）

#### 主要功能区域
- **Issues（问题列表）**：所有错误的汇总视图
- **Performance（性能监控）**：页面和接口性能数据
- **Releases（版本管理）**：按版本查看错误分布
- **Alerts（告警规则）**：配置错误告警和通知
- **Settings（项目设置）**：DSN、集成配置等

#### 关键指标解读
- **Error Rate**：错误率趋势
- **Affected Users**：受影响用户数
- **Event Volume**：事件数量统计
- **APDEX Score**：应用性能指数

### 7.2 错误查询和筛选技巧

#### 基础筛选
```
# 按环境筛选
environment:production

# 按版本筛选
release:"my-app@1.2.0"

# 按用户筛选
user.id:"12345"

# 按标签筛选
tag:module:"payment"
```

#### 高级查询语法
```
# 组合条件
environment:production AND release:"my-app@1.2.0"

# 排除条件
!browser.name:"Chrome"

# 时间范围
timestamp:>2024-01-01

# 错误类型
error.type:"TypeError"

# URL 匹配
url:"*/checkout/*"
```

#### 实用查询示例
```
# 查找支付相关错误
tag:module:"payment" AND environment:production

# 查找特定用户的错误
user.id:"user123" OR user.email:"*@example.com"

# 查找移动端错误
tag:platform:"mobile" AND browser.name:"Mobile Safari"

# 查找接口超时错误
message:"*timeout*" OR message:"*超时*"

# 查找新版本引入的错误
release:"my-app@2.0.0" AND first_seen:>2024-01-15
```

### 7.3 错误详情分析（配合截图演示）

#### 错误概览信息
- **错误标题和类型**：快速了解错误性质
- **发生频率**：错误发生的时间分布
- **影响用户**：受影响的用户数量和列表
- **首次/最后发生时间**：错误的生命周期

#### 错误堆栈分析
- **完整堆栈信息**：精确定位错误位置
- **Source Map 支持**：还原压缩代码
- **代码上下文**：错误发生前后的代码
- **变量状态**：错误发生时的变量值

#### 用户上下文信息
- **用户信息**：ID、邮箱、用户名等
- **设备信息**：浏览器、操作系统、设备类型
- **操作路径**：用户操作的面包屑轨迹
- **自定义标签**：业务相关的标签信息

#### 请求上下文
- **请求 URL**：错误发生的页面地址
- **请求参数**：GET/POST 参数
- **请求头信息**：User-Agent、Referer 等
- **接口调用**：相关的 API 请求信息

### 7.4 性能监控查看

#### 页面性能指标
- **LCP（Largest Contentful Paint）**：最大内容绘制时间
- **FID（First Input Delay）**：首次输入延迟
- **CLS（Cumulative Layout Shift）**：累积布局偏移
- **TTFB（Time to First Byte）**：首字节时间

#### 接口性能分析
- **响应时间分布**：P50、P75、P95、P99
- **错误率统计**：按状态码分类
- **吞吐量趋势**：请求量变化
- **慢查询识别**：超时和慢响应接口

### 7.5 告警配置演示

#### 告警规则类型
- **错误率告警**：错误率超过阈值时触发
- **新错误告警**：出现新类型错误时触发
- **用户影响告警**：受影响用户数超过阈值
- **性能告警**：页面性能指标异常时触发

#### 通知方式配置
- **邮件通知**：发送到指定邮箱
- **Slack 集成**：发送到团队频道
- **钉钉/企业微信**：国内常用的团队协作工具
- **Webhook**：自定义通知处理
