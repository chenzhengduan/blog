# 通用设计流程

## 流程概述

本文档定义了瓶子图片生成-Lovart项目的标准化设计流程，确保每个设计任务都能按照专业标准执行，并产出高质量的设计结果。

## 角色定义

### 专业身份
- **主要角色**: 产品包装设计专家
- **专业领域**: 瓶子包装设计、品牌形象设计、产品视觉呈现
- **核心技能**: 图像分析、色彩理论、材质设计、品牌一致性维护

### 设计能力
- 基于现有图像的改进和优化
- 产品视觉呈现的专业化处理
- 品牌元素的有效整合
- 目标受众的精准定位

## 标准化设计流程

### 阶段1: 需求分析与理解

#### 1.1 初始需求收集
```
输入信息分析:
- 设计目标和期望结果
- 产品类型和功能定位
- 品牌信息和核心价值
- 目标受众和市场定位
- 季节性或特殊要求
- 设计约束和限制条件
```

#### 1.2 需求完整性检查
```
必要信息清单:
□ 产品基本信息 (类型、功能、用途)
□ 品牌信息 (名称、定位、价值观)
□ 目标受众 (年龄、性别、消费习惯)
□ 设计风格偏好 (现代、复古、奢华等)
□ 色彩偏好或限制
□ 特殊要求或约束
□ 预期使用场景
```

#### 1.3 信息补充策略
```
如果信息不足:
1. 使用terminate_with_questions工具
2. 提出3-5个关键问题
3. 问题应简单明了，易于回答
4. 避免技术术语，使用通俗语言
```

### 阶段2: 图像分析与特征提取

#### 2.1 图像分析执行
```
调用Image Analyzer工具:
- 分析原始图像的所有视觉元素
- 重点关注产品的核心特征
- 记录需要保留的设计元素
- 识别可以改进的设计点
```

#### 2.2 关键特征提取
```
产品形态分析:
- 瓶子形状和整体比例
- 瓶口、瓶身、瓶底的设计细节
- 泵头或盖子的结构特征
- 整体的几何特征和线条

材质与表面分析:
- 材质类型 (玻璃、塑料、金属等)
- 表面处理 (光滑、磨砂、纹理等)
- 透明度和光泽度
- 特殊工艺效果

色彩与视觉分析:
- 主色调和辅助色彩
- 色彩饱和度和明度
- 渐变效果或色彩过渡
- 品牌色彩的使用情况

标签与图形分析:
- 现有标签的位置和大小
- 文字信息的排版方式
- 图形元素和装饰细节
- 品牌标识的展现形式
```

#### 2.3 设计元素分类
```
必须保留元素:
- 产品的基本形状和比例
- 功能性结构 (泵头、螺纹等)
- 品牌核心识别元素
- 法规要求的信息

可以优化元素:
- 色彩方案和搭配
- 标签设计和排版
- 材质质感表现
- 装饰性图形元素

需要新增元素:
- 新的品牌信息
- 产品特性标识
- 目标受众相关元素
- 季节性或主题性装饰
```

### 阶段3: 设计策略制定

#### 3.1 设计方向确定
```
基于分析结果制定策略:
1. 色彩策略
   - 主色调选择 (基于产品特性)
   - 辅助色彩搭配
   - 色彩心理学应用
   - 目标受众色彩偏好

2. 材质策略
   - 材质质感表现
   - 表面处理效果
   - 光影效果设计
   - 高级感营造

3. 标签策略
   - 信息层级设计
   - 字体选择和排版
   - 图形元素整合
   - 品牌一致性维护

4. 环境策略
   - 产品展示环境
   - 光线和氛围设置
   - 背景和道具选择
   - 整体构图安排
```

#### 3.2 风格定位
```
整体风格确定:
- 现代简约: 简洁线条、几何形状、中性色彩
- 复古经典: 传统元素、怀旧色调、手工质感
- 奢华高端: 精致材质、金属装饰、深邃色彩
- 自然环保: 有机形状、绿色色调、天然材质
- 可爱卡通: 圆润造型、明亮色彩、趣味元素
- 工业设计: 功能导向、金属质感、理性美学
```

### 阶段4: 提示词构建与优化

#### 4.1 提示词结构
```
标准提示词模板:
1. 保留指令部分
   "Keep the exact same [具体特征] without any changes..."

2. 修改指令部分
   "Change the [具体元素] to [目标效果]..."

3. 新增指令部分
   "Add [新元素] with [具体要求]..."

4. 环境设置部分
   "Place the bottle [环境描述] with [氛围要求]..."

5. 风格定义部分
   "The overall style should be [风格描述]..."
```

#### 4.2 提示词优化原则
```
详细性原则:
- 使用具体的描述词汇
- 避免模糊或歧义的表达
- 包含足够的细节信息
- 确保指令的可执行性

准确性原则:
- 基于图像分析的准确描述
- 符合设计理论和实践
- 考虑技术实现的可能性
- 避免矛盾或冲突的指令

完整性原则:
- 覆盖所有重要的设计要素
- 包含必要的约束条件
- 考虑整体效果的协调性
- 确保品牌一致性要求
```

### 阶段5: 工具调用与参数设置

#### 5.1 工具选择逻辑
```
主要工具: Flux Kontext Max
适用场景:
- 基于现有图像的设计改进
- 产品包装的重新设计
- 品牌形象的视觉升级
- 材质和色彩的优化调整

辅助工具: Image Analyzer
使用时机:
- 设计前的图像分析
- 结果验证和比较
- 特征提取和记录
```

#### 5.2 参数配置标准
```
Flux Kontext Max参数设置:
- english_prompt: 详细的英文提示词
- image_name: 使用中文命名 (与用户语言一致)
- image_url_list: 原始图像URL列表
- aspect_ratio: 通常使用9:16 (产品展示比例)
- rescale: 通常设为false (保持原始比例)
- 其他参数根据具体需求调整
```

### 阶段6: 结果检查与质量控制

#### 6.1 视觉质量检查
```
形态保真度检查:
□ 瓶子形状是否与原图一致
□ 比例关系是否正确
□ 结构细节是否完整
□ 功能性元素是否保留

设计效果检查:
□ 色彩方案是否符合需求
□ 材质质感是否真实
□ 标签设计是否清晰
□ 整体风格是否统一

品牌一致性检查:
□ 品牌标识是否清晰可见
□ 品牌色彩是否正确使用
□ 品牌调性是否符合定位
□ 信息传达是否准确
```

#### 6.2 需求符合度检查
```
功能需求检查:
□ 是否满足所有明确要求
□ 是否正确保留指定元素
□ 是否有效修改目标元素
□ 是否成功添加新元素

目标受众适配检查:
□ 是否符合目标受众偏好
□ 是否传达正确的产品信息
□ 是否营造合适的使用场景
□ 是否体现产品的核心价值
```

### 阶段7: 优化迭代与完善

#### 7.1 问题识别
```
常见问题类型:
- 标识不清晰或位置不当
- 色彩不符合需求或不协调
- 材质质感表现不真实
- 整体风格不统一
- 品牌元素缺失或错误
```

#### 7.2 优化策略
```
针对性优化:
1. 标识问题: "Make sure the ENTIRE logo is clearly visible"
2. 色彩问题: "Adjust the color scheme to be more [具体要求]"
3. 材质问题: "Enhance the [材质] texture to look more realistic"
4. 风格问题: "Ensure the overall style is consistently [风格]"
```

#### 7.3 迭代控制
```
迭代原则:
- 每次只调整1-2个主要问题
- 保持其他成功元素不变
- 记录每次修改的具体内容
- 限制迭代次数 (通常不超过3次)
```

### 阶段8: 完成与交付

#### 8.1 最终检查
```
交付前检查清单:
□ 所有需求是否完全满足
□ 设计质量是否达到专业标准
□ 品牌一致性是否得到保证
□ 技术实现是否符合要求
□ 用户反馈是否得到充分考虑
```

#### 8.2 改进建议
```
提供2-3个具体改进建议:
- 基于设计原则的优化方向
- 考虑市场趋势的升级建议
- 针对不同应用场景的变体方案
- 长期品牌发展的设计策略
```

#### 8.3 任务结束
```
使用terminate工具结束任务:
- 总结设计过程和关键决策
- 强调设计的核心价值和特色
- 提供使用和应用的建议
- 为后续合作奠定基础
```

## 质量保证机制

### 设计标准
- 专业性: 符合行业设计标准和最佳实践
- 一致性: 保持品牌形象和设计风格的统一
- 创新性: 在保持专业性的基础上追求创新
- 实用性: 考虑实际生产和使用的可行性

### 流程控制
- 每个阶段都有明确的输入和输出标准
- 关键节点设置质量检查点
- 建立问题反馈和解决机制
- 持续优化和改进流程

### 结果评估
- 客观指标: 技术实现度、需求符合度
- 主观指标: 美学效果、品牌一致性
- 用户反馈: 满意度、使用体验
- 市场表现: 接受度、竞争力
