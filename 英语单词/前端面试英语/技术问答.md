# 前端技术问答

## JavaScript 基础

### Q1: What is the difference between `let`, `const`, and `var`?

**Sample Answer:**
```
The main differences are scope, hoisting, and reassignment:

- `var` has function scope and is hoisted to the top of the function. It can be redeclared and reassigned.
- `let` has block scope and is not hoisted. It can be reassigned but not redeclared in the same scope.
- `const` also has block scope and is not hoisted. It cannot be reassigned or redeclared, but objects and arrays declared with const can still be mutated.

I prefer using `const` by default, and `let` when I need to reassign the variable. I avoid `var` in modern JavaScript development.
```

**中文对照：**
```
主要区别在于作用域、提升和重新赋值：

- `var`具有函数作用域并被提升到函数顶部。它可以被重新声明和重新赋值。
- `let`具有块作用域且不会被提升。它可以重新赋值但不能在同一作用域中重新声明。
- `const`也具有块作用域且不会被提升。它不能被重新赋值或重新声明，但用const声明的对象和数组仍然可以被修改。

我默认使用`const`，当需要重新赋值变量时使用`let`。在现代JavaScript开发中我避免使用`var`。
```

### Q2: Explain event delegation in JavaScript.

**Sample Answer:**
```
Event delegation is a technique where you attach a single event listener to a parent element instead of attaching individual listeners to multiple child elements. This works because of event bubbling - when an event occurs on a child element, it bubbles up through its parent elements.

For example, instead of adding click listeners to each button in a list, you can add one listener to the parent container and use event.target to determine which button was clicked.

The benefits include:
- Better performance with fewer event listeners
- Automatic handling of dynamically added elements
- Reduced memory usage
- Easier maintenance
```

**中文对照：**
```
事件委托是一种技术，你将单个事件监听器附加到父元素上，而不是为多个子元素分别附加监听器。这之所以有效是因为事件冒泡——当子元素上发生事件时，它会向上冒泡到其父元素。

例如，不是为列表中的每个按钮添加点击监听器，你可以为父容器添加一个监听器，并使用event.target来确定点击了哪个按钮。

好处包括：
- 更少的事件监听器带来更好的性能
- 自动处理动态添加的元素
- 减少内存使用
- 更容易维护
```

## React 相关

### Q3: What are React Hooks and why were they introduced?

**Sample Answer:**
```
React Hooks are functions that allow you to use state and other React features in functional components. They were introduced in React 16.8 to solve several problems:

1. **Reusing stateful logic**: Before hooks, sharing stateful logic between components required complex patterns like higher-order components or render props.

2. **Complex components**: Class components often became hard to understand with lifecycle methods scattered throughout.

3. **Classes confusion**: Classes can be confusing for both humans and machines, with issues like `this` binding.

Common hooks include:
- `useState` for managing local state
- `useEffect` for side effects and lifecycle events
- `useContext` for consuming context
- `useMemo` and `useCallback` for performance optimization

Hooks make components more predictable and easier to test.
```

**中文对照：**
```
React Hooks是允许你在函数组件中使用状态和其他React特性的函数。它们在React 16.8中被引入来解决几个问题：

1. **重用有状态逻辑**：在hooks之前，在组件间共享有状态逻辑需要复杂的模式，如高阶组件或render props。

2. **复杂组件**：类组件经常因为生命周期方法分散在各处而变得难以理解。

3. **类的困惑**：类对人和机器都可能造成困惑，包括`this`绑定等问题。

常见的hooks包括：
- `useState`用于管理本地状态
- `useEffect`用于副作用和生命周期事件
- `useContext`用于消费上下文
- `useMemo`和`useCallback`用于性能优化

Hooks使组件更可预测且更容易测试。
```

### Q4: Explain the Virtual DOM and its benefits.

**Sample Answer:**
```
The Virtual DOM is a JavaScript representation of the actual DOM (Document Object Model). It's a programming concept where a "virtual" representation of the UI is kept in memory and synced with the "real" DOM through a process called reconciliation.

Here's how it works:
1. When state changes, React creates a new virtual DOM tree
2. It compares (diffs) this new tree with the previous virtual DOM tree
3. It calculates the minimum changes needed
4. It updates only the changed parts in the real DOM

Benefits include:
- **Performance**: Batch updates and minimal DOM manipulation
- **Predictability**: Easier to understand how UI changes
- **Cross-browser compatibility**: Abstracts away browser differences
- **Developer experience**: Enables features like time-travel debugging
```

**中文对照：**
```
虚拟DOM是实际DOM（文档对象模型）的JavaScript表示。这是一个编程概念，UI的"虚拟"表示保存在内存中，并通过称为协调的过程与"真实"DOM同步。

工作原理：
1. 当状态改变时，React创建一个新的虚拟DOM树
2. 它将这个新树与之前的虚拟DOM树进行比较（diff）
3. 它计算所需的最小更改
4. 它只更新真实DOM中已更改的部分

好处包括：
- **性能**：批量更新和最小化DOM操作
- **可预测性**：更容易理解UI如何变化
- **跨浏览器兼容性**：抽象了浏览器差异
- **开发体验**：启用时间旅行调试等功能
```

## CSS 相关

### Q5: What is the difference between Flexbox and CSS Grid?

**Sample Answer:**
```
Flexbox and CSS Grid are both layout systems, but they serve different purposes:

**Flexbox (1-dimensional):**
- Designed for laying out items in a single direction (row or column)
- Great for component-level layout
- Excellent for centering and distributing space
- Items can grow and shrink based on available space
- Best for: navigation bars, card layouts, centering content

**CSS Grid (2-dimensional):**
- Designed for laying out items in both rows and columns simultaneously
- Great for page-level layout
- More control over item placement
- Can create complex layouts with overlapping elements
- Best for: page layouts, complex forms, magazine-style layouts

In practice, I often use them together - Grid for the overall page structure and Flexbox for component internals.
```

**中文对照：**
```
Flexbox和CSS Grid都是布局系统，但它们服务于不同的目的：

**Flexbox（一维）：**
- 设计用于在单一方向（行或列）布局项目
- 适合组件级布局
- 非常适合居中和分配空间
- 项目可以根据可用空间增长和收缩
- 最适合：导航栏、卡片布局、内容居中

**CSS Grid（二维）：**
- 设计用于同时在行和列中布局项目
- 适合页面级布局
- 对项目放置有更多控制
- 可以创建具有重叠元素的复杂布局
- 最适合：页面布局、复杂表单、杂志风格布局

在实践中，我经常将它们结合使用——Grid用于整体页面结构，Flexbox用于组件内部。
```

## 性能优化

### Q6: How do you optimize the performance of a web application?

**Sample Answer:**
```
There are several strategies I use for performance optimization:

**Frontend Optimization:**
- **Code splitting**: Break code into smaller chunks and load them on demand
- **Lazy loading**: Load images and components only when needed
- **Minification and compression**: Reduce file sizes using tools like Webpack
- **Caching strategies**: Implement proper browser caching and CDN usage
- **Image optimization**: Use appropriate formats (WebP, AVIF) and responsive images

**React-specific:**
- **Memoization**: Use React.memo, useMemo, and useCallback to prevent unnecessary re-renders
- **Virtual scrolling**: For large lists, render only visible items
- **Bundle analysis**: Use tools like webpack-bundle-analyzer to identify large dependencies

**General practices:**
- **Minimize HTTP requests**: Combine files where appropriate
- **Use a CDN**: Serve static assets from geographically distributed servers
- **Database optimization**: Efficient queries and proper indexing
- **Monitoring**: Use tools like Lighthouse and Web Vitals to measure performance
```

**中文对照：**
```
我使用几种策略进行性能优化：

**前端优化：**
- **代码分割**：将代码分解为更小的块并按需加载
- **懒加载**：仅在需要时加载图像和组件
- **压缩和最小化**：使用Webpack等工具减少文件大小
- **缓存策略**：实施适当的浏览器缓存和CDN使用
- **图像优化**：使用适当的格式（WebP、AVIF）和响应式图像

**React特定：**
- **记忆化**：使用React.memo、useMemo和useCallback防止不必要的重新渲染
- **虚拟滚动**：对于大列表，只渲染可见项目
- **包分析**：使用webpack-bundle-analyzer等工具识别大型依赖项

**一般实践：**
- **最小化HTTP请求**：在适当的地方合并文件
- **使用CDN**：从地理分布的服务器提供静态资产
- **数据库优化**：高效查询和适当索引
- **监控**：使用Lighthouse和Web Vitals等工具测量性能
```
