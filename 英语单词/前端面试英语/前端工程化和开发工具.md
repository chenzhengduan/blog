## 1. 构建工具和打包器

### Webpack vs Vite 对比讨论

**English:**
```
Interviewer: "Can you compare Webpack and Vite? When would you choose one over the other?"

Candidate: "Both are excellent build tools, but they serve different purposes and have distinct advantages:

**Webpack**:
- **Mature ecosystem**: Extensive plugin system with thousands of community plugins
- **Configuration flexibility**: Highly customizable for complex build requirements
- **Production optimization**: Excellent code splitting, tree shaking, and bundle optimization
- **Universal compatibility**: Works well with all frameworks and legacy projects
- **Hot Module Replacement**: Robust HMR implementation for development

**Vite**:
- **Lightning-fast development**: Uses native ES modules for instant server start
- **Modern approach**: Built for modern browsers with ES6+ support
- **Simplified configuration**: Convention over configuration philosophy
- **Framework agnostic**: Works seamlessly with Vue, React, Svelte, and vanilla JS
- **Optimized bundling**: Uses Rollup for production builds

**When to choose Webpack**:
- Complex enterprise applications with specific build requirements
- Legacy browser support is critical
- Need extensive customization and plugin ecosystem
- Working with micro-frontends or module federation

**When to choose Vite**:
- Modern applications targeting evergreen browsers
- Rapid prototyping and development speed is priority
- Smaller to medium-sized projects
- Teams preferring minimal configuration

**Personal experience**: I've used both extensively. For a recent React project, I chose Vite because the development experience was significantly faster - cold starts went from 30 seconds with Webpack to under 2 seconds with Vite. However, for a large-scale enterprise application with complex micro-frontend architecture, Webpack's module federation capabilities made it the better choice."
```

```
面试官："你能比较一下Webpack和Vite吗？什么时候你会选择其中一个？"

**Webpack**：
- **成熟的生态系统**：拥有数千个社区插件的广泛插件系统
- **配置灵活性**：对复杂构建需求高度可定制
- **生产优化**：出色的代码分割、tree shaking和包优化
- **通用兼容性**：与所有框架和遗留项目配合良好
- **热模块替换**：强大的HMR开发实现

**Vite**：
- **闪电般快速的开发**：使用原生ES模块实现即时服务器启动
- **现代方法**：为支持ES6+的现代浏览器而构建
- **简化配置**：约定优于配置的理念

**何时选择Webpack**：
- 具有特定构建需求的复杂企业应用
- 遗留浏览器支持至关重要
- 需要广泛的定制和插件生态系统
- 使用微前端或模块联邦

**何时选择Vite**：
- 针对现代浏览器的现代应用
- 快速原型和开发速度是优先考虑
- 中小型项目
- 偏好最小配置的团队

**个人经验**：我广泛使用过两者。对于最近的React项目，我选择了Vite，因为开发体验明显更快 - 冷启动从Webpack的30秒减少到Vite的不到2秒。然而，对于具有复杂微前端架构的大型企业应用，Webpack的模块联邦功能使其成为更好的选择。"
```

### 构建优化策略

**English:**
```
Interviewer: "How do you optimize build performance and bundle size in a large frontend application?"

Candidate: "I use a multi-layered approach to optimize both build performance and bundle size:

**Build Performance Optimization**:

```javascript
// webpack.config.js
module.exports = {
  optimization: {
    minimizer: [
      new TerserPlugin({
        parallel: true, // Use multiple processes
        terserOptions: {
          compress: {
            drop_console: true,
            drop_debugger: true
          }
        }
      })
    ]
  }
};
```

2. **Incremental Builds**:
```javascript
// Enable persistent caching
module.exports = {
  cache: {
    type: 'filesystem',
    buildDependencies: {
      config: [__filename]
    }
  }
};
```

3. **Selective Compilation**:
```javascript
// Only rebuild changed modules
module.exports = {
  watchOptions: {
    ignored: /node_modules/,
    aggregateTimeout: 300
  }
};
```

**Bundle Size Optimization**:

1. **Code Splitting**:
```javascript
// Dynamic imports for route-based splitting
const HomePage = lazy(() => import('./pages/HomePage'));
const ProfilePage = lazy(() => import('./pages/ProfilePage'));

// Component-level splitting
const HeavyComponent = lazy(() => 
  import('./components/HeavyComponent')
);
```

2. **Tree Shaking**:
```javascript
// webpack.config.js
module.exports = {
  mode: 'production',
  optimization: {
    usedExports: true,
    sideEffects: false
  }
};

// package.json
{
  "sideEffects": [
    "*.css",
    "*.scss",
    "./src/polyfills.js"
  ]
}
```

3. **Bundle Analysis**:
```javascript
// Using webpack-bundle-analyzer
const BundleAnalyzerPlugin = require('webpack-bundle-analyzer').BundleAnalyzerPlugin;

module.exports = {
  plugins: [
    new BundleAnalyzerPlugin({
      analyzerMode: 'static',
      openAnalyzer: false,
      reportFilename: 'bundle-report.html'
    })
  ]
};
```

**Results achieved**:
- Reduced build time from 8 minutes to 2 minutes
- Decreased initial bundle size by 40%
- Improved First Contentful Paint by 1.2 seconds
- Achieved 95+ Lighthouse performance score

**Monitoring and maintenance**:
- Set up bundle size budgets in CI/CD pipeline
- Regular dependency audits to remove unused packages
- Performance monitoring with Core Web Vitals tracking"
```

import { render, screen, waitFor } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import { UserProfile } from './UserProfile';
import { api } from '../services/api';

// Mock API calls
jest.mock('../services/api');

describe('UserProfile', () => {
  beforeEach(() => {
    api.getUser.mockResolvedValue({
      id: 1,
      name: 'John Doe',
      email: 'john@example.com'
    });
  });
  
  it('should display user information after loading', async () => {
    render(<UserProfile userId={1} />);
    
    // Check loading state
    expect(screen.getByText('Loading...')).toBeInTheDocument();
    
    // Wait for data to load
    await waitFor(() => {
      expect(screen.getByText('John Doe')).toBeInTheDocument();
      expect(screen.getByText('john@example.com')).toBeInTheDocument();
    });
  });
  
  it('should handle edit mode correctly', async () => {
    const user = userEvent.setup();
    render(<UserProfile userId={1} />);
    
    await waitFor(() => {
      expect(screen.getByText('John Doe')).toBeInTheDocument();
    });
    
    // Click edit button
    await user.click(screen.getByRole('button', { name: 'Edit' }));
    
    // Check if form is displayed
    expect(screen.getByDisplayValue('John Doe')).toBeInTheDocument();
    expect(screen.getByDisplayValue('john@example.com')).toBeInTheDocument();
  });
});
```

```yaml
# .github/workflows/ci-cd.yml
name: CI/CD Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

env:
  NODE_VERSION: '18'
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  # Quality checks
  quality:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4
        
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'
          
      - name: Install dependencies
        run: npm ci
        
      - name: Run linting
        run: npm run lint
        
      - name: Run type checking
        run: npm run type-check
        
      - name: Run tests
        run: npm run test:coverage
        
      - name: Upload coverage reports
        uses: codecov/codecov-action@v3
        with:
          file: ./coverage/lcov.info
          
  # Security scanning
  security:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4
        
      - name: Run security audit
        run: npm audit --audit-level=high
        
      - name: Run Snyk security scan
        uses: snyk/actions/node@master
        env:
          SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
          
  # Build and test
  build:
    needs: [quality, security]
    runs-on: ubuntu-latest
    strategy:
      matrix:
        environment: [staging, production]
    steps:
      - name: Checkout code
        uses: actions/checkout@v4
        
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'
          
      - name: Install dependencies
        run: npm ci
        
      - name: Build application
        run: npm run build:${{ matrix.environment }}
        env:
          REACT_APP_API_URL: ${{ secrets.API_URL }}
          REACT_APP_ENV: ${{ matrix.environment }}
          
      - name: Run E2E tests
        run: npm run test:e2e
        
      - name: Upload build artifacts
        uses: actions/upload-artifact@v3
        with:
          name: build-${{ matrix.environment }}
          path: build/
          
  # Deploy to staging
  deploy-staging:
    needs: build
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/develop'
    environment: staging
    steps:
      - name: Download build artifacts
        uses: actions/download-artifact@v3
        with:
          name: build-staging
          
      - name: Deploy to Netlify
        uses: nwtgck/actions-netlify@v2
        with:
          publish-dir: './'
          production-branch: main
          github-token: ${{ secrets.GITHUB_TOKEN }}
          deploy-message: "Deploy from GitHub Actions"
        env:
          NETLIFY_AUTH_TOKEN: ${{ secrets.NETLIFY_AUTH_TOKEN }}

// Shared event bus
class EventBus {
  constructor() {
    this.events = {};
  }

  subscribe(event, callback) {
    if (!this.events[event]) {
      this.events[event] = [];
    }
    this.events[event].push(callback);
    
    // Return unsubscribe function
    return () => {
      this.events[event] = this.events[event].filter(cb => cb !== callback);
    };
  }

  emit(event, data) {
    if (this.events[event]) {
      this.events[event].forEach(callback => callback(data));
    }
  }
}

// Global instance
window.eventBus = new EventBus();

    window.eventBus.emit('user:updated', userData);

    </form>

// Global error tracking
class ErrorMonitor {
  constructor() {
    this.setupGlobalHandlers();
  }

  setupGlobalHandlers() {
    // JavaScript errors
    window.addEventListener('error', (event) => {
      this.logError({
        type: 'javascript',
        message: event.message,
        filename: event.filename,
        lineno: event.lineno,
        colno: event.colno,
        stack: event.error?.stack
      });
    });

    // Promise rejections
    window.addEventListener('unhandledrejection', (event) => {
      this.logError({
        type: 'promise-rejection',
        message: event.reason?.message || 'Unhandled promise rejection',
        stack: event.reason?.stack
      });
    });

    // Resource loading errors
    window.addEventListener('error', (event) => {
      if (event.target !== window) {
        this.logError({
          type: 'resource',
          message: `Failed to load ${event.target.tagName}`,
          source: event.target.src || event.target.href
        });
      }
    }, true);
  }

  logError(errorData) {
    const enrichedError = {
      ...errorData,
      url: window.location.href,
      timestamp: Date.now(),
      userAgent: navigator.userAgent,
      userId: this.getCurrentUserId()
    };

    // Send to error tracking service
    fetch('/api/errors', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(enrichedError)
    }).catch(() => {
      // Fallback: store in localStorage for later retry
      const errors = JSON.parse(localStorage.getItem('pending-errors') || '[]');
      errors.push(enrichedError);
      localStorage.setItem('pending-errors', JSON.stringify(errors));
    });
  }

  getCurrentUserId() {
    // Get user ID from your auth system
    return localStorage.getItem('userId') || 'anonymous';
  }
}
