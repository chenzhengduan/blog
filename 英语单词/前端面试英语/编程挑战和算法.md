## 1. 算法基础讨论

### 时间复杂度和空间复杂度

**English:**
```
Interviewer: "Can you explain the time and space complexity of this sorting algorithm?"

Candidate: "Let me analyze the complexity of this merge sort implementation:

**Time Complexity**: O(n log n) in all cases (best, average, and worst). The algorithm divides the array into halves recursively (log n levels) and merges them back together (n operations per level).

**Space Complexity**: O(n) due to the auxiliary arrays created during the merge process. We need additional space proportional to the input size for temporary storage.

**Comparison with other algorithms**:
- **Quick Sort**: O(n log n) average case, but O(n²) worst case
- **Bubble Sort**: O(n²) time complexity, O(1) space complexity
- **Heap Sort**: O(n log n) time complexity, O(1) space complexity

Merge sort is preferred when stable sorting is required and when worst-case performance guarantees are important, despite the higher space complexity."
```

**中文对照：**
```

候选人："让我分析这个归并排序实现的复杂度：

**时间复杂度**：在所有情况下（最好、平均和最坏）都是O(n log n)。算法递归地将数组分成两半（log n层），然后将它们合并回来（每层n个操作）。

**空间复杂度**：由于合并过程中创建的辅助数组，复杂度为O(n)。我们需要与输入大小成比例的额外空间用于临时存储。

**与其他算法的比较**：
- **快速排序**：平均情况O(n log n)，但最坏情况O(n²)
- **冒泡排序**：时间复杂度O(n²)，空间复杂度O(1)
- **堆排序**：时间复杂度O(n log n)，空间复杂度O(1)

当需要稳定排序且最坏情况性能保证很重要时，尽管空间复杂度较高，归并排序是首选。"
```

### 数据结构选择

**English:**
```
Interviewer: "How would you choose the appropriate data structure for a frontend autocomplete feature?"

Candidate: "For an autocomplete feature, I'd consider several data structures based on requirements:

**Trie (Prefix Tree)**: Most suitable for autocomplete because:
- **Efficient prefix matching**: O(m) time complexity where m is the prefix length
- **Memory efficient**: Shared prefixes reduce storage overhead
- **Easy traversal**: Can easily find all words with a given prefix

**Hash Table**: Good for exact matches but less efficient for prefix searches:
- **Fast lookup**: O(1) for exact matches
- **Poor prefix support**: Would need to iterate through all keys

**Sorted Array with Binary Search**: Reasonable alternative:
- **Space efficient**: No additional pointer overhead
- **Prefix search**: Can use binary search to find range, then linear scan
- **Time complexity**: O(log n + k) where k is the number of results

**Implementation considerations**:
- **Dataset size**: Trie for large datasets, array for smaller ones
- **Update frequency**: Trie handles insertions/deletions better
- **Memory constraints**: Array uses less memory overhead
- **Search patterns**: Trie excels at prefix-based searches

For most frontend autocomplete scenarios, I'd implement a Trie with lazy loading for large datasets."
```

## 2. JavaScript编程挑战

### 异步编程问题

**English:**
```
Interviewer: "Can you implement a function that fetches data from multiple APIs concurrently and handles errors gracefully?"

Candidate: "I'll implement a robust solution using Promise.allSettled for concurrent requests with error handling:

```javascript
async function fetchMultipleAPIs(urls, options = {}) {
  const { timeout = 5000, retries = 2 } = options;
  
  // Create fetch promises with timeout and retry logic
  const fetchPromises = urls.map(url => 
    fetchWithRetry(url, { timeout, retries })
  );
  
  // Execute all requests concurrently
  const results = await Promise.allSettled(fetchPromises);
  
  // Process results and separate successful from failed requests
  const successful = [];
  const failed = [];
  
  results.forEach((result, index) => {
    if (result.status === 'fulfilled') {
      successful.push({ url: urls[index], data: result.value });
    } else {
      failed.push({ url: urls[index], error: result.reason });
    }
  });
  
  return { successful, failed, total: urls.length };
}

async function fetchWithRetry(url, { timeout, retries }) {
  for (let attempt = 0; attempt <= retries; attempt++) {
    try {
      const controller = new AbortController();
      const timeoutId = setTimeout(() => controller.abort(), timeout);
      
      const response = await fetch(url, { 
        signal: controller.signal 
      });
      
      clearTimeout(timeoutId);
      
      if (!response.ok) {
        throw new Error(`HTTP ${response.status}: ${response.statusText}`);
      }
      
      return await response.json();
    } catch (error) {
      if (attempt === retries) throw error;
      await new Promise(resolve => setTimeout(resolve, 1000 * attempt));
    }
  }
}
```

**Key features**:
- **Concurrent execution**: Uses Promise.allSettled to avoid failing fast
- **Timeout handling**: AbortController for request timeouts
- **Retry mechanism**: Exponential backoff for failed requests
- **Error categorization**: Separates successful and failed requests
- **Graceful degradation**: Continues even if some requests fail"
```

const compose = (...functions) => (value) =>

// Using pipe for readable left-to-right flow
const processUsers = pipe(
  filterActive,
  sortByAge,
  extractNames,

);

const result = processUsers(users); // "John Doe, Bob Johnson"

## 3. 前端特定算法问题

### DOM操作优化

**English:**
```
Interviewer: "How would you efficiently update a large list of DOM elements without causing performance issues?"

Candidate: "I'd use several optimization techniques to handle large DOM updates efficiently:

**1. Virtual Scrolling/Windowing**:
```javascript
class VirtualList {
  constructor(container, items, itemHeight) {
    this.container = container;
    this.items = items;
    this.itemHeight = itemHeight;
    this.visibleCount = Math.ceil(container.clientHeight / itemHeight);
    this.startIndex = 0;
    this.endIndex = this.visibleCount;
    
    this.setupScrollListener();
    this.render();
  }
  
  setupScrollListener() {
    this.container.addEventListener('scroll', () => {
      const scrollTop = this.container.scrollTop;
      const newStartIndex = Math.floor(scrollTop / this.itemHeight);
      
      if (newStartIndex !== this.startIndex) {
        this.startIndex = newStartIndex;
        this.endIndex = Math.min(
          this.startIndex + this.visibleCount + 1,
          this.items.length
        );
        this.render();
      }
    });
  }
  
  render() {
    const fragment = document.createDocumentFragment();
    
    // Clear existing content
    this.container.innerHTML = '';
    
    // Create spacer for items above viewport
    const topSpacer = document.createElement('div');
    topSpacer.style.height = `${this.startIndex * this.itemHeight}px`;
    fragment.appendChild(topSpacer);
    
    // Render visible items
    for (let i = this.startIndex; i < this.endIndex; i++) {
      const item = this.createItemElement(this.items[i], i);
      fragment.appendChild(item);
    }
    
    // Create spacer for items below viewport
    const bottomSpacer = document.createElement('div');
    const remainingItems = this.items.length - this.endIndex;
    bottomSpacer.style.height = `${remainingItems * this.itemHeight}px`;
    fragment.appendChild(bottomSpacer);
    
    this.container.appendChild(fragment);
  }
  
  createItemElement(item, index) {
    const element = document.createElement('div');
    element.style.height = `${this.itemHeight}px`;
    element.textContent = `Item ${index}: ${item.name}`;
    return element;
  }
}
```

**2. Batch DOM Updates**:
```javascript
function batchDOMUpdates(updates) {
  // Use requestAnimationFrame for optimal timing
  requestAnimationFrame(() => {
    const fragment = document.createDocumentFragment();
    
    updates.forEach(update => {
      const element = update.createElement();
      fragment.appendChild(element);
    });
    
    // Single DOM insertion
    document.getElementById('container').appendChild(fragment);
  });
}
```

**Key optimization strategies**:
- **Minimize reflows**: Batch DOM modifications to reduce layout recalculations
- **Use DocumentFragment**: Perform operations off-DOM before insertion
- **Debounce scroll events**: Prevent excessive render calls during scrolling
- **Lazy rendering**: Only create DOM elements for visible items
- **CSS containment**: Use `contain` property to limit layout scope"
```

```javascript
class FuzzySearch {
  constructor(items, options = {}) {
    this.items = items;
    this.threshold = options.threshold || 0.6;
    this.caseSensitive = options.caseSensitive || false;
    this.searchFields = options.searchFields || ['name'];
    
    // Pre-process items for faster searching
    this.processedItems = this.preprocessItems();
  }
  
  preprocessItems() {
    return this.items.map((item, index) => ({
      original: item,
      index,
      searchText: this.extractSearchText(item).toLowerCase()
    }));
  }
  
  extractSearchText(item) {
    return this.searchFields
      .map(field => item[field] || '')
      .join(' ');
  }
  
  search(query, limit = 10) {
    if (!query.trim()) return [];
    
    const normalizedQuery = this.caseSensitive ? query : query.toLowerCase();
    const results = [];
    
    for (const item of this.processedItems) {
      const similarity = this.calculateSimilarity(
        normalizedQuery,
        item.searchText
      );
      
      if (similarity >= this.threshold) {
        results.push({
          item: item.original,
          score: similarity,
          matches: this.findMatches(normalizedQuery, item.searchText)
        });
      }
    }
    
    // Sort by similarity score (descending) and return top results
    return results
      .sort((a, b) => b.score - a.score)
      .slice(0, limit);
  }
  
  calculateSimilarity(str1, str2) {
    // Handle exact matches
    if (str1 === str2) return 1.0;
    
    // Handle substring matches
    if (str2.includes(str1)) {
      return 0.9 - (str2.length - str1.length) / str2.length * 0.1;
    }
    
    // Calculate Levenshtein distance
    const distance = this.levenshteinDistance(str1, str2);
    const maxLength = Math.max(str1.length, str2.length);
    
    return maxLength === 0 ? 1.0 : 1.0 - distance / maxLength;
  }
  
  levenshteinDistance(str1, str2) {
    const matrix = Array(str2.length + 1)
      .fill(null)
      .map(() => Array(str1.length + 1).fill(null));
    
    // Initialize first row and column
    for (let i = 0; i <= str1.length; i++) matrix[0][i] = i;
    for (let j = 0; j <= str2.length; j++) matrix[j][0] = j;
    
    // Fill the matrix
    for (let j = 1; j <= str2.length; j++) {
      for (let i = 1; i <= str1.length; i++) {
        const cost = str1[i - 1] === str2[j - 1] ? 0 : 1;
        matrix[j][i] = Math.min(
          matrix[j][i - 1] + 1,     // deletion
          matrix[j - 1][i] + 1,     // insertion
          matrix[j - 1][i - 1] + cost // substitution
        );
      }
    }
    
    return matrix[str2.length][str1.length];
  }
  
  findMatches(query, text) {
    const matches = [];
    const words = query.split(' ').filter(word => word.length > 0);
    
    words.forEach(word => {
      const index = text.indexOf(word);
      if (index !== -1) {
        matches.push({ word, start: index, end: index + word.length });
      }
    });
    
    return matches;
  }
}
