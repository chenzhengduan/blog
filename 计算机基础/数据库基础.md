# 数据库基础

## 1. 数据库概述

### 1.1 什么是数据库
数据库（Database，DB）是按照数据结构来组织、存储和管理数据的仓库。数据库管理系统（Database Management System，DBMS）是一种操纵和管理数据库的大型软件，用于建立、使用和维护数据库。

### 1.2 数据库系统的特点
- **数据结构化**：数据按一定的数据模型组织
- **数据共享性高**：多个用户可以同时访问
- **数据独立性**：数据与程序相互独立
- **数据冗余小**：避免数据重复存储
- **数据一致性**：保证数据的正确性
- **数据安全性**：提供访问控制和权限管理

### 1.3 数据库系统的组成
- **数据库（DB）**：存储数据的仓库
- **数据库管理系统（DBMS）**：管理数据库的软件
- **数据库应用程序**：使用数据库的应用软件
- **数据库管理员（DBA）**：管理数据库的人员
- **用户**：使用数据库的人员

### 1.4 数据模型

#### 层次模型
- **结构**：树形结构
- **特点**：有且仅有一个根节点
- **优点**：结构简单、查询效率高
- **缺点**：缺乏灵活性

#### 网状模型
- **结构**：图形结构
- **特点**：允许多对多关系
- **优点**：能表示复杂关系
- **缺点**：结构复杂

#### 关系模型
- **结构**：二维表格
- **特点**：基于数学理论
- **优点**：结构简单、理论完善
- **应用**：目前最主流的数据模型

#### 面向对象模型
- **结构**：对象和类
- **特点**：支持复杂数据类型
- **应用**：适合复杂应用

## 2. 关系数据库理论

### 2.1 关系模型基本概念

#### 关系（Relation）
- **定义**：一个关系对应一张二维表
- **表示**：R(A₁, A₂, ..., Aₙ)

#### 属性（Attribute）
- **定义**：表中的列
- **域（Domain）**：属性的取值范围

#### 元组（Tuple）
- **定义**：表中的行
- **表示**：一条记录

#### 主键（Primary Key）
- **定义**：唯一标识元组的属性组合
- **特性**：唯一性、非空性

#### 外键（Foreign Key）
- **定义**：引用其他表主键的属性
- **作用**：维护表间关系

### 2.2 关系代数

#### 基本运算
- **选择（σ）**：选择满足条件的元组
- **投影（π）**：选择指定的属性列
- **并（∪）**：两个关系的并集
- **差（-）**：两个关系的差集
- **笛卡尔积（×）**：两个关系的笛卡尔积

#### 扩展运算
- **交（∩）**：两个关系的交集
- **连接（⋈）**：根据条件连接两个关系
- **除法（÷）**：关系除法运算

### 2.3 函数依赖

#### 函数依赖定义
设R(U)是一个关系模式，X和Y是U的子集。若对于R(U)的任意一个可能的关系r，r中不可能存在两个元组在X上的属性值相等，而在Y上的属性值不等，则称X函数确定Y或Y函数依赖于X，记作X→Y。

#### 函数依赖类型
- **完全函数依赖**：Y完全函数依赖于X
- **部分函数依赖**：Y部分函数依赖于X
- **传递函数依赖**：X→Y，Y→Z，则X→Z

### 2.4 范式理论

#### 第一范式（1NF）
- **定义**：关系中每个属性都是不可分的基本数据项
- **要求**：属性值具有原子性

#### 第二范式（2NF）
- **定义**：满足1NF，且非主属性完全函数依赖于主键
- **消除**：部分函数依赖

#### 第三范式（3NF）
- **定义**：满足2NF，且非主属性不传递依赖于主键
- **消除**：传递函数依赖

#### BC范式（BCNF）
- **定义**：满足3NF，且每个决定因素都是候选键
- **更严格**：比3NF更严格的范式

#### 第四范式（4NF）
- **定义**：满足BCNF，且消除多值依赖
- **应用**：处理多值依赖问题

## 3. SQL语言

### 3.1 SQL概述

SQL（Structured Query Language）是结构化查询语言，是关系数据库的标准语言。SQL包括数据定义、数据操纵、数据控制等功能。

#### SQL的组成
- **DDL（Data Definition Language）**：数据定义语言
- **DML（Data Manipulation Language）**：数据操纵语言
- **DCL（Data Control Language）**：数据控制语言
- **DQL（Data Query Language）**：数据查询语言

### 3.2 数据定义语言（DDL）

#### 创建数据库
```sql
CREATE DATABASE database_name;
```

#### 创建表
```sql
CREATE TABLE table_name (
    column1 datatype constraints,
    column2 datatype constraints,
    ...
    PRIMARY KEY (column1),
    FOREIGN KEY (column2) REFERENCES other_table(column)
);
```

#### 修改表结构
```sql
-- 添加列
ALTER TABLE table_name ADD column_name datatype;

-- 修改列
ALTER TABLE table_name MODIFY column_name new_datatype;

-- 删除列
ALTER TABLE table_name DROP COLUMN column_name;
```

#### 删除表
```sql
DROP TABLE table_name;
```

#### 常用数据类型
- **数值类型**：INT, BIGINT, DECIMAL, FLOAT, DOUBLE
- **字符类型**：CHAR, VARCHAR, TEXT
- **日期时间类型**：DATE, TIME, DATETIME, TIMESTAMP
- **布尔类型**：BOOLEAN

#### 约束条件
- **PRIMARY KEY**：主键约束
- **FOREIGN KEY**：外键约束
- **UNIQUE**：唯一约束
- **NOT NULL**：非空约束
- **CHECK**：检查约束
- **DEFAULT**：默认值约束

### 3.3 数据操纵语言（DML）

#### 插入数据
```sql
-- 插入单行
INSERT INTO table_name (column1, column2, ...)
VALUES (value1, value2, ...);

-- 插入多行
INSERT INTO table_name (column1, column2, ...)
VALUES 
    (value1, value2, ...),
    (value3, value4, ...),
    ...;

-- 从其他表插入
INSERT INTO table_name (column1, column2, ...)
SELECT column1, column2, ...
FROM other_table
WHERE condition;
```

#### 更新数据
```sql
UPDATE table_name
SET column1 = value1, column2 = value2, ...
WHERE condition;
```

#### 删除数据
```sql
DELETE FROM table_name
WHERE condition;
```

### 3.4 数据查询语言（DQL）

#### 基本查询
```sql
SELECT column1, column2, ...
FROM table_name
WHERE condition
ORDER BY column1 ASC/DESC;
```

#### 聚合函数
```sql
SELECT 
    COUNT(*) as total_count,
    SUM(column1) as total_sum,
    AVG(column1) as average,
    MAX(column1) as maximum,
    MIN(column1) as minimum
FROM table_name
WHERE condition;
```

#### 分组查询
```sql
SELECT column1, COUNT(*)
FROM table_name
WHERE condition
GROUP BY column1
HAVING COUNT(*) > 1
ORDER BY COUNT(*) DESC;
```

#### 连接查询

##### 内连接
```sql
SELECT t1.column1, t2.column2
FROM table1 t1
INNER JOIN table2 t2 ON t1.id = t2.table1_id;
```

##### 左外连接
```sql
SELECT t1.column1, t2.column2
FROM table1 t1
LEFT JOIN table2 t2 ON t1.id = t2.table1_id;
```

##### 右外连接
```sql
SELECT t1.column1, t2.column2
FROM table1 t1
RIGHT JOIN table2 t2 ON t1.id = t2.table1_id;
```

##### 全外连接
```sql
SELECT t1.column1, t2.column2
FROM table1 t1
FULL OUTER JOIN table2 t2 ON t1.id = t2.table1_id;
```

#### 子查询

##### 标量子查询
```sql
SELECT *
FROM table1
WHERE column1 = (SELECT MAX(column1) FROM table2);
```

##### 列子查询
```sql
SELECT *
FROM table1
WHERE column1 IN (SELECT column1 FROM table2 WHERE condition);
```

##### 行子查询
```sql
SELECT *
FROM table1
WHERE (column1, column2) = (SELECT column1, column2 FROM table2 WHERE condition);
```

##### 表子查询
```sql
SELECT *
FROM (SELECT column1, column2 FROM table1 WHERE condition) AS subquery
WHERE subquery.column1 > 100;
```

#### 集合运算

##### 并集
```sql
SELECT column1 FROM table1
UNION
SELECT column1 FROM table2;
```

##### 交集
```sql
SELECT column1 FROM table1
INTERSECT
SELECT column1 FROM table2;
```

##### 差集
```sql
SELECT column1 FROM table1
EXCEPT
SELECT column1 FROM table2;
```

### 3.5 数据控制语言（DCL）

#### 创建用户
```sql
CREATE USER 'username'@'host' IDENTIFIED BY 'password';
```

#### 授权
```sql
GRANT SELECT, INSERT, UPDATE, DELETE
ON database_name.table_name
TO 'username'@'host';
```

#### 撤销权限
```sql
REVOKE SELECT, INSERT
ON database_name.table_name
FROM 'username'@'host';
```

#### 删除用户
```sql
DROP USER 'username'@'host';
```

## 4. 数据库设计

### 4.1 数据库设计过程

#### 需求分析
- **用户需求**：了解用户的数据需求
- **数据需求**：确定需要存储的数据
- **处理需求**：确定数据的处理要求

#### 概念设计
- **实体-关系模型（E-R模型）**
- **实体（Entity）**：现实世界中的对象
- **属性（Attribute）**：实体的特征
- **关系（Relationship）**：实体间的联系

#### 逻辑设计
- **关系模式设计**：将E-R图转换为关系模式
- **规范化**：消除数据冗余和异常
- **完整性约束**：定义数据完整性规则

#### 物理设计
- **存储结构设计**：选择合适的存储结构
- **索引设计**：创建合适的索引
- **分区设计**：大表的分区策略

### 4.2 E-R模型

#### E-R图符号
- **矩形**：表示实体
- **椭圆**：表示属性
- **菱形**：表示关系
- **直线**：连接实体、属性、关系

#### 实体间关系
- **一对一（1:1）**：一个实体对应另一个实体
- **一对多（1:N）**：一个实体对应多个实体
- **多对多（M:N）**：多个实体对应多个实体

#### E-R图转换为关系模式
1. **实体转换**：每个实体转换为一个关系
2. **属性转换**：实体属性成为关系属性
3. **关系转换**：根据关系类型进行转换

### 4.3 数据库完整性

#### 实体完整性
- **定义**：主键不能为空且唯一
- **实现**：PRIMARY KEY约束

#### 参照完整性
- **定义**：外键必须引用存在的主键值
- **实现**：FOREIGN KEY约束

#### 用户定义完整性
- **定义**：用户根据业务规则定义的约束
- **实现**：CHECK约束、触发器

## 5. 事务处理

### 5.1 事务概念

#### 事务定义
事务（Transaction）是数据库操作的基本单位，是一个不可分割的工作单位。事务中包含的所有操作要么全部成功，要么全部失败。

#### 事务的ACID特性

##### 原子性（Atomicity）
- **定义**：事务是不可分割的最小工作单位
- **保证**：要么全部执行，要么全部不执行

##### 一致性（Consistency）
- **定义**：事务执行前后数据库状态保持一致
- **保证**：不违反数据库的完整性约束

##### 隔离性（Isolation）
- **定义**：并发执行的事务相互隔离
- **保证**：一个事务的执行不受其他事务干扰

##### 持久性（Durability）
- **定义**：已提交事务的修改永久保存
- **保证**：即使系统故障也不会丢失

### 5.2 并发控制

#### 并发问题

##### 丢失修改
- **问题**：两个事务同时修改同一数据，后提交的覆盖先提交的
- **解决**：加锁机制

##### 不可重复读
- **问题**：事务内多次读取同一数据得到不同结果
- **解决**：读锁

##### 脏读
- **问题**：读取了其他事务未提交的数据
- **解决**：写锁

##### 幻读
- **问题**：事务内多次查询得到不同的行数
- **解决**：范围锁

#### 隔离级别

##### 读未提交（Read Uncommitted）
- **特点**：最低隔离级别
- **问题**：可能出现脏读、不可重复读、幻读

##### 读已提交（Read Committed）
- **特点**：只能读取已提交的数据
- **问题**：可能出现不可重复读、幻读

##### 可重复读（Repeatable Read）
- **特点**：保证同一事务内多次读取结果一致
- **问题**：可能出现幻读

##### 串行化（Serializable）
- **特点**：最高隔离级别
- **保证**：完全隔离，无并发问题
- **缺点**：性能最低

#### 锁机制

##### 锁的类型
- **共享锁（S锁）**：读锁，多个事务可同时持有
- **排他锁（X锁）**：写锁，只有一个事务可持有

##### 锁的粒度
- **行级锁**：锁定单行数据
- **页级锁**：锁定数据页
- **表级锁**：锁定整张表

##### 死锁
- **定义**：两个或多个事务相互等待对方释放锁
- **检测**：死锁检测算法
- **解决**：回滚其中一个事务

### 5.3 恢复技术

#### 故障类型
- **事务故障**：事务执行过程中的异常
- **系统故障**：系统崩溃、断电等
- **介质故障**：磁盘损坏等硬件故障

#### 恢复技术

##### 日志技术
- **前向恢复（Redo）**：重做已提交事务的操作
- **后向恢复（Undo）**：撤销未提交事务的操作

##### 检查点技术
- **目的**：减少恢复时需要处理的日志量
- **原理**：定期将内存中的数据写入磁盘

##### 备份与恢复
- **完全备份**：备份整个数据库
- **增量备份**：备份自上次备份后的变化
- **差异备份**：备份自上次完全备份后的变化

## 6. 索引与查询优化

### 6.1 索引

#### 索引概念
索引是数据库中用于快速查找数据的数据结构。索引提供了数据的快速访问路径，类似于书籍的目录。

#### 索引类型

##### 按数据结构分类
- **B树索引**：最常用的索引类型
- **B+树索引**：B树的变种，叶子节点包含所有数据
- **哈希索引**：基于哈希表的索引
- **位图索引**：适用于低基数列

##### 按物理存储分类
- **聚集索引**：索引顺序与数据存储顺序一致
- **非聚集索引**：索引顺序与数据存储顺序无关

##### 按逻辑分类
- **主索引**：基于主键的索引
- **辅助索引**：基于非主键列的索引
- **唯一索引**：保证列值唯一性的索引
- **复合索引**：基于多个列的索引

#### 索引操作

##### 创建索引
```sql
-- 创建普通索引
CREATE INDEX index_name ON table_name (column1);

-- 创建唯一索引
CREATE UNIQUE INDEX index_name ON table_name (column1);

-- 创建复合索引
CREATE INDEX index_name ON table_name (column1, column2);
```

##### 删除索引
```sql
DROP INDEX index_name ON table_name;
```

#### 索引的优缺点

##### 优点
- **提高查询速度**：快速定位数据
- **加速连接操作**：提高JOIN性能
- **加速排序和分组**：ORDER BY和GROUP BY更快

##### 缺点
- **占用存储空间**：需要额外的存储空间
- **降低写操作性能**：INSERT、UPDATE、DELETE需要维护索引
- **维护成本**：需要定期维护索引

### 6.2 查询优化

#### 查询优化器
查询优化器是数据库管理系统的核心组件，负责为SQL查询选择最优的执行计划。

#### 优化策略

##### 基于规则的优化（RBO）
- **原理**：根据预定义的规则进行优化
- **特点**：确定性，但可能不是最优

##### 基于成本的优化（CBO）
- **原理**：根据统计信息估算成本，选择成本最低的执行计划
- **特点**：更智能，但依赖统计信息的准确性

#### 查询优化技术

##### 选择操作优化
- **选择条件下推**：尽早执行选择操作
- **索引选择**：选择合适的索引

##### 连接操作优化
- **连接顺序**：选择最优的连接顺序
- **连接算法**：嵌套循环、排序合并、哈希连接

##### 子查询优化
- **子查询展开**：将子查询转换为连接
- **半连接**：优化EXISTS子查询

#### 执行计划
执行计划是查询优化器为SQL语句选择的具体执行方案。

##### 查看执行计划
```sql
-- MySQL
EXPLAIN SELECT * FROM table_name WHERE condition;

-- PostgreSQL
EXPLAIN ANALYZE SELECT * FROM table_name WHERE condition;

-- SQL Server
SET SHOWPLAN_ALL ON;
SELECT * FROM table_name WHERE condition;
```

## 7. 数据库管理

### 7.1 数据库安全

#### 安全威胁
- **非授权访问**：未经授权的用户访问数据库
- **数据泄露**：敏感数据被窃取
- **数据篡改**：数据被恶意修改
- **拒绝服务**：系统无法正常提供服务

#### 安全措施

##### 身份认证
- **用户名密码**：基本的身份认证方式
- **数字证书**：基于PKI的认证
- **生物特征**：指纹、虹膜等

##### 访问控制
- **自主访问控制（DAC）**：用户可以控制自己资源的访问权限
- **强制访问控制（MAC）**：系统强制执行访问控制策略
- **基于角色的访问控制（RBAC）**：基于用户角色分配权限

##### 数据加密
- **传输加密**：SSL/TLS加密网络传输
- **存储加密**：加密存储在磁盘上的数据
- **列级加密**：对敏感列进行加密

##### 审计
- **登录审计**：记录用户登录信息
- **操作审计**：记录用户的数据库操作
- **权限审计**：定期检查用户权限

### 7.2 备份与恢复

#### 备份策略

##### 完全备份
- **定义**：备份整个数据库
- **优点**：恢复简单
- **缺点**：备份时间长，占用空间大

##### 增量备份
- **定义**：备份自上次备份后的变化
- **优点**：备份时间短，占用空间小
- **缺点**：恢复复杂

##### 差异备份
- **定义**：备份自上次完全备份后的变化
- **特点**：介于完全备份和增量备份之间

#### 备份方法

##### 物理备份
- **定义**：直接复制数据库文件
- **工具**：文件系统备份、专用备份工具

##### 逻辑备份
- **定义**：导出数据库的逻辑结构和数据
- **工具**：mysqldump、pg_dump等

#### 恢复策略

##### 完全恢复
- **场景**：系统完全崩溃
- **方法**：从最近的完全备份恢复

##### 不完全恢复
- **场景**：恢复到某个特定时间点
- **方法**：结合备份和事务日志

### 7.3 性能监控与调优

#### 性能指标

##### 响应时间
- **定义**：从请求发出到收到响应的时间
- **影响因素**：查询复杂度、索引、硬件性能

##### 吞吐量
- **定义**：单位时间内处理的事务数量
- **衡量标准**：TPS（每秒事务数）、QPS（每秒查询数）

##### 资源利用率
- **CPU利用率**：处理器使用情况
- **内存利用率**：内存使用情况
- **磁盘I/O**：磁盘读写情况
- **网络I/O**：网络传输情况

#### 性能调优

##### 硬件调优
- **CPU**：增加CPU核心数或提高主频
- **内存**：增加内存容量
- **存储**：使用SSD、RAID等
- **网络**：提高网络带宽

##### 数据库配置调优
- **缓冲池大小**：调整数据库缓冲池
- **连接数**：设置合适的最大连接数
- **日志配置**：优化事务日志设置

##### SQL调优
- **索引优化**：创建合适的索引
- **查询重写**：优化SQL语句
- **统计信息更新**：保持统计信息的准确性

## 8. NoSQL数据库

### 8.1 NoSQL概述

NoSQL（Not Only SQL）是对不同于传统关系数据库的数据库管理系统的统称。NoSQL数据库通常具有高可扩展性、高性能、灵活的数据模型等特点。

#### NoSQL的特点
- **灵活的数据模型**：支持非结构化和半结构化数据
- **水平扩展**：易于横向扩展
- **高性能**：针对特定场景优化
- **高可用性**：分布式架构，容错能力强

### 8.2 NoSQL数据库类型

#### 键值存储（Key-Value Store）
- **特点**：简单的键值对存储
- **代表**：Redis、Amazon DynamoDB
- **应用场景**：缓存、会话存储

#### 文档数据库（Document Database）
- **特点**：存储JSON、XML等文档
- **代表**：MongoDB、CouchDB
- **应用场景**：内容管理、目录服务

#### 列族数据库（Column Family）
- **特点**：按列存储数据
- **代表**：Cassandra、HBase
- **应用场景**：大数据分析、时间序列数据

#### 图数据库（Graph Database）
- **特点**：存储图结构数据
- **代表**：Neo4j、Amazon Neptune
- **应用场景**：社交网络、推荐系统

### 8.3 CAP定理

CAP定理指出，在分布式系统中，一致性（Consistency）、可用性（Availability）、分区容错性（Partition Tolerance）三者不能同时满足，最多只能满足其中两个。

#### 一致性（Consistency）
- **定义**：所有节点在同一时间看到相同的数据
- **强一致性**：立即一致
- **弱一致性**：最终一致

#### 可用性（Availability）
- **定义**：系统在任何时候都能响应用户请求
- **要求**：系统持续可用

#### 分区容错性（Partition Tolerance）
- **定义**：系统在网络分区时仍能继续工作
- **必要性**：分布式系统必须考虑

## 9. 数据仓库与大数据

### 9.1 数据仓库

#### 数据仓库概念
数据仓库是一个面向主题的、集成的、相对稳定的、反映历史变化的数据集合，用于支持管理决策。

#### 数据仓库特点
- **面向主题**：按业务主题组织数据
- **集成性**：来自多个数据源的集成
- **非易失性**：数据相对稳定
- **时变性**：反映历史变化

#### 数据仓库架构
- **数据源层**：各种业务系统
- **数据集成层**：ETL过程
- **数据存储层**：数据仓库核心
- **数据访问层**：OLAP、报表工具

### 9.2 ETL过程

#### 提取（Extract）
- **功能**：从源系统提取数据
- **方式**：全量提取、增量提取

#### 转换（Transform）
- **功能**：数据清洗、格式转换、业务规则应用
- **操作**：过滤、聚合、计算、验证

#### 加载（Load）
- **功能**：将处理后的数据加载到目标系统
- **方式**：全量加载、增量加载

### 9.3 OLAP

#### OLAP概念
OLAP（Online Analytical Processing）是联机分析处理，支持复杂的分析操作，侧重决策支持。

#### OLAP vs OLTP
- **OLTP**：联机事务处理，支持日常业务操作
- **OLAP**：联机分析处理，支持决策分析

#### OLAP操作
- **钻取（Drill Down）**：从汇总数据到详细数据
- **上卷（Roll Up）**：从详细数据到汇总数据
- **切片（Slice）**：选择维度的特定值
- **切块（Dice）**：选择多个维度的值范围
