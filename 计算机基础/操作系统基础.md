# 操作系统基础

## 1. 操作系统概述

### 1.1 什么是操作系统

操作系统（Operating System，OS）是管理计算机硬件与软件资源的计算机程序，同时也是计算机系统的内核与基石。操作系统是用户和计算机硬件之间的接口，它隐藏了硬件的复杂性，为用户和应用程序提供了一个简洁、统一的计算环境。

#### 操作系统的定义层次

**从用户角度**：操作系统是一个控制程序，它控制用户程序的执行以防止错误和不当使用计算机。

**从系统角度**：操作系统是一个资源管理器，它管理计算机系统的各种资源，包括处理器、内存、文件和I/O设备。

**从实现角度**：操作系统是一个常驻内存的程序，它为其他程序提供服务。

#### 操作系统的发展历程

1. **第一代（1940s-1950s）**：无操作系统，程序员直接操作硬件
2. **第二代（1950s-1960s）**：批处理系统，作业按批次处理
3. **第三代（1960s-1980s）**：多道程序系统，分时系统
4. **第四代（1980s-现在）**：个人计算机，图形用户界面
5. **第五代（现在-未来）**：移动计算，云计算，物联网

### 1.2 操作系统的主要功能

#### 1.2.1 进程管理
- **进程创建与终止**：管理进程的生命周期
- **进程调度**：决定哪个进程获得CPU时间
- **进程同步**：协调多个进程的执行
- **进程通信**：提供进程间数据交换机制
- **死锁处理**：预防、避免、检测和解除死锁

#### 1.2.2 内存管理
- **内存分配**：为进程分配内存空间
- **内存保护**：防止进程访问非法内存
- **内存共享**：允许多个进程共享内存
- **虚拟内存**：提供比物理内存更大的地址空间
- **内存回收**：回收不再使用的内存

#### 1.2.3 文件系统管理
- **文件操作**：创建、删除、读写、修改文件
- **目录管理**：组织和管理文件目录结构
- **存储空间管理**：管理磁盘空间的分配和回收
- **文件保护**：控制文件的访问权限
- **文件系统维护**：检查和修复文件系统

#### 1.2.4 设备管理
- **设备驱动**：提供设备的抽象接口
- **I/O调度**：优化设备访问顺序
- **缓冲管理**：管理I/O缓冲区
- **设备分配**：管理设备的独占和共享
- **错误处理**：处理设备错误和异常

#### 1.2.5 用户接口
- **命令行界面（CLI）**：基于文本的交互方式
- **图形用户界面（GUI）**：基于图形的交互方式
- **系统调用接口**：为应用程序提供服务接口
- **应用程序接口（API）**：标准化的编程接口

### 1.3 操作系统的类型

#### 1.3.1 按处理方式分类

**批处理系统**
- 特点：作业成批处理，无交互
- 优点：系统吞吐量高，资源利用率高
- 缺点：响应时间长，无法交互
- 应用：大型科学计算，数据处理

**分时系统**
- 特点：多用户同时使用，时间片轮转
- 优点：响应时间短，支持交互
- 缺点：系统开销大，实时性差
- 应用：多用户计算环境

**实时系统**
- 特点：严格的时间约束，快速响应
- 分类：硬实时系统、软实时系统
- 应用：工业控制、航空航天、医疗设备

#### 1.3.2 按用户数量分类

**单用户系统**
- 特点：同时只能一个用户使用
- 例子：早期的MS-DOS

**多用户系统**
- 特点：支持多个用户同时使用
- 例子：Unix、Linux、Windows Server

#### 1.3.3 按处理器数量分类

**单处理器系统**
- 特点：只有一个CPU
- 挑战：进程调度，资源竞争

**多处理器系统**
- **对称多处理（SMP）**：所有处理器地位相等
- **非对称多处理（ASMP）**：有主从关系
- **大规模并行处理（MPP）**：大量处理器协同工作

### 1.4 操作系统的结构

#### 1.4.1 单体结构
- **特点**：所有功能在一个地址空间
- **优点**：效率高，通信简单
- **缺点**：可靠性差，难以维护
- **例子**：早期Unix

#### 1.4.2 层次结构
- **特点**：分层设计，上层调用下层
- **优点**：结构清晰，易于调试
- **缺点**：效率较低，层次划分困难
- **例子**：THE系统

#### 1.4.3 微内核结构
- **特点**：内核功能最小化，服务在用户空间
- **优点**：可靠性高，易于扩展
- **缺点**：性能开销大
- **例子**：Mach、QNX

#### 1.4.4 混合结构
- **特点**：结合多种结构的优点
- **例子**：Windows NT、Linux

### 1.5 系统调用

系统调用是操作系统提供给应用程序的编程接口，是应用程序请求操作系统服务的唯一途径。

5. **内核根据系统调用号调用相应处理程序**
6. **执行系统调用**
7. **返回结果给用户程序**
8. **CPU切换回用户态**

## 2. 进程管理

### 2.1 进程的概念

#### 2.1.1 进程的定义

进程是程序在执行过程中的一个实例，是系统进行资源分配和调度的基本单位。进程不仅包括程序代码（文本段），还包括当前活动的状态，如程序计数器的值、处理器寄存器的内容、进程堆栈等。

#### 2.1.2 进程与程序的区别

| 特征 | 程序 | 进程 |
|------|------|------|
| 定义 | 静态的代码集合 | 动态的执行实体 |
| 生命周期 | 永久存在 | 有创建和终止 |
| 资源 | 不占用系统资源 | 占用CPU、内存等资源 |
| 状态 | 无状态概念 | 有多种状态 |
| 数量关系 | 一个程序 | 可对应多个进程 |

#### 2.1.3 进程控制块（PCB）

进程控制块是操作系统用来管理进程的数据结构，包含了进程的所有信息：

```c
struct process_control_block {
    int pid;                    // 进程标识符
    int state;                  // 进程状态
    int priority;               // 进程优先级
    struct cpu_context cpu;     // CPU上下文
    struct memory_info memory;   // 内存信息
    struct file_table files;    // 打开文件表
    struct process *parent;      // 父进程指针
    struct list_head children;  // 子进程链表
    struct list_head siblings;  // 兄弟进程链表
    unsigned long start_time;    // 创建时间
    unsigned long cpu_time;      // CPU使用时间
};
```

### 2.2 进程状态

#### 2.2.1 五状态模型

- **新建状态（New）**：进程正在被创建，PCB已分配但尚未完全初始化
- **就绪状态（Ready）**：进程已准备好运行，等待CPU分配
- **运行状态（Running）**：进程正在CPU上执行
- **阻塞状态（Blocked/Waiting）**：进程等待某个事件发生（如I/O完成）
- **终止状态（Terminated）**：进程执行完毕，等待父进程回收资源

#### 2.2.2 状态转换图

```
    创建
     ↓
   新建状态
     ↓ 调度
   就绪状态 ←──── 时间片用完 ──── 运行状态
     ↑                           ↓
     └──── 事件完成 ──── 阻塞状态 ←┘
                           ↓ 等待事件
                       终止状态
```

#### 2.2.3 进程状态的实现

```c
// 进程状态枚举
enum process_state {
    TASK_NEW = 0,
    TASK_RUNNING = 1,
    TASK_READY = 2,
    TASK_BLOCKED = 4,
    TASK_TERMINATED = 8
};

// 状态转换函数
void change_process_state(struct process *p, enum process_state new_state) {
    switch (new_state) {
        case TASK_READY:
            add_to_ready_queue(p);
            break;
        case TASK_BLOCKED:
            remove_from_ready_queue(p);
            add_to_wait_queue(p);
            break;
        case TASK_TERMINATED:
            cleanup_process_resources(p);
            break;
    }
    p->state = new_state;
}
```

### 2.3 进程调度算法

#### 2.3.1 调度算法的评价指标

- **CPU利用率**：CPU忙碌时间的百分比
- **吞吐量**：单位时间内完成的进程数
- **周转时间**：从进程提交到完成的总时间
- **等待时间**：进程在就绪队列中等待的总时间
- **响应时间**：从提交请求到首次响应的时间

#### 2.3.2 先来先服务（FCFS）

**算法描述**：按照进程到达的先后顺序进行调度。

**实现代码**：
```c
struct process *fcfs_schedule(struct ready_queue *queue) {
    if (queue_empty(queue)) {
        return NULL;
    }
    return queue_front(queue);  // 返回队首进程
}

void fcfs_add_process(struct ready_queue *queue, struct process *p) {
    queue_push_back(queue, p);  // 添加到队尾
}
```

**示例分析**：
```
进程  到达时间  服务时间
P1      0        24
P2      1         3
P3      2         3

执行顺序：P1 → P2 → P3
完成时间：P1=24, P2=27, P3=30
周转时间：P1=24, P2=26, P3=28
平均周转时间：(24+26+28)/3 = 26
```

**优缺点**：
- 优点：简单易实现，公平
- 缺点：平均等待时间长，不适合交互式系统

#### 2.3.3 短作业优先（SJF）

**算法描述**：优先调度执行时间最短的进程。

**实现代码**：
```c
struct process *sjf_schedule(struct ready_queue *queue) {
    struct process *shortest = NULL;
    struct process *current;
    
    queue_for_each(current, queue) {
        if (shortest == NULL || 
            current->burst_time < shortest->burst_time) {
            shortest = current;
        }
    }
    
    if (shortest) {
        queue_remove(queue, shortest);
    }
    
    return shortest;
}
```

**抢占式SJF（SRTF）**：
```c
void srtf_check_preemption(struct process *running, struct process *new) {
    if (new->remaining_time < running->remaining_time) {
        // 抢占当前进程
        preempt_process(running);
        schedule_process(new);
    }
}
```

#### 2.3.4 时间片轮转（RR）

**算法描述**：每个进程分配固定的时间片，时间片用完后切换到下一个进程。

**实现代码**：
```c
#define TIME_QUANTUM 10

struct process *rr_schedule(struct ready_queue *queue) {
    static int time_left = TIME_QUANTUM;
    struct process *current = queue_front(queue);
    
    if (current == NULL) {
        return NULL;
    }
    
    time_left--;
    
    if (time_left <= 0 || current->burst_time <= 0) {
        // 时间片用完或进程完成
        queue_pop_front(queue);
        
        if (current->burst_time > 0) {
            // 进程未完成，重新加入队尾
            queue_push_back(queue, current);
        }
        
        time_left = TIME_QUANTUM;
        return queue_front(queue);
    }
    
    return current;
}
```

**时间片大小的影响**：
- 时间片太大：退化为FCFS
- 时间片太小：上下文切换开销大
- 一般设置为10-100ms

#### 2.3.5 优先级调度

**静态优先级**：
```c
struct process *priority_schedule(struct ready_queue *queue) {
    struct process *highest = NULL;
    struct process *current;
    
    queue_for_each(current, queue) {
        if (highest == NULL || 
            current->priority > highest->priority) {
            highest = current;
        }
    }
    
    if (highest) {
        queue_remove(queue, highest);
    }
    
    return highest;
}
```

**动态优先级（老化技术）**：
```c
void aging_update_priority(struct ready_queue *queue) {
    struct process *p;
    
    queue_for_each(p, queue) {
        p->waiting_time++;
        if (p->waiting_time > AGING_THRESHOLD) {
            p->priority++;  // 提高优先级
            p->waiting_time = 0;
        }
    }
}
```

#### 2.3.6 多级反馈队列

**算法特点**：
- 多个优先级队列
- 高优先级队列使用较小时间片
- 进程在队列间动态移动

**实现代码**：
```c
#define MAX_QUEUES 3

struct multilevel_queue {
    struct ready_queue queues[MAX_QUEUES];
    int time_quantum[MAX_QUEUES];
};

struct process *mlfq_schedule(struct multilevel_queue *mq) {
    for (int i = 0; i < MAX_QUEUES; i++) {
        if (!queue_empty(&mq->queues[i])) {
            return queue_front(&mq->queues[i]);
        }
    }
    return NULL;
}

void mlfq_time_expired(struct multilevel_queue *mq, 
                       struct process *p, int current_level) {
    // 降级到下一个队列
    if (current_level < MAX_QUEUES - 1) {
        queue_push_back(&mq->queues[current_level + 1], p);
    } else {
        queue_push_back(&mq->queues[current_level], p);
    }
}
```

### 2.4 线程

#### 2.4.1 线程的概念

线程是进程内的执行单元，是CPU调度的基本单位。同一进程内的线程共享进程的地址空间和资源，但每个线程有自己的程序计数器、寄存器集合和栈空间。

#### 2.4.2 线程与进程的比较

| 特征 | 进程 | 线程 |
|------|------|------|
| 定义 | 资源分配的基本单位 | CPU调度的基本单位 |
| 地址空间 | 独立的地址空间 | 共享进程地址空间 |
| 通信 | 需要IPC机制 | 直接读写共享变量 |
| 创建开销 | 大（需要分配地址空间） | 小（只需分配栈和寄存器） |
| 切换开销 | 大（需要切换地址空间） | 小（只需切换寄存器） |
| 崩溃影响 | 不影响其他进程 | 可能影响同进程其他线程 |

#### 2.4.3 线程的实现模型

**用户级线程**：
```c
// 用户级线程控制块
struct user_thread {
    int tid;
    void *stack_ptr;
    struct cpu_context context;
    enum thread_state state;
    struct user_thread *next;
};

// 用户级线程调度
void user_thread_yield() {
    struct user_thread *current = get_current_thread();
    struct user_thread *next = get_next_thread();
    
    if (next != current) {
        save_context(&current->context);
        restore_context(&next->context);
        set_current_thread(next);
    }
}
```

**内核级线程**：
```c
// 内核级线程创建
int kernel_thread_create(void (*func)(void *), void *arg) {
    struct task_struct *thread = alloc_task_struct();
    
    thread->pid = get_next_pid();
    thread->stack = alloc_kernel_stack();
    thread->mm = current->mm;  // 共享地址空间
    
    setup_thread_context(thread, func, arg);
    add_to_ready_queue(thread);
    
    return thread->pid;
}
```

**混合模型**：
- 多个用户线程映射到多个内核线程
- 结合两种模型的优点
- 复杂度较高

#### 2.4.4 线程同步

**互斥锁（Mutex）**：
```c
struct mutex {
    int locked;
    struct thread_queue wait_queue;
};

void mutex_lock(struct mutex *m) {
    while (atomic_test_and_set(&m->locked)) {
        add_to_wait_queue(&m->wait_queue, current_thread());
        thread_block();
    }
}

void mutex_unlock(struct mutex *m) {
    m->locked = 0;
    struct thread *waiting = remove_from_wait_queue(&m->wait_queue);
    if (waiting) {
        thread_wakeup(waiting);
    }
}
```

**条件变量**：
```c
struct condition {
    struct thread_queue wait_queue;
};

void cond_wait(struct condition *c, struct mutex *m) {
    add_to_wait_queue(&c->wait_queue, current_thread());
    mutex_unlock(m);
    thread_block();
    mutex_lock(m);
}

void cond_signal(struct condition *c) {
    struct thread *waiting = remove_from_wait_queue(&c->wait_queue);
    if (waiting) {
        thread_wakeup(waiting);
    }
}
```

#### 2.4.5 线程池

**线程池的实现**：
```c
struct thread_pool {
    struct thread *threads;
    int num_threads;
    struct task_queue tasks;
    struct mutex task_mutex;
    struct condition task_cond;
    int shutdown;
};

void *worker_thread(void *arg) {
    struct thread_pool *pool = (struct thread_pool *)arg;
    
    while (!pool->shutdown) {
        mutex_lock(&pool->task_mutex);
        
        while (task_queue_empty(&pool->tasks) && !pool->shutdown) {
            cond_wait(&pool->task_cond, &pool->task_mutex);
        }
        
        if (pool->shutdown) {
            mutex_unlock(&pool->task_mutex);
            break;
        }
        
        struct task *task = task_queue_pop(&pool->tasks);
        mutex_unlock(&pool->task_mutex);
        
        // 执行任务
        task->func(task->arg);
        free(task);
    }
    
    return NULL;
}

int thread_pool_add_task(struct thread_pool *pool, 
                        void (*func)(void *), void *arg) {
    struct task *task = malloc(sizeof(struct task));
    task->func = func;
    task->arg = arg;
    
    mutex_lock(&pool->task_mutex);
    task_queue_push(&pool->tasks, task);
    cond_signal(&pool->task_cond);
    mutex_unlock(&pool->task_mutex);
    
    return 0;
}
```

**线程的优势：**
- **响应性更好**：一个线程阻塞时，其他线程可以继续执行
- **资源共享**：线程间可以直接共享内存和文件
- **经济性**：创建和切换成本低于进程
- **多处理器架构的利用**：可以在多个CPU上并行执行

## 3. 内存管理

### 3.1 内存管理的目标

#### 3.1.1 基本目标

- **重定位**：程序可以加载到内存的任意位置，支持动态加载
- **保护**：防止进程访问其他进程的内存，确保系统安全
- **共享**：允许多个进程共享内存区域，提高资源利用率
- **逻辑组织**：支持模块化编程，便于程序开发和维护
- **物理组织**：在主存和辅存之间高效移动信息

#### 3.1.2 性能目标

- **最小化内存碎片**：减少内存浪费，提高利用率
- **快速分配和释放**：提高内存操作效率
- **局部性优化**：利用程序的时间和空间局部性

```c
    int is_free;
    void *address;

// 首次适应算法
void *first_fit_alloc(size_t size) {
    struct memory_block *current = memory_list;
    
    while (current != NULL) {
        if (current->is_free && current->size >= size) {
            if (current->size > size + sizeof(struct memory_block)) {
                // 分割内存块
                struct memory_block *new_block = 
                    (struct memory_block *)((char *)current->address + size);
                new_block->size = current->size - size;
                new_block->is_free = 1;
                new_block->address = (char *)current->address + size;
                new_block->next = current->next;
                new_block->prev = current;
                
                if (current->next) {
                    current->next->prev = new_block;
                }
                current->next = new_block;
                current->size = size;
            }
            
            current->is_free = 0;
            return current->address;
        }
        current = current->next;
    }
    
    return NULL;
}

// 内存合并
void coalesce_free_blocks(struct memory_block *block) {
    // 向后合并
    while (block->next && block->next->is_free) {
        struct memory_block *next = block->next;
        block->size += next->size;
        block->next = next->next;
        if (next->next) {
            next->next->prev = block;
        }
        free(next);
    }
    
    // 向前合并
    while (block->prev && block->prev->is_free) {
        struct memory_block *prev = block->prev;
        prev->size += block->size;
        prev->next = block->next;
        if (block->next) {
            block->next->prev = prev;
        }
        free(block);
        block = prev;
    }
}
```

**分页系统**：

```c
#define PAGE_SIZE 4096
#define PAGE_TABLE_SIZE 1024

struct page_table_entry {
    unsigned int present : 1;      // 页面是否在内存中
    unsigned int writable : 1;     // 是否可写
    unsigned int user : 1;         // 用户态是否可访问
    unsigned int accessed : 1;     // 是否被访问过
    unsigned int dirty : 1;        // 是否被修改过
    unsigned int reserved : 7;     // 保留位
    unsigned int frame : 20;       // 物理页框号
};

struct page_table {
    struct page_table_entry entries[PAGE_TABLE_SIZE];
};

// 地址转换
physical_addr_t translate_address(virtual_addr_t virtual_addr, struct page_table *pt) {
    unsigned int page_number = virtual_addr >> 12;  // 高20位为页号
    unsigned int offset = virtual_addr & 0xFFF;     // 低12位为页内偏移
    
    if (page_number >= PAGE_TABLE_SIZE) {
        return 0;  // 页号超出范围
    }
    
    struct page_table_entry *pte = &pt->entries[page_number];
    
    if (!pte->present) {
        handle_page_fault(virtual_addr);
        return 0;
    }
    
    pte->accessed = 1;
    
    physical_addr_t physical_addr = (pte->frame << 12) | offset;
    return physical_addr;
}
```

### 3.3 虚拟内存

#### 3.3.1 虚拟内存的概念

虚拟内存是一种内存管理技术，使程序认为自己拥有连续可用的内存，而实际上物理内存可能是分散的，甚至部分存储在磁盘上。它通过硬件和操作系统的协作，为每个进程提供独立的地址空间。

#### 3.3.2 虚拟内存的实现机制

**请求分页**：
```c
struct vm_area_struct {
    unsigned long vm_start;         // 虚拟地址起始
    unsigned long vm_end;           // 虚拟地址结束
    unsigned long vm_flags;         // 访问权限
    struct file *vm_file;           // 映射的文件
    unsigned long vm_pgoff;         // 文件偏移
    struct vm_area_struct *vm_next; // 下一个VMA
};

int handle_page_fault(unsigned long address, unsigned long error_code) {
    struct vm_area_struct *vma = find_vma(current->mm, address);
    
    if (!vma || address < vma->vm_start) {
        // 访问无效地址
        send_signal(current, SIGSEGV);
        return -1;
    }
    
    // 检查访问权限
    if ((error_code & PAGE_FAULT_WRITE) && !(vma->vm_flags & VM_WRITE)) {
        send_signal(current, SIGSEGV);
        return -1;
    }
    
    // 分配物理页面
    struct page *page = alloc_page(GFP_KERNEL);
    if (!page) {
        return -ENOMEM;
    }
    
    // 如果是文件映射，从文件读取数据
    if (vma->vm_file) {
        read_file_page(vma->vm_file, page, 
                      (address - vma->vm_start) + vma->vm_pgoff);
    } else {
        // 匿名映射，清零页面
        clear_page(page_address(page));
    }
    
    // 更新页表
    install_page(current->mm, address, page, vma->vm_flags);
    
    return 0;
}
```

**写时复制（COW）**：
```c
int copy_process_memory(struct task_struct *child, struct task_struct *parent) {
    struct vm_area_struct *vma;
    
    // 复制VMA结构
    for (vma = parent->mm->mmap; vma; vma = vma->vm_next) {
        struct vm_area_struct *new_vma = copy_vma(vma);
        add_vma_to_mm(child->mm, new_vma);
    }
    
    // 复制页表，设置COW标志
    copy_page_tables(child->mm, parent->mm);
    
    return 0;
}

int handle_cow_fault(struct vm_area_struct *vma, unsigned long address) {
    struct page *old_page = get_page_from_address(address);
    struct page *new_page;
    
    // 检查是否真的需要COW
    if (page_count(old_page) == 1) {
        // 只有一个引用，直接设置为可写
        make_page_writable(vma->vm_mm, address);
        return 0;
    }
    
    // 分配新页面
    new_page = alloc_page(GFP_KERNEL);
    if (!new_page) {
        return -ENOMEM;
    }
    
    // 复制页面内容
    copy_page(page_address(new_page), page_address(old_page));
    
    // 更新页表
    install_page(vma->vm_mm, address, new_page, vma->vm_flags | VM_WRITE);
    
    // 减少旧页面引用计数
    put_page(old_page);
    
    return 0;
}
```

#### 3.3.3 虚拟内存的优势

- **程序大小不受物理内存限制**：通过页面换入换出机制
- **提高内存利用率**：多个进程可以共享相同的物理页面
- **简化程序设计**：程序员不需要考虑物理内存布局
- **提供内存保护**：每个进程有独立的地址空间
- **支持内存映射文件**：文件可以直接映射到内存中

```c
struct fifo_page_queue {
    struct page *pages[MAX_PAGES];
    int front;
    int rear;
    int count;
};

struct page *fifo_select_victim(struct fifo_page_queue *queue) {
    if (queue->count == 0) {
        return NULL;
    }
    
    struct page *victim = queue->pages[queue->front];
    queue->front = (queue->front + 1) % MAX_PAGES;
    queue->count--;
    
    return victim;
}

void fifo_add_page(struct fifo_page_queue *queue, struct page *page) {
    if (queue->count < MAX_PAGES) {
        queue->pages[queue->rear] = page;
        queue->rear = (queue->rear + 1) % MAX_PAGES;
        queue->count++;
    }
}
```

#### 3.4.3 最近最少使用（LRU）

**链表实现**：
```c
struct lru_node {
    struct page *page;
    struct lru_node *prev;
    struct lru_node *next;
};

struct lru_list {
    struct lru_node *head;
    struct lru_node *tail;
    int count;
};

void lru_access_page(struct lru_list *list, struct page *page) {
    struct lru_node *node = find_node(list, page);
    
    if (node) {
        // 页面已在内存中，移到链表头部
        if (node != list->head) {
            // 从当前位置移除
            if (node->prev) node->prev->next = node->next;
            if (node->next) node->next->prev = node->prev;
            if (node == list->tail) list->tail = node->prev;
            
            // 移到头部
            node->prev = NULL;
            node->next = list->head;
            if (list->head) list->head->prev = node;
            list->head = node;
            if (!list->tail) list->tail = node;
        }
    } else {
        // 页面不在内存中
        if (list->count >= MAX_PAGES) {
            // 内存已满，移除尾部节点
            struct lru_node *victim = list->tail;
            if (victim->prev) {
                victim->prev->next = NULL;
                list->tail = victim->prev;
            } else {
                list->head = list->tail = NULL;
            }
            swap_out_page(victim->page);
            free(victim);
            list->count--;
        }
        
        // 添加新页面到头部
        struct lru_node *new_node = malloc(sizeof(struct lru_node));
        new_node->page = page;
        new_node->prev = NULL;
        new_node->next = list->head;
        
        if (list->head) {
            list->head->prev = new_node;
        } else {
            list->tail = new_node;
        }
        list->head = new_node;
        list->count++;
    }
}
```

#### 3.4.4 时钟算法（Clock）

**基本时钟算法**：
```c
struct clock_page {
    struct page *page;
    int reference_bit;
    struct clock_page *next;
};

struct clock_algorithm {
    struct clock_page *current;  // 时钟指针
    int page_count;
};

struct page *clock_select_victim(struct clock_algorithm *clock) {
    while (1) {
        if (clock->current->reference_bit == 0) {
            // 找到未被引用的页面
            struct page *victim = clock->current->page;
            clock->current = clock->current->next;
            return victim;
        } else {
            // 清除引用位，继续查找
            clock->current->reference_bit = 0;
            clock->current = clock->current->next;
        }
    }
}
```

**增强时钟算法**：
```c
struct enhanced_clock_page {
    struct page *page;
    int reference_bit;
    int modify_bit;
    struct enhanced_clock_page *next;
};

// 四个类别的页面优先级：
// (0,0) - 最近未访问且未修改 (最佳选择)
// (0,1) - 最近未访问但已修改
// (1,0) - 最近访问但未修改
// (1,1) - 最近访问且已修改 (最差选择)

struct page *enhanced_clock_select_victim(struct enhanced_clock_algorithm *clock) {
    struct enhanced_clock_page *start = clock->current;
    
    // 第一轮：查找(0,0)页面
    do {
        if (clock->current->reference_bit == 0 && clock->current->modify_bit == 0) {
            struct page *victim = clock->current->page;
            clock->current = clock->current->next;
            return victim;
        }
        clock->current = clock->current->next;
    } while (clock->current != start);
    
    // 第二轮：查找(0,1)页面，同时清除引用位
    do {
        if (clock->current->reference_bit == 0 && clock->current->modify_bit == 1) {
            struct page *victim = clock->current->page;
            clock->current = clock->current->next;
            return victim;
        }
        clock->current->reference_bit = 0;  // 清除引用位
        clock->current = clock->current->next;
    } while (clock->current != start);
    
    // 重复第一轮和第二轮
    return enhanced_clock_select_victim(clock);
}
```

```c
struct sequential_file {

int read_sequential_record(struct sequential_file *sf, int record_num, void *buffer) {
    if (record_num >= sf->record_count) {

    size_t bytes_written = fwrite(data, 1, sf->record_size, sf->file_handle);
    if (bytes_written == sf->record_size) {
        sf->record_count++;
        return 0;
    }
    
    return -1;
}
```

#### 4.2.2 索引文件

```c
        if_file->index_table = realloc(if_file->index_table,

    strncpy(entry->key, key, sizeof(entry->key) - 1);
    entry->key[sizeof(entry->key) - 1] = '\0';
    entry->file_offset = offset;
    entry->record_size = size;

// 通过索引查找记录
int find_indexed_record(struct indexed_file *if_file, const char *key, 
                       void *buffer, size_t buffer_size) {
    // 二分查找索引
    int left = 0, right = if_file->index_count - 1;
    
    while (left <= right) {
        int mid = (left + right) / 2;
        int cmp = strcmp(key, if_file->index_table[mid].key);
        
        if (cmp == 0) {
            // 找到索引项
            struct index_entry *entry = &if_file->index_table[mid];

                return -1;  // 缓冲区太小
            }
            
            // 读取数据
            fseek(if_file->data_file, entry->file_offset, SEEK_SET);
            size_t bytes_read = fread(buffer, 1, entry->record_size, 
                                    if_file->data_file);
            
            return (bytes_read == entry->record_size) ? 0 : -1;
        } else if (cmp < 0) {
            right = mid - 1;
        } else {
            left = mid + 1;
        }
    }
    
    return -1;  // 未找到
}
```

#### 4.2.3 哈希文件

**特点**：使用哈希函数快速定位记录，平均访问时间为O(1)。

```c
#define HASH_TABLE_SIZE 1024

struct hash_bucket {
    char key[64];
    long file_offset;
    size_t record_size;
    struct hash_bucket *next;  // 处理冲突的链表
};

struct hash_file {
    char filename[256];
    FILE *data_file;
    struct hash_bucket *hash_table[HASH_TABLE_SIZE];
};

// 哈希函数
unsigned int hash_function(const char *key) {
    unsigned int hash = 0;
    while (*key) {
        hash = hash * 31 + *key;
        key++;
    }
    return hash % HASH_TABLE_SIZE;
}

// 插入记录
int insert_hash_record(struct hash_file *hf, const char *key, 
                      const void *data, size_t size) {
    unsigned int index = hash_function(key);
    
    // 检查是否已存在
    struct hash_bucket *bucket = hf->hash_table[index];
    while (bucket) {
        if (strcmp(bucket->key, key) == 0) {
            return -1;  // 键已存在
        }
        bucket = bucket->next;
    }
    
    // 写入数据文件
    fseek(hf->data_file, 0, SEEK_END);
    long offset = ftell(hf->data_file);
    size_t bytes_written = fwrite(data, 1, size, hf->data_file);
    
    if (bytes_written != size) {
        return -1;
    }
    
    // 创建哈希桶
    struct hash_bucket *new_bucket = malloc(sizeof(struct hash_bucket));
    strncpy(new_bucket->key, key, sizeof(new_bucket->key) - 1);
    new_bucket->key[sizeof(new_bucket->key) - 1] = '\0';
    new_bucket->file_offset = offset;
    new_bucket->record_size = size;
    new_bucket->next = hf->hash_table[index];
    
    hf->hash_table[index] = new_bucket;
    
    return 0;
}

// 查找记录
int find_hash_record(struct hash_file *hf, const char *key, 
                    void *buffer, size_t buffer_size) {
    unsigned int index = hash_function(key);
    struct hash_bucket *bucket = hf->hash_table[index];
    
    while (bucket) {
        if (strcmp(bucket->key, key) == 0) {
            if (bucket->record_size > buffer_size) {
                return -1;
            }
            
            fseek(hf->data_file, bucket->file_offset, SEEK_SET);
            size_t bytes_read = fread(buffer, 1, bucket->record_size, 
                                    hf->data_file);
            
            return (bytes_read == bucket->record_size) ? 0 : -1;
        }
        bucket = bucket->next;
    }
    
    return -1;  // 未找到
```

#### 4.3.2 树形目录

**特点**：层次化的目录结构，支持子目录。

```c
struct directory_entry {
    char name[256];
    int is_directory;
    union {
        struct {
            size_t file_size;
            long inode_number;
            int permissions;
        } file_info;
        struct {
            struct directory_entry *entries;
            int entry_count;
            int entry_capacity;
        } dir_info;
    };
    time_t create_time;
    time_t modify_time;
};

struct tree_directory {
    struct directory_entry root;
};

// 路径解析
struct directory_entry *resolve_path(struct tree_directory *tree, 
                                   const char *path) {
    if (path[0] != '/') {
        return NULL;  // 必须是绝对路径
    }
    
    struct directory_entry *current = &tree->root;
    char *path_copy = strdup(path + 1);  // 跳过根目录的'/'
    char *token = strtok(path_copy, "/");
    
    while (token && current && current->is_directory) {
        struct directory_entry *found = NULL;
        
        for (int i = 0; i < current->dir_info.entry_count; i++) {
            if (strcmp(current->dir_info.entries[i].name, token) == 0) {
                found = &current->dir_info.entries[i];
                break;
            }
        }
        
        if (!found) {
            free(path_copy);
            return NULL;  // 路径不存在
        }
        
        current = found;
        token = strtok(NULL, "/");
    }
    
    free(path_copy);
    return current;
}

// 创建目录
int create_directory(struct tree_directory *tree, const char *path) {
    char *parent_path = strdup(path);
    char *last_slash = strrchr(parent_path, '/');
    
    if (!last_slash) {
        free(parent_path);
        return -1;
    }
    
    *last_slash = '\0';
    const char *dir_name = last_slash + 1;
    
    struct directory_entry *parent = resolve_path(tree, parent_path);
    free(parent_path);
    
    if (!parent || !parent->is_directory) {
        return -1;  // 父目录不存在
    }
    
    // 检查是否已存在
    for (int i = 0; i < parent->dir_info.entry_count; i++) {
        if (strcmp(parent->dir_info.entries[i].name, dir_name) == 0) {
            return -1;  // 已存在
        }
    }
    
    // 扩展目录项数组
    if (parent->dir_info.entry_count >= parent->dir_info.entry_capacity) {
        parent->dir_info.entry_capacity *= 2;
        parent->dir_info.entries = realloc(parent->dir_info.entries,
            parent->dir_info.entry_capacity * sizeof(struct directory_entry));
    }
    
    // 创建新目录项
    struct directory_entry *new_entry = 
        &parent->dir_info.entries[parent->dir_info.entry_count];
    
    strncpy(new_entry->name, dir_name, sizeof(new_entry->name) - 1);
    new_entry->name[sizeof(new_entry->name) - 1] = '\0';
    new_entry->is_directory = 1;
    new_entry->dir_info.entries = malloc(sizeof(struct directory_entry) * 10);
    new_entry->dir_info.entry_count = 0;
    new_entry->dir_info.entry_capacity = 10;
    new_entry->create_time = time(NULL);
    new_entry->modify_time = new_entry->create_time;
    
    parent->dir_info.entry_count++;
    
    return 0;
}
```

### 4.4 磁盘调度算法

#### 4.4.1 先来先服务（FCFS）

**算法描述**：按请求到达的顺序处理磁盘I/O请求。

```c
struct disk_request {
    int track_number;
    int sector_number;
    int operation;  // READ or WRITE
    void *buffer;
    struct disk_request *next;
};

struct fcfs_scheduler {
    struct disk_request *head;
    struct disk_request *tail;
    int current_track;
};

void fcfs_add_request(struct fcfs_scheduler *scheduler, 
                     struct disk_request *request) {
    request->next = NULL;
    
    if (scheduler->tail) {
        scheduler->tail->next = request;
    } else {
        scheduler->head = request;
    }
    
    scheduler->tail = request;
}

struct disk_request *fcfs_get_next_request(struct fcfs_scheduler *scheduler) {
    if (!scheduler->head) {
        return NULL;
    }
    
    struct disk_request *request = scheduler->head;
    scheduler->head = request->next;
    
    if (!scheduler->head) {
        scheduler->tail = NULL;
    }
    
    return request;
}

// 计算寻道时间
int calculate_seek_time(struct fcfs_scheduler *scheduler, 
                       struct disk_request *request) {
    int seek_distance = abs(request->track_number - scheduler->current_track);
    scheduler->current_track = request->track_number;
    return seek_distance;  // 简化模型，实际时间需要考虑硬件特性
}
```

#### 4.4.2 最短寻道时间优先（SSTF）

**算法描述**：选择距离当前磁头位置最近的请求。

```c
struct sstf_scheduler {
    struct disk_request *requests[MAX_REQUESTS];
    int request_count;
    int current_track;
};

struct disk_request *sstf_get_next_request(struct sstf_scheduler *scheduler) {
    if (scheduler->request_count == 0) {
        return NULL;
    }
    
    int best_index = 0;
    int min_seek_distance = abs(scheduler->requests[0]->track_number - 
                               scheduler->current_track);
    
    // 找到寻道距离最短的请求
    for (int i = 1; i < scheduler->request_count; i++) {
        int seek_distance = abs(scheduler->requests[i]->track_number - 
                              scheduler->current_track);
        
        if (seek_distance < min_seek_distance) {
            min_seek_distance = seek_distance;
            best_index = i;
        }
    }
    
    struct disk_request *request = scheduler->requests[best_index];
    scheduler->current_track = request->track_number;
    
    // 移除已选择的请求
    for (int i = best_index; i < scheduler->request_count - 1; i++) {
        scheduler->requests[i] = scheduler->requests[i + 1];
    }
    scheduler->request_count--;
    
    return request;
}
```

#### 4.4.3 扫描算法（SCAN）

**算法描述**：磁头在磁盘上来回扫描，类似电梯算法。

```c
struct scan_scheduler {
    struct disk_request *requests[MAX_REQUESTS];
    int request_count;
    int current_track;
    int direction;  // 1 for up, -1 for down
    int max_track;
};

struct disk_request *scan_get_next_request(struct scan_scheduler *scheduler) {
    if (scheduler->request_count == 0) {
        return NULL;
    }
    
    int best_index = -1;
    
    if (scheduler->direction == 1) {
        // 向上扫描，找最小的大于等于当前位置的磁道
        int min_track = scheduler->max_track + 1;
        
        for (int i = 0; i < scheduler->request_count; i++) {
            if (scheduler->requests[i]->track_number >= scheduler->current_track &&
                scheduler->requests[i]->track_number < min_track) {
                min_track = scheduler->requests[i]->track_number;
                best_index = i;
            }
        }
        
        if (best_index == -1) {
            // 没有向上的请求，改变方向
            scheduler->direction = -1;
            return scan_get_next_request(scheduler);
        }
    } else {
        // 向下扫描，找最大的小于等于当前位置的磁道
        int max_track = -1;
        
        for (int i = 0; i < scheduler->request_count; i++) {
            if (scheduler->requests[i]->track_number <= scheduler->current_track &&
                scheduler->requests[i]->track_number > max_track) {
                max_track = scheduler->requests[i]->track_number;
                best_index = i;
            }
        }
        
        if (best_index == -1) {
            // 没有向下的请求，改变方向
            scheduler->direction = 1;
            return scan_get_next_request(scheduler);
        }
    }
    
    struct disk_request *request = scheduler->requests[best_index];
    scheduler->current_track = request->track_number;
    
    // 移除已选择的请求
    for (int i = best_index; i < scheduler->request_count - 1; i++) {
        scheduler->requests[i] = scheduler->requests[i + 1];
    }
    scheduler->request_count--;
    
    return request;
}
```

### 5.2 I/O控制方式

#### 5.2.1 程序直接控制（轮询方式）

**特点**：CPU直接控制I/O操作，通过轮询检查设备状态。

```c
struct polling_io_controller {
    volatile unsigned int *status_register;
    volatile unsigned int *data_register;
    volatile unsigned int *control_register;
};

// 轮询方式读取数据
int polling_read(struct polling_io_controller *ctrl, char *buffer, size_t count) {
    size_t bytes_read = 0;
    
    while (bytes_read < count) {
        // 轮询设备状态
        while ((*ctrl->status_register & STATUS_DATA_READY) == 0) {
            cpu_relax();  // 让出CPU一个时钟周期
        }
        
        // 读取数据
        buffer[bytes_read] = *ctrl->data_register & 0xFF;
        bytes_read++;
        
        // 清除数据就绪标志
        *ctrl->control_register |= CONTROL_CLEAR_DATA_READY;
    }
    
    return bytes_read;
}
```

#### 5.2.2 中断驱动方式

**特点**：设备完成操作后发送中断信号，CPU响应中断处理。

```c
struct interrupt_io_controller {
    volatile unsigned int *status_register;
    volatile unsigned int *data_register;
    volatile unsigned int *interrupt_register;
    
    int irq_number;
    
    // 缓冲区管理
    char *read_buffer;
    size_t read_buffer_size;
    size_t read_pos;
    
    // 同步原语
    struct semaphore read_sem;
    struct wait_queue *read_wait;
};

// 中断处理函数
void io_interrupt_handler(int irq, void *dev_data) {
    struct interrupt_io_controller *ctrl = (struct interrupt_io_controller *)dev_data;
    unsigned int status = *ctrl->status_register;
    
    if (status & STATUS_DATA_READY) {
        // 有数据可读
        if (ctrl->read_pos < ctrl->read_buffer_size) {
            ctrl->read_buffer[ctrl->read_pos++] = *ctrl->data_register & 0xFF;
            
            // 如果缓冲区满了，唤醒读进程
            if (ctrl->read_pos >= ctrl->read_buffer_size) {
                wake_up(&ctrl->read_wait);
            }
        }
        
        // 清除中断标志
        *ctrl->control_register |= CONTROL_CLEAR_READ_INT;
    }
}

// 中断驱动的读操作
int interrupt_read(struct interrupt_io_controller *ctrl, char *buffer, size_t count) {
    ctrl->read_buffer = buffer;
    ctrl->read_buffer_size = count;
    ctrl->read_pos = 0;
    
    // 启用读中断
    *ctrl->interrupt_register |= INT_ENABLE_READ;
    
    // 等待数据到达
    wait_event(ctrl->read_wait, ctrl->read_pos >= ctrl->read_buffer_size);
    
    // 禁用读中断
    *ctrl->interrupt_register &= ~INT_ENABLE_READ;
    
    return ctrl->read_pos;
}
```

#### 5.2.3 DMA方式

**特点**：直接内存访问，减少CPU干预，提高数据传输效率。

```c
struct dma_controller {
    volatile unsigned int *source_addr_reg;
    volatile unsigned int *dest_addr_reg;
    volatile unsigned int *transfer_count_reg;
    volatile unsigned int *control_reg;
    volatile unsigned int *status_reg;
    
    int channel_number;
    struct completion transfer_complete;
};

// 初始化DMA传输
int dma_setup_transfer(struct dma_controller *dma, 
                      void *source, void *dest, size_t size) {
    if (size > DMA_MAX_TRANSFER_SIZE) {
        return -EINVAL;
    }
    
    // 检查DMA控制器是否空闲
    if (*dma->status_reg & DMA_STATUS_BUSY) {
        return -EBUSY;
    }
    
    // 设置传输参数
    *dma->source_addr_reg = virt_to_phys(source);
    *dma->dest_addr_reg = virt_to_phys(dest);
    *dma->transfer_count_reg = size;
    
    // 设置控制寄存器
    *dma->control_reg = DMA_CONTROL_ENABLE | DMA_CONTROL_INT_ENABLE;
    
    init_completion(&dma->transfer_complete);
    
    return 0;
}

// DMA中断处理函数
void dma_interrupt_handler(int irq, void *dev_data) {
    struct dma_controller *dma = (struct dma_controller *)dev_data;
    unsigned int status = *dma->status_reg;
    
    if (status & DMA_STATUS_COMPLETE) {
        complete(&dma->transfer_complete);
        *dma->status_reg |= DMA_STATUS_CLEAR_INT;
    }
}
```

### 5.3 缓冲技术

#### 5.3.1 单缓冲

**特点**：使用一个缓冲区，简单但效率较低。

```c
struct single_buffer {
    char *buffer;
    size_t buffer_size;
    size_t data_size;
    int is_full;
    int is_empty;
    
    struct mutex buffer_mutex;
    struct condition_variable not_full;
    struct condition_variable not_empty;
};

// 生产者写入数据
int single_buffer_write(struct single_buffer *sb, const void *data, size_t size) {
    mutex_lock(&sb->buffer_mutex);
    
    // 等待缓冲区不满
    while (sb->is_full) {
        condition_wait(&sb->not_full, &sb->buffer_mutex);
    }
    
    // 写入数据
    size_t write_size = (size > sb->buffer_size) ? sb->buffer_size : size;
    memcpy(sb->buffer, data, write_size);
    sb->data_size = write_size;
    sb->is_full = 1;
    sb->is_empty = 0;
    
    // 通知消费者
    condition_signal(&sb->not_empty);
    mutex_unlock(&sb->buffer_mutex);
    
    return write_size;
}
```

#### 5.3.2 双缓冲

**特点**：使用两个缓冲区交替工作，提高并发性。

```c
struct double_buffer {
    char *buffer1;
    char *buffer2;
    size_t buffer_size;
    
    char *active_buffer;    // 生产者使用
    char *ready_buffer;     // 消费者使用
    
    size_t active_size;
    size_t ready_size;
    
    struct mutex switch_mutex;
    struct semaphore producer_sem;
    struct semaphore consumer_sem;
};

// 生产者填充缓冲区
int double_buffer_produce(struct double_buffer *db, const void *data, size_t size) {
    semaphore_wait(&db->producer_sem);
    
    // 填充活动缓冲区
    size_t write_size = (size > db->buffer_size) ? db->buffer_size : size;
    memcpy(db->active_buffer, data, write_size);
    db->active_size = write_size;
    
    // 切换缓冲区
    mutex_lock(&db->switch_mutex);
    char *temp = db->active_buffer;
    db->active_buffer = db->ready_buffer;
    db->ready_buffer = temp;
    db->ready_size = db->active_size;
    db->active_size = 0;
    mutex_unlock(&db->switch_mutex);
    
    semaphore_signal(&db->consumer_sem);
    
    return write_size;
}
```

#### 5.3.3 环形缓冲

**特点**：多个缓冲区组成环形结构，支持多生产者多消费者。

```c
    char **buffers;
    size_t buffer_size;
    size_t *data_sizes;
```

```c
        bp->stats[pool_index].alloc_failures++;

    }

    spin_unlock(&pool->free_lock);
    
    return pool->buffers[buffer_index];
}
```

## 6. 进程同步与通信

### 6.1 临界区问题

#### 6.1.1 临界区的基本概念

**临界区**：访问共享资源的代码段，同一时刻只能有一个进程执行。

```c
// 临界区问题的经典示例：银行账户操作
struct bank_account {
    int balance;
    struct mutex account_lock;
};

// 不安全的转账操作（可能导致竞态条件）
void unsafe_transfer(struct bank_account *from, struct bank_account *to, int amount) {
    // 临界区开始
    int from_balance = from->balance;  // 读取源账户余额
    int to_balance = to->balance;      // 读取目标账户余额
    
    if (from_balance >= amount) {
        from->balance = from_balance - amount;  // 扣除源账户
        to->balance = to_balance + amount;      // 增加目标账户
    }
    // 临界区结束
}

// 安全的转账操作（使用互斥锁）
void safe_transfer(struct bank_account *from, struct bank_account *to, int amount) {
    // 避免死锁：按地址顺序获取锁
    struct bank_account *first = (from < to) ? from : to;
    struct bank_account *second = (from < to) ? to : from;
    
    mutex_lock(&first->account_lock);
    mutex_lock(&second->account_lock);
    
    // 临界区
    if (from->balance >= amount) {
        from->balance -= amount;
        to->balance += amount;
        printf("Transfer successful: %d from account %p to %p\n", amount, from, to);
    } else {
        printf("Transfer failed: insufficient funds\n");
    }
    
    mutex_unlock(&second->account_lock);
    mutex_unlock(&first->account_lock);
}
```

#### 6.1.2 临界区问题的解决准则

**解决临界区问题的要求：**
1. **互斥（Mutual Exclusion）**：同一时刻只能有一个进程在临界区
2. **进步（Progress）**：不在临界区的进程不能阻止其他进程进入
3. **有限等待（Bounded Waiting）**：进程等待进入临界区的时间有限

```c
// Peterson算法：两进程互斥的软件解决方案
static volatile int flag[2] = {0, 0};  // 进程意愿标志
static volatile int turn = 0;          // 轮转标志

void peterson_enter_critical_section(int process_id) {
    int other = 1 - process_id;
    
    flag[process_id] = 1;              // 表示想要进入临界区
    turn = other;                      // 让对方先进入
    
    // 等待条件：对方不想进入 或者 轮到自己
    while (flag[other] && turn == other) {
        cpu_relax();  // 忙等待
    }
}

void peterson_exit_critical_section(int process_id) {
    flag[process_id] = 0;  // 表示退出临界区
}

// 使用Peterson算法的临界区
void peterson_critical_section_example(int process_id) {
    peterson_enter_critical_section(process_id);
    
    // 临界区代码
    printf("Process %d is in critical section\n", process_id);
    shared_resource++;
    
    peterson_exit_critical_section(process_id);
}
```

### 6.2 同步机制

#### 6.2.1 信号量（Semaphore）

**信号量**：整型变量，支持原子的P（wait）、V（signal）操作。

```c
struct semaphore {
    int value;                    // 信号量值
    struct list_head wait_list;  // 等待队列
    struct spinlock lock;        // 保护信号量的自旋锁
};

// 初始化信号量
void semaphore_init(struct semaphore *sem, int initial_value) {
    sem->value = initial_value;
    INIT_LIST_HEAD(&sem->wait_list);
    spin_lock_init(&sem->lock);
}

// P操作（wait/down）
void semaphore_wait(struct semaphore *sem) {
    spin_lock(&sem->lock);
    
    if (sem->value > 0) {
        sem->value--;
        spin_unlock(&sem->lock);
    } else {
        // 将当前进程加入等待队列
        struct wait_entry entry;
        entry.process = current_process();
        list_add_tail(&entry.list, &sem->wait_list);
        
        spin_unlock(&sem->lock);
        
        // 阻塞当前进程
        schedule_block();
    }
}

// V操作（signal/up）
void semaphore_signal(struct semaphore *sem) {
    spin_lock(&sem->lock);
    
    if (list_empty(&sem->wait_list)) {
        sem->value++;
    } else {
        // 唤醒等待队列中的一个进程
        struct wait_entry *entry = list_first_entry(&sem->wait_list, 
                                                   struct wait_entry, list);
        list_del(&entry->list);
        wake_up_process(entry->process);
    }
    
    spin_unlock(&sem->lock);
}
```

**生产者-消费者问题的信号量解决方案**：

```c
#define BUFFER_SIZE 10

struct producer_consumer {
    int buffer[BUFFER_SIZE];
    int in, out;                    // 生产者和消费者索引
    
    struct semaphore empty;         // 空缓冲区数量
    struct semaphore full;          // 满缓冲区数量
    struct semaphore mutex;         // 互斥访问缓冲区
};

void producer_consumer_init(struct producer_consumer *pc) {
    pc->in = pc->out = 0;
    semaphore_init(&pc->empty, BUFFER_SIZE);  // 初始全为空
    semaphore_init(&pc->full, 0);             // 初始无满缓冲区
    semaphore_init(&pc->mutex, 1);            // 互斥锁
}

void producer(struct producer_consumer *pc, int item) {
    semaphore_wait(&pc->empty);    // 等待空缓冲区
    semaphore_wait(&pc->mutex);    // 获取互斥锁
    
    // 临界区：放入物品
    pc->buffer[pc->in] = item;
    pc->in = (pc->in + 1) % BUFFER_SIZE;
    printf("Produced item %d\n", item);
    
    semaphore_signal(&pc->mutex);  // 释放互斥锁
    semaphore_signal(&pc->full);   // 增加满缓冲区计数
}

int consumer(struct producer_consumer *pc) {
    semaphore_wait(&pc->full);     // 等待满缓冲区
    semaphore_wait(&pc->mutex);    // 获取互斥锁
    
    // 临界区：取出物品
    int item = pc->buffer[pc->out];
    pc->out = (pc->out + 1) % BUFFER_SIZE;
    printf("Consumed item %d\n", item);
    
    semaphore_signal(&pc->mutex);  // 释放互斥锁
    semaphore_signal(&pc->empty);  // 增加空缓冲区计数
    
    return item;
}
```

#### 6.2.2 互斥锁（Mutex）

**互斥锁**：二进制信号量，保证互斥访问共享资源。

```c
struct mutex {
    atomic_t locked;              // 锁状态（0=未锁定，1=已锁定）
    struct list_head wait_list;   // 等待队列
    struct process *owner;        // 锁的拥有者
    struct spinlock wait_lock;    // 保护等待队列的自旋锁
};

// 初始化互斥锁
void mutex_init(struct mutex *mutex) {
    atomic_set(&mutex->locked, 0);
    INIT_LIST_HEAD(&mutex->wait_list);
    mutex->owner = NULL;
    spin_lock_init(&mutex->wait_lock);
}

// 获取互斥锁
void mutex_lock(struct mutex *mutex) {
    // 尝试原子地获取锁
    if (atomic_cmpxchg(&mutex->locked, 0, 1) == 0) {
        mutex->owner = current_process();
        return;  // 成功获取锁
    }
    
    // 锁已被占用，加入等待队列
    struct wait_entry entry;
    entry.process = current_process();
    
    spin_lock(&mutex->wait_lock);
    list_add_tail(&entry.list, &mutex->wait_list);
    spin_unlock(&mutex->wait_lock);
    
    // 阻塞等待
    while (atomic_cmpxchg(&mutex->locked, 0, 1) != 0) {
        schedule_block();
    }
    
    mutex->owner = current_process();
}

// 释放互斥锁
void mutex_unlock(struct mutex *mutex) {
    if (mutex->owner != current_process()) {
        panic("Mutex unlock by non-owner");
    }
    
    mutex->owner = NULL;
    atomic_set(&mutex->locked, 0);
    
    // 唤醒等待队列中的一个进程
    spin_lock(&mutex->wait_lock);
    if (!list_empty(&mutex->wait_list)) {
        struct wait_entry *entry = list_first_entry(&mutex->wait_list,
                                                   struct wait_entry, list);
        list_del(&entry->list);
        wake_up_process(entry->process);
    }
    spin_unlock(&mutex->wait_lock);
}
```

#### 6.2.3 条件变量（Condition Variable）

**条件变量**：用于线程间的条件同步，必须与互斥锁配合使用。

```c
struct condition_variable {
    struct list_head wait_list;
    struct spinlock wait_lock;
};

void condition_init(struct condition_variable *cond) {
    INIT_LIST_HEAD(&cond->wait_list);
    spin_lock_init(&cond->wait_lock);
}

// 等待条件
void condition_wait(struct condition_variable *cond, struct mutex *mutex) {
    struct wait_entry entry;
    entry.process = current_process();
    
    spin_lock(&cond->wait_lock);
    list_add_tail(&entry.list, &cond->wait_list);
    spin_unlock(&cond->wait_lock);
    
    // 释放互斥锁并阻塞
    mutex_unlock(mutex);
    schedule_block();
    
    // 被唤醒后重新获取互斥锁
    mutex_lock(mutex);
}

// 唤醒一个等待的线程
void condition_signal(struct condition_variable *cond) {
    spin_lock(&cond->wait_lock);
    if (!list_empty(&cond->wait_list)) {
        struct wait_entry *entry = list_first_entry(&cond->wait_list,
                                                   struct wait_entry, list);
        list_del(&entry->list);
        wake_up_process(entry->process);
    }
    spin_unlock(&cond->wait_lock);
}
```

#### 6.2.4 管程（Monitor）

**管程**：高级同步原语，封装共享数据和操作，保证同一时刻只有一个进程在管程内执行。

```c
// 管程的抽象实现
struct monitor {
    struct mutex entry_mutex;           // 管程入口互斥锁
    struct condition_variable entry_cv; // 入口条件变量
    int active_count;                   // 活跃进程数
    int waiting_count;                  // 等待进程数
    
    // 管程内的共享数据
    void *shared_data;
    
    // 管程操作函数指针
    int (*operation1)(struct monitor *mon, void *args);
    int (*operation2)(struct monitor *mon, void *args);
};

// 进入管程
void monitor_enter(struct monitor *mon) {
    mutex_lock(&mon->entry_mutex);
    
    while (mon->active_count > 0) {
        mon->waiting_count++;
        condition_wait(&mon->entry_cv, &mon->entry_mutex);
        mon->waiting_count--;
    }
    
    mon->active_count++;
    mutex_unlock(&mon->entry_mutex);
}

// 退出管程
void monitor_exit(struct monitor *mon) {
    mutex_lock(&mon->entry_mutex);
    
    mon->active_count--;
    
    if (mon->waiting_count > 0) {
        condition_signal(&mon->entry_cv);
    }
    
    mutex_unlock(&mon->entry_mutex);
}
```

### 6.3 进程间通信（IPC）

#### 6.3.1 共享内存（Shared Memory）

**共享内存**：多个进程共享同一块内存区域，是最快的IPC方式。

```c
struct shared_memory_segment {
    key_t key;                    // 共享内存键
    int shmid;                   // 共享内存ID
    size_t size;                 // 内存段大小
    void *addr;                  // 虚拟地址
    int ref_count;               // 引用计数
    
    // 权限和状态
    uid_t owner_uid;
    gid_t owner_gid;
    mode_t permissions;
    time_t create_time;
    time_t attach_time;
    time_t detach_time;
    pid_t creator_pid;
    pid_t last_attach_pid;
    pid_t last_detach_pid;
};

// 创建共享内存
int shmget(key_t key, size_t size, int shmflg) {
    struct shared_memory_segment *shm;
    
    // 查找现有的共享内存段
    if (key != IPC_PRIVATE) {
        shm = find_shared_memory_by_key(key);
        if (shm) {
            if (shmflg & IPC_CREAT && shmflg & IPC_EXCL) {
                return -EEXIST;
            }
            if (size > shm->size) {
                return -EINVAL;
            }
            return shm->shmid;
        }
    }
    
    // 创建新的共享内存段
    if (!(shmflg & IPC_CREAT)) {
        return -ENOENT;
    }
    
    shm = malloc(sizeof(struct shared_memory_segment));
    if (!shm) return -ENOMEM;
    
    shm->key = key;
    shm->shmid = allocate_shmid();
    shm->size = size;
    shm->ref_count = 0;
    shm->owner_uid = current_process()->uid;
    shm->owner_gid = current_process()->gid;
    shm->permissions = shmflg & 0777;
    shm->create_time = time(NULL);
    shm->creator_pid = current_process()->pid;
    
    // 分配物理内存
    shm->addr = allocate_physical_memory(size);
    if (!shm->addr) {
        free(shm);
        return -ENOMEM;
    }
    
    // 添加到全局共享内存表
    add_shared_memory_segment(shm);
    
    return shm->shmid;
}

// 连接共享内存
void *shmat(int shmid, const void *shmaddr, int shmflg) {
    struct shared_memory_segment *shm = find_shared_memory_by_id(shmid);
    if (!shm) return (void *)-EINVAL;
    
    // 检查权限
    if (!check_shm_permissions(shm, shmflg)) {
        return (void *)-EACCES;
    }
    
    void *addr;
    if (shmaddr == NULL) {
        // 系统选择地址
        addr = find_free_virtual_address(current_process(), shm->size);
    } else {
        addr = (void *)shmaddr;
        if (shmflg & SHM_RND) {
            addr = (void *)((uintptr_t)addr & ~(PAGE_SIZE - 1));
        }
    }
    
    // 映射到进程地址空间
    if (map_shared_memory(current_process(), addr, shm) != 0) {
        return (void *)-ENOMEM;
    }
    
    shm->ref_count++;
    shm->attach_time = time(NULL);
    shm->last_attach_pid = current_process()->pid;
    
    return addr;
}

// 分离共享内存
int shmdt(const void *shmaddr) {
    struct shared_memory_segment *shm = find_shared_memory_by_addr(current_process(), shmaddr);
    if (!shm) return -EINVAL;
    
    // 从进程地址空间解除映射
    if (unmap_shared_memory(current_process(), shmaddr) != 0) {
        return -EINVAL;
    }
    
    shm->ref_count--;
    shm->detach_time = time(NULL);
    shm->last_detach_pid = current_process()->pid;
    
    // 如果引用计数为0且标记为删除，则释放内存
    if (shm->ref_count == 0 && shm->marked_for_deletion) {
        free_shared_memory_segment(shm);
    }
    
    return 0;
}
```

#### 6.3.2 消息传递（Message Passing）

**消息队列**：消息的链表，存储在内核中，支持按类型发送和接收消息。

```c
struct message {
    long msg_type;              // 消息类型
    size_t msg_size;           // 消息大小
    char *msg_data;            // 消息数据
    struct list_head list;     // 链表节点
};

struct message_queue {
    int msqid;                 // 消息队列ID
    key_t key;                 // 键值
    
    struct list_head msg_list; // 消息链表
    size_t msg_count;          // 消息数量
    size_t total_bytes;        // 总字节数
    
    size_t max_bytes;          // 最大字节数限制
    size_t max_messages;       // 最大消息数限制
    
    struct mutex queue_mutex;
    struct condition_variable not_full;
    struct condition_variable not_empty;
    
    // 权限和统计信息
    uid_t owner_uid;
    gid_t owner_gid;
    mode_t permissions;
    time_t last_send_time;
    time_t last_recv_time;
    pid_t last_send_pid;
    pid_t last_recv_pid;
};

// 发送消息
int msgsnd(int msqid, const void *msgp, size_t msgsz, int msgflg) {
    struct message_queue *mq = find_message_queue(msqid);
    if (!mq) return -EINVAL;
    
    struct message *msg = malloc(sizeof(struct message));
    if (!msg) return -ENOMEM;
    
    // 复制消息内容
    msg->msg_type = ((struct msgbuf *)msgp)->mtype;
    msg->msg_size = msgsz;
    msg->msg_data = malloc(msgsz);
    memcpy(msg->msg_data, ((struct msgbuf *)msgp)->mtext, msgsz);
    
    mutex_lock(&mq->queue_mutex);
    
    // 检查队列是否已满
    while (mq->msg_count >= mq->max_messages || 
           mq->total_bytes + msgsz > mq->max_bytes) {
        if (msgflg & IPC_NOWAIT) {
            mutex_unlock(&mq->queue_mutex);
            free(msg->msg_data);
            free(msg);
            return -EAGAIN;
        }
        condition_wait(&mq->not_full, &mq->queue_mutex);
    }
    
    // 添加消息到队列
    list_add_tail(&msg->list, &mq->msg_list);
    mq->msg_count++;
    mq->total_bytes += msgsz;
    mq->last_send_time = time(NULL);
    mq->last_send_pid = current_process()->pid;
    
    // 通知接收者
    condition_signal(&mq->not_empty);
    mutex_unlock(&mq->queue_mutex);
    
    return 0;
}
```

#### 6.3.3 管道（Pipe）

**匿名管道**：半双工通信，只能在具有亲缘关系的进程间使用。

```c
struct pipe {
    char buffer[PIPE_BUFFER_SIZE];
    int read_pos;
    int write_pos;
    int count;
    
    struct mutex pipe_mutex;
    struct condition_variable not_full;
    struct condition_variable not_empty;
    
    int readers;
    int writers;
};

// 创建管道
int pipe_create(int pipefd[2]) {
    struct pipe *pipe = malloc(sizeof(struct pipe));
    if (!pipe) return -ENOMEM;
    
    pipe->read_pos = pipe->write_pos = pipe->count = 0;
    pipe->readers = pipe->writers = 1;
    
    mutex_init(&pipe->pipe_mutex);
    condition_init(&pipe->not_full);
    condition_init(&pipe->not_empty);
    
    // 创建文件描述符
    pipefd[0] = create_fd(pipe, O_RDONLY);  // 读端
    pipefd[1] = create_fd(pipe, O_WRONLY);  // 写端
    
    return 0;
}

// 向管道写入数据
ssize_t pipe_write(struct pipe *pipe, const void *buf, size_t count) {
    mutex_lock(&pipe->pipe_mutex);
    
    if (pipe->readers == 0) {
        mutex_unlock(&pipe->pipe_mutex);
        return -EPIPE;  // 没有读者，发送SIGPIPE信号
    }
    
    size_t written = 0;
    const char *data = (const char *)buf;
    
    while (written < count) {
        // 等待管道不满
        while (pipe->count == PIPE_BUFFER_SIZE) {
            condition_wait(&pipe->not_full, &pipe->pipe_mutex);
        }
        
        // 写入数据
        size_t to_write = min(count - written, PIPE_BUFFER_SIZE - pipe->count);
        for (size_t i = 0; i < to_write; i++) {
            pipe->buffer[pipe->write_pos] = data[written + i];
            pipe->write_pos = (pipe->write_pos + 1) % PIPE_BUFFER_SIZE;
        }
        
        pipe->count += to_write;
        written += to_write;
        
        // 通知读者
        condition_signal(&pipe->not_empty);
    }
    
    mutex_unlock(&pipe->pipe_mutex);
    return written;
}
```

## 7. 死锁

### 7.1 死锁的概念

**死锁**：两个或多个进程在执行过程中，因争夺资源而造成的一种互相等待的现象，若无外力作用，这些进程都将无法推进。

```c
// 死锁的经典示例：哲学家就餐问题
#define NUM_PHILOSOPHERS 5

struct philosopher {
    int id;
    struct mutex *left_fork;
    struct mutex *right_fork;
};

struct mutex forks[NUM_PHILOSOPHERS];
struct philosopher philosophers[NUM_PHILOSOPHERS];

// 可能导致死锁的实现
void philosopher_eat_deadlock_prone(struct philosopher *phil) {
    printf("Philosopher %d is thinking\n", phil->id);
    
    // 获取左边的叉子
    mutex_lock(phil->left_fork);
    printf("Philosopher %d picked up left fork\n", phil->id);
    
    // 模拟思考时间
    usleep(1000);
    
    // 获取右边的叉子（可能导致死锁）
    mutex_lock(phil->right_fork);
    printf("Philosopher %d picked up right fork\n", phil->id);
    
    // 进餐
    printf("Philosopher %d is eating\n", phil->id);
    usleep(2000);
    
    // 放下叉子
    mutex_unlock(phil->right_fork);
    mutex_unlock(phil->left_fork);
    printf("Philosopher %d finished eating\n", phil->id);
}

// 避免死锁的实现：按序获取资源
void philosopher_eat_deadlock_free(struct philosopher *phil) {
    printf("Philosopher %d is thinking\n", phil->id);
    
    // 按照叉子地址顺序获取，避免死锁
    struct mutex *first_fork = (phil->left_fork < phil->right_fork) ? 
                              phil->left_fork : phil->right_fork;
    struct mutex *second_fork = (phil->left_fork < phil->right_fork) ? 
                               phil->right_fork : phil->left_fork;
    
    mutex_lock(first_fork);
    printf("Philosopher %d picked up first fork\n", phil->id);
    
    mutex_lock(second_fork);
    printf("Philosopher %d picked up second fork\n", phil->id);
    
    // 进餐
    printf("Philosopher %d is eating\n", phil->id);
    usleep(2000);
    
    // 按相反顺序释放
    mutex_unlock(second_fork);
    mutex_unlock(first_fork);
    printf("Philosopher %d finished eating\n", phil->id);
}
```

### 7.2 死锁产生的必要条件

死锁的产生必须同时满足以下四个条件（**Coffman条件**）：

#### 7.2.1 互斥条件（Mutual Exclusion）
资源不能被多个进程同时使用，每个资源要么空闲，要么恰好被一个进程占用。

```c
// 互斥资源的示例
struct exclusive_resource {
    int resource_id;
    struct process *owner;     // 当前拥有者
    struct mutex access_lock;  // 访问锁
    bool is_available;         // 可用状态
};

int acquire_exclusive_resource(struct exclusive_resource *res, struct process *proc) {
    mutex_lock(&res->access_lock);
    
    if (res->is_available) {
        res->owner = proc;
        res->is_available = false;
        mutex_unlock(&res->access_lock);
        return 0;  // 成功获取
    } else {
        mutex_unlock(&res->access_lock);
        return -EBUSY;  // 资源被占用
    }
}
```

#### 7.2.2 请求和保持条件（Hold and Wait）
进程已经获得了至少一个资源，但又申请新的资源，而该资源已被其他进程占有，此时请求进程阻塞，但又对自己已获得的其他资源保持不放。

```c
// 展示请求和保持的场景
struct resource_request {
    struct process *requesting_process;
    struct exclusive_resource *held_resources[MAX_HELD_RESOURCES];
    int held_count;
    struct exclusive_resource *requested_resource;
    struct list_head wait_list;
};

void demonstrate_hold_and_wait() {
    struct process *proc1 = create_process("Process1");
    struct process *proc2 = create_process("Process2");
    
    struct exclusive_resource *res_A = create_resource("ResourceA");
    struct exclusive_resource *res_B = create_resource("ResourceB");
    
    // Process1 获取资源A
    acquire_exclusive_resource(res_A, proc1);
    printf("Process1 acquired ResourceA\n");
    
    // Process2 获取资源B
    acquire_exclusive_resource(res_B, proc2);
    printf("Process2 acquired ResourceB\n");
    
    // Process1 请求资源B（被Process2占有）
    if (acquire_exclusive_resource(res_B, proc1) == -EBUSY) {
        printf("Process1 waiting for ResourceB while holding ResourceA\n");
        // Process1 进入等待状态，但仍持有资源A
    }
    
    // Process2 请求资源A（被Process1占有）
    if (acquire_exclusive_resource(res_A, proc2) == -EBUSY) {
        printf("Process2 waiting for ResourceA while holding ResourceB\n");
        // Process2 进入等待状态，但仍持有资源B
        // 此时形成死锁
    }
}
```

#### 7.2.3 不剥夺条件（No Preemption）
进程已获得的资源，在未使用完之前，不能被剥夺，只能在使用完时由自己释放。

```c
// 不可剥夺资源的管理
struct non_preemptive_resource {
    int resource_id;
    struct process *owner;
    bool can_be_preempted;  // 是否可以被剥夺
    
    // 资源使用状态
    enum resource_state {
        RESOURCE_FREE,
        RESOURCE_ALLOCATED,
        RESOURCE_IN_USE
    } state;
};

// 尝试剥夺资源（在不可剥夺条件下会失败）
int try_preempt_resource(struct non_preemptive_resource *res, 
                        struct process *new_owner) {
    if (!res->can_be_preempted) {
        printf("Cannot preempt resource %d from process %d\n", 
               res->resource_id, res->owner->pid);
        return -EPERM;  // 不允许剥夺
    }
    
    if (res->state == RESOURCE_IN_USE) {
        printf("Resource %d is in use, cannot preempt\n", res->resource_id);
        return -EBUSY;
    }
    
    // 强制剥夺资源
    res->owner = new_owner;
    printf("Resource %d preempted and assigned to process %d\n", 
           res->resource_id, new_owner->pid);
    return 0;
}
```

#### 7.2.4 环路等待条件（Circular Wait）
存在一种进程资源的循环等待链，即进程集合{P0, P1, P2, ..., Pn}中的P0正在等待一个P1占用的资源；P1正在等待P2占用的资源，……，Pn正在等待已被P0占用的资源。

```c
// 资源分配图的表示
struct resource_allocation_graph {
    struct list_head processes;    // 进程节点列表
    struct list_head resources;    // 资源节点列表
    struct list_head edges;        // 边的列表
};

struct graph_edge {
    enum edge_type {
        REQUEST_EDGE,    // 请求边：进程 -> 资源
        ALLOCATION_EDGE  // 分配边：资源 -> 进程
    } type;
    
    void *from;  // 源节点（进程或资源）
    void *to;    // 目标节点（资源或进程）
    struct list_head list;
};

// 检测环路等待
bool detect_circular_wait(struct resource_allocation_graph *graph) {
    // 使用DFS检测图中的环
    struct process_node *proc;
    
    list_for_each_entry(proc, &graph->processes, list) {
        if (dfs_detect_cycle(graph, proc, NULL, 0)) {
            printf("Circular wait detected starting from process %d\n", proc->pid);
            return true;
        }
    }
    
    return false;
}

// 深度优先搜索检测环
bool dfs_detect_cycle(struct resource_allocation_graph *graph, 
                     void *current_node, void *start_node, int depth) {
    if (depth > 0 && current_node == start_node) {
        return true;  // 找到环
    }
    
    if (depth > MAX_CYCLE_DEPTH) {
        return false;  // 防止无限递归
    }
    
    // 遍历从当前节点出发的所有边
    struct graph_edge *edge;
    list_for_each_entry(edge, &graph->edges, list) {
        if (edge->from == current_node) {
            if (start_node == NULL) {
                start_node = current_node;
            }
            
            if (dfs_detect_cycle(graph, edge->to, start_node, depth + 1)) {
                return true;
            }
        }
    }
    
    return false;
}
```

### 7.3 死锁的处理策略

#### 7.3.1 死锁预防（Deadlock Prevention）

通过破坏死锁产生的四个必要条件之一来预防死锁。

**破坏互斥条件**：
```c
// 使用资源池和共享访问来破坏互斥条件
struct shared_resource_pool {
    int max_instances;           // 最大实例数
    int available_instances;     // 可用实例数
    struct semaphore pool_sem;   // 信号量控制访问
    struct mutex pool_mutex;     // 保护池状态
};

void init_shared_pool(struct shared_resource_pool *pool, int max_instances) {
    pool->max_instances = max_instances;
    pool->available_instances = max_instances;
    semaphore_init(&pool->pool_sem, max_instances);
    mutex_init(&pool->pool_mutex);
}

int acquire_from_pool(struct shared_resource_pool *pool) {
    semaphore_wait(&pool->pool_sem);  // 等待可用实例
    
    mutex_lock(&pool->pool_mutex);
    pool->available_instances--;
    printf("Acquired resource, %d instances remaining\n", pool->available_instances);
    mutex_unlock(&pool->pool_mutex);
    
    return 0;
}

void release_to_pool(struct shared_resource_pool *pool) {
    mutex_lock(&pool->pool_mutex);
    pool->available_instances++;
    printf("Released resource, %d instances available\n", pool->available_instances);
    mutex_unlock(&pool->pool_mutex);
    
    semaphore_signal(&pool->pool_sem);
}
```

**破坏请求和保持条件**：
```c
// 一次性申请所有需要的资源
struct resource_bundle {
    struct exclusive_resource **resources;
    int resource_count;
    bool all_acquired;
};

int acquire_all_or_none(struct resource_bundle *bundle, struct process *proc) {
    // 尝试获取所有资源
    for (int i = 0; i < bundle->resource_count; i++) {
        if (acquire_exclusive_resource(bundle->resources[i], proc) != 0) {
            // 获取失败，释放已获取的资源
            for (int j = 0; j < i; j++) {
                release_exclusive_resource(bundle->resources[j], proc);
            }
            return -EAGAIN;
        }
    }
    
    bundle->all_acquired = true;
    printf("Process %d acquired all %d resources\n", proc->pid, bundle->resource_count);
    return 0;
}
```

**破坏不剥夺条件**：
```c
// 实现资源的强制回收机制
struct preemptive_resource {
    int resource_id;
    struct process *owner;
    int priority;  // 资源优先级
    
    // 回调函数：资源被剥夺时调用
    void (*on_preempted)(struct preemptive_resource *res, struct process *old_owner);
};

int preempt_resource_if_needed(struct preemptive_resource *res, 
                              struct process *new_owner) {
    if (res->owner == NULL) {
        res->owner = new_owner;
        return 0;
    }
    
    // 检查是否可以剥夺
    if (new_owner->priority > res->owner->priority) {
        struct process *old_owner = res->owner;
        
        // 通知原拥有者资源被剥夺
        if (res->on_preempted) {
            res->on_preempted(res, old_owner);
        }
        
        // 将资源分配给新进程
        res->owner = new_owner;
        
        printf("Resource %d preempted from process %d (priority %d) to process %d (priority %d)\n",
               res->resource_id, old_owner->pid, old_owner->priority, 
               new_owner->pid, new_owner->priority);
        
        return 0;
    }
    
    return -EBUSY;
}
```

**破坏环路等待条件**：
```c
// 资源排序法：为所有资源分配唯一的序号
#define MAX_RESOURCES 100

struct ordered_resource {
    int order_number;  // 资源序号
    struct exclusive_resource *resource;
};

struct ordered_resource resource_table[MAX_RESOURCES];

// 按序申请资源
int acquire_resources_in_order(int *resource_orders, int count, struct process *proc) {
    // 对请求的资源按序号排序
    qsort(resource_orders, count, sizeof(int), compare_resource_order);
    
    // 按序获取资源
    for (int i = 0; i < count; i++) {
        int order = resource_orders[i];
        struct ordered_resource *ord_res = &resource_table[order];
        
        if (acquire_exclusive_resource(ord_res->resource, proc) != 0) {
            // 获取失败，释放已获取的资源
            for (int j = 0; j < i; j++) {
                int prev_order = resource_orders[j];
                release_exclusive_resource(resource_table[prev_order].resource, proc);
            }
            return -EAGAIN;
        }
        
        printf("Process %d acquired resource with order %d\n", proc->pid, order);
    }
    
    return 0;
}

int compare_resource_order(const void *a, const void *b) {
    return (*(int*)a - *(int*)b);
}
```

#### 7.3.2 死锁避免（Deadlock Avoidance）

**银行家算法**：动态检查资源分配状态，确保系统始终处于安全状态。

```c
#define MAX_PROCESSES 10
#define MAX_RESOURCE_TYPES 5

struct bankers_algorithm {
    int num_processes;
    int num_resource_types;
    
    // 系统资源总量
    int total_resources[MAX_RESOURCE_TYPES];
    
    // 当前可用资源
    int available[MAX_RESOURCE_TYPES];
    
    // 进程的最大需求矩阵
    int max_demand[MAX_PROCESSES][MAX_RESOURCE_TYPES];
    
    // 进程当前分配矩阵
    int allocation[MAX_PROCESSES][MAX_RESOURCE_TYPES];
    
    // 进程还需要的资源矩阵
    int need[MAX_PROCESSES][MAX_RESOURCE_TYPES];
};

// 安全性检查算法
bool is_safe_state(struct bankers_algorithm *ba) {
    int work[MAX_RESOURCE_TYPES];
    bool finish[MAX_PROCESSES];
    int safe_sequence[MAX_PROCESSES];
    int safe_count = 0;
    
    // 初始化工作向量和完成标志
    for (int i = 0; i < ba->num_resource_types; i++) {
        work[i] = ba->available[i];
    }
    
    for (int i = 0; i < ba->num_processes; i++) {
        finish[i] = false;
    }
    
    // 寻找安全序列
    while (safe_count < ba->num_processes) {
        bool found = false;
        
        for (int i = 0; i < ba->num_processes; i++) {
            if (!finish[i]) {
                // 检查进程i的需求是否可以满足
                bool can_allocate = true;
                for (int j = 0; j < ba->num_resource_types; j++) {
                    if (ba->need[i][j] > work[j]) {
                        can_allocate = false;
                        break;
                    }
                }
                
                if (can_allocate) {
                    // 模拟分配资源给进程i
                    for (int j = 0; j < ba->num_resource_types; j++) {
                        work[j] += ba->allocation[i][j];
                    }
                    
                    finish[i] = true;
                    safe_sequence[safe_count++] = i;
                    found = true;
                    
                    printf("Process %d can be safely executed\n", i);
                }
            }
        }
        
        if (!found) {
            printf("No safe sequence found - system is in unsafe state\n");
            return false;
        }
    }
    
    printf("Safe sequence found: ");
    for (int i = 0; i < safe_count; i++) {
        printf("P%d ", safe_sequence[i]);
    }
    printf("\n");
    
    return true;
}
```

#### 7.3.3 死锁检测和解除（Deadlock Detection and Recovery）

**死锁检测算法**：
```c
struct deadlock_detector {
    int num_processes;
    int num_resource_types;
    
    // 当前可用资源
    int available[MAX_RESOURCE_TYPES];
    
    // 当前分配矩阵
    int allocation[MAX_PROCESSES][MAX_RESOURCE_TYPES];
    
    // 请求矩阵
    int request[MAX_PROCESSES][MAX_RESOURCE_TYPES];
    
    // 检测结果
    bool is_deadlocked[MAX_PROCESSES];
    int deadlock_count;
};

// 死锁检测算法
bool detect_deadlock(struct deadlock_detector *dd) {
    int work[MAX_RESOURCE_TYPES];
    bool finish[MAX_PROCESSES];
    
    // 初始化
    for (int i = 0; i < dd->num_resource_types; i++) {
        work[i] = dd->available[i];
    }
    
    for (int i = 0; i < dd->num_processes; i++) {
        finish[i] = false;
        dd->is_deadlocked[i] = false;
        
        // 如果进程没有分配任何资源，标记为完成
        bool has_allocation = false;
        for (int j = 0; j < dd->num_resource_types; j++) {
            if (dd->allocation[i][j] > 0) {
                has_allocation = true;
                break;
            }
        }
        if (!has_allocation) {
            finish[i] = true;
        }
    }
    
    // 寻找可以完成的进程
    bool changed = true;
    while (changed) {
        changed = false;
        
        for (int i = 0; i < dd->num_processes; i++) {
            if (!finish[i]) {
                // 检查进程i的请求是否可以满足
                bool can_satisfy = true;
                for (int j = 0; j < dd->num_resource_types; j++) {
                    if (dd->request[i][j] > work[j]) {
                        can_satisfy = false;
                        break;
                    }
                }
                
                if (can_satisfy) {
                    // 释放进程i的资源
                    for (int j = 0; j < dd->num_resource_types; j++) {
                        work[j] += dd->allocation[i][j];
                    }
                    
                    finish[i] = true;
                    changed = true;
                    printf("Process %d can complete\n", i);
                }
            }
        }
    }
    
    // 检查是否存在死锁
    dd->deadlock_count = 0;
    for (int i = 0; i < dd->num_processes; i++) {
        if (!finish[i]) {
            dd->is_deadlocked[i] = true;
            dd->deadlock_count++;
            printf("Process %d is deadlocked\n", i);
        }
    }
    
    return dd->deadlock_count > 0;
}
```

```c
pid_t linux_fork_process() {
    pid_t pid = fork();
    
    if (pid == -1) {
        perror("fork failed");
        return -1;
    } else if (pid == 0) {
        // 子进程代码
        printf("Child process: PID = %d, PPID = %d\n", 
               getpid(), getppid());
        
        // 执行新程序
        char *args[] = {"/bin/ls", "-l", NULL};
        execv("/bin/ls", args);
        
        // 如果execv失败
        perror("execv failed");
        exit(1);
    } else {
        // 父进程代码
        printf("Parent process: PID = %d, Child PID = %d\n", 
               getpid(), pid);
        
        // 等待子进程结束
        int status;
        wait(&status);
        printf("Child process exited with status %d\n", status);
    }
    
    return pid;
}
```

```c
        perror("lseek failed");
        close(fd);
        return -1;
    }
    
    // 读取数据
    bytes_read = read(fd, buffer, sizeof(buffer) - 1);
    if (bytes_read == -1) {
        perror("read failed");
        close(fd);
        return -1;
    }
    
    buffer[bytes_read] = '\0';
    printf("Read %zd bytes: %s", bytes_read, buffer);

    struct stat file_stat;

        printf("File permissions: %o\n", file_stat.st_mode & 0777);
```
