# 计算机网络基础

## 1. 计算机网络概述

### 1.1 什么是计算机网络
计算机网络是指将地理位置不同的具有独立功能的多台计算机及其外部设备，通过通信线路连接起来，在网络操作系统、网络管理软件及网络通信协议的管理和协调下，实现资源共享和信息传递的计算机系统。

### 1.2 网络的功能
- **资源共享**：硬件、软件、数据共享
- **信息传输**：电子邮件、文件传输、远程登录
- **分布式处理**：将大型任务分解到网络中的多台计算机上处理
- **提高可靠性**：通过网络备份提高系统可靠性
- **负载均衡**：合理分配网络资源

### 1.3 网络的分类

#### 按覆盖范围分类
- **局域网（LAN）**：覆盖范围小，通常在几公里以内
- **城域网（MAN）**：覆盖一个城市范围
- **广域网（WAN）**：覆盖范围大，可跨越国家和大洲
- **个人区域网（PAN）**：个人设备间的短距离网络

#### 按传输技术分类
- **广播式网络**：所有节点共享通信信道
- **点对点网络**：每条线路连接一对节点

## 2. 网络体系结构

### 2.1 OSI七层模型

#### 物理层（Physical Layer）
- **功能**：传输比特流，定义电气、机械、功能特性
- **设备**：集线器、中继器、网线
- **协议**：RS-232、RJ-45

#### 数据链路层（Data Link Layer）
- **功能**：帧的封装、错误检测和纠正、流量控制
- **设备**：交换机、网桥
- **协议**：Ethernet、PPP、HDLC

#### 网络层（Network Layer）
- **功能**：路由选择、逻辑地址（IP地址）
- **设备**：路由器
- **协议**：IP、ICMP、ARP、OSPF、BGP

#### 传输层（Transport Layer）
- **功能**：端到端的可靠传输、流量控制、错误恢复
- **协议**：TCP、UDP

#### 会话层（Session Layer）
- **功能**：建立、管理、终止会话
- **协议**：NetBIOS、RPC

#### 表示层（Presentation Layer）
- **功能**：数据格式转换、加密解密、压缩解压
- **协议**：SSL/TLS、JPEG、MPEG

#### 应用层（Application Layer）
- **功能**：为应用程序提供网络服务
- **协议**：HTTP、HTTPS、FTP、SMTP、DNS

### 2.2 TCP/IP四层模型

#### 网络接口层
- 对应OSI的物理层和数据链路层
- 处理与物理网络的接口

#### 网际层
- 对应OSI的网络层
- 主要协议：IP、ICMP、ARP

#### 传输层
- 对应OSI的传输层
- 主要协议：TCP、UDP

#### 应用层
- 对应OSI的会话层、表示层、应用层
- 主要协议：HTTP、FTP、SMTP、DNS

## 3. 网络协议详解

### 3.1 IP协议（Internet Protocol）

#### IPv4
- **地址长度**：32位，分为4个8位字段
- **地址格式**：点分十进制，如192.168.1.1
- **地址分类**：
  - A类：1.0.0.0 - 126.255.255.255
  - B类：128.0.0.0 - 191.255.255.255
  - C类：192.0.0.0 - 223.255.255.255
  - D类：224.0.0.0 - 239.255.255.255（组播）
  - E类：240.0.0.0 - 255.255.255.255（保留）

#### 私有IP地址
- A类：10.0.0.0 - 10.255.255.255
- B类：172.16.0.0 - 172.31.255.255
- C类：192.168.0.0 - 192.168.255.255

#### IPv6
- **地址长度**：128位
- **地址格式**：冒号分隔的十六进制，如2001:0db8:85a3:0000:0000:8a2e:0370:7334
- **优势**：地址空间大、内置安全性、简化路由

#### 子网掩码和CIDR
- **子网掩码**：区分网络部分和主机部分
- **CIDR**：无类别域间路由，如192.168.1.0/24

### 3.2 TCP协议（Transmission Control Protocol）

#### TCP特点
- **面向连接**：通信前需建立连接
- **可靠传输**：保证数据完整性和顺序
- **流量控制**：防止发送方发送过快
- **拥塞控制**：防止网络拥塞

#### TCP三次握手
```
客户端                    服务器
   |                         |
   |----SYN(seq=x)---------->|
   |                         |
   |<---SYN+ACK(seq=y,ack=x+1)---|
   |                         |
   |----ACK(ack=y+1)------->|
   |                         |
   |      连接建立            |
```

#### TCP四次挥手
```
客户端                    服务器
   |                         |
   |----FIN(seq=u)---------->|
   |                         |
   |<---ACK(ack=u+1)--------|
   |                         |
   |<---FIN(seq=v)----------|
   |                         |
   |----ACK(ack=v+1)------->|
   |                         |
   |      连接关闭            |
```

#### TCP状态转换
- **CLOSED**：初始状态
- **LISTEN**：服务器等待连接
- **SYN_SENT**：客户端发送SYN后
- **SYN_RCVD**：服务器收到SYN后
- **ESTABLISHED**：连接建立
- **FIN_WAIT_1**：主动关闭方发送FIN后
- **FIN_WAIT_2**：收到对方ACK后
- **TIME_WAIT**：收到对方FIN并发送ACK后
- **CLOSE_WAIT**：被动关闭方收到FIN后
- **LAST_ACK**：被动关闭方发送FIN后

### 3.3 UDP协议（User Datagram Protocol）

#### UDP特点
- **无连接**：发送数据前不需建立连接
- **不可靠**：不保证数据到达和顺序
- **开销小**：头部只有8字节
- **支持广播和组播**

#### UDP应用场景
- 实时应用（视频、音频流）
- 简单请求-响应应用（DNS查询）
- 广播和组播应用

### 3.4 HTTP协议（HyperText Transfer Protocol）

#### HTTP特点
- **无状态**：每个请求独立
- **基于TCP**：可靠传输
- **请求-响应模式**
- **支持多种方法**

#### HTTP方法
- **GET**：获取资源
- **POST**：提交数据
- **PUT**：更新资源
- **DELETE**：删除资源
- **HEAD**：获取资源头信息
- **OPTIONS**：获取服务器支持的方法

#### HTTP状态码
- **1xx**：信息性状态码
- **2xx**：成功状态码（200 OK）
- **3xx**：重定向状态码（301 Moved Permanently）
- **4xx**：客户端错误（404 Not Found）
- **5xx**：服务器错误（500 Internal Server Error）

#### HTTP/1.1 vs HTTP/2 vs HTTP/3
- **HTTP/1.1**：持久连接、管道化
- **HTTP/2**：多路复用、服务器推送、头部压缩
- **HTTP/3**：基于QUIC、更快的连接建立

### 3.5 HTTPS协议

#### HTTPS = HTTP + SSL/TLS
- **加密传输**：防止数据被窃听
- **身份认证**：确认服务器身份
- **数据完整性**：防止数据被篡改

#### SSL/TLS握手过程
1. 客户端发送Client Hello
2. 服务器发送Server Hello和证书
3. 客户端验证证书
4. 密钥交换
5. 建立加密连接

### 3.6 DNS协议（Domain Name System）

#### DNS功能
- 将域名解析为IP地址
- 分布式数据库系统
- 层次化命名空间

#### DNS查询过程
1. 客户端查询本地DNS缓存
2. 查询本地DNS服务器
3. 递归查询根域名服务器
4. 查询顶级域名服务器
5. 查询权威域名服务器
6. 返回IP地址

#### DNS记录类型
- **A记录**：域名到IPv4地址
- **AAAA记录**：域名到IPv6地址
- **CNAME记录**：域名别名
- **MX记录**：邮件服务器
- **NS记录**：域名服务器
- **PTR记录**：反向解析

## 4. 网络设备

### 4.1 物理层设备

#### 中继器（Repeater）
- **功能**：放大和重新发送信号
- **工作层次**：物理层
- **应用**：扩展网络传输距离

#### 集线器（Hub）
- **功能**：多端口中继器
- **特点**：共享带宽、半双工、冲突域
- **现状**：基本被交换机取代

### 4.2 数据链路层设备

#### 网桥（Bridge）
- **功能**：连接两个LAN段
- **特点**：学习MAC地址、过滤帧
- **优势**：分割冲突域

#### 交换机（Switch）
- **功能**：多端口网桥
- **特点**：全双工、独享带宽、MAC地址表
- **类型**：
  - 二层交换机：基于MAC地址转发
  - 三层交换机：支持路由功能

### 4.3 网络层设备

#### 路由器（Router）
- **功能**：连接不同网络、路径选择
- **特点**：基于IP地址转发、维护路由表
- **协议**：RIP、OSPF、BGP

#### 三层交换机
- **功能**：结合交换和路由功能
- **优势**：高速转发、VLAN支持

### 4.4 应用层设备

#### 网关（Gateway）
- **功能**：协议转换、应用层代理
- **类型**：应用网关、协议网关

#### 防火墙（Firewall）
- **功能**：网络安全防护
- **类型**：包过滤、状态检测、应用层防火墙

## 5. 网络安全

### 5.1 网络安全威胁

#### 被动攻击
- **窃听**：截获网络通信
- **流量分析**：分析通信模式

#### 主动攻击
- **伪装**：冒充合法用户
- **重放**：重复发送截获的消息
- **消息篡改**：修改传输中的数据
- **拒绝服务**：使系统无法正常服务

### 5.2 网络安全技术

#### 加密技术
- **对称加密**：AES、DES
- **非对称加密**：RSA、ECC
- **哈希函数**：MD5、SHA-256

#### 身份认证
- **密码认证**：用户名密码
- **数字证书**：PKI体系
- **生物特征**：指纹、虹膜

#### 访问控制
- **自主访问控制（DAC）**
- **强制访问控制（MAC）**
- **基于角色的访问控制（RBAC）**

### 5.3 常见网络攻击

#### DDoS攻击
- **原理**：大量请求使服务器过载
- **类型**：流量型、协议型、应用型
- **防护**：流量清洗、CDN、负载均衡

#### 中间人攻击
- **原理**：攻击者插入通信双方之间
- **防护**：使用HTTPS、证书验证

#### SQL注入
- **原理**：在输入中插入恶意SQL代码
- **防护**：参数化查询、输入验证

#### XSS攻击
- **原理**：在网页中插入恶意脚本
- **防护**：输入过滤、输出编码

## 6. 无线网络

### 6.1 无线网络标准

#### IEEE 802.11系列
- **802.11a**：5GHz，54Mbps
- **802.11b**：2.4GHz，11Mbps
- **802.11g**：2.4GHz，54Mbps
- **802.11n**：2.4/5GHz，600Mbps
- **802.11ac**：5GHz，6.93Gbps
- **802.11ax（Wi-Fi 6）**：2.4/5GHz，9.6Gbps

### 6.2 无线网络安全

#### 加密协议
- **WEP**：已过时，安全性差
- **WPA**：改进的安全协议
- **WPA2**：基于AES加密
- **WPA3**：最新标准，更强安全性

#### 认证方式
- **开放系统**：无认证
- **共享密钥**：预共享密钥
- **802.1X**：企业级认证

### 6.3 移动网络

#### 蜂窝网络演进
- **1G**：模拟语音
- **2G**：数字语音（GSM、CDMA）
- **3G**：数据服务（UMTS、CDMA2000）
- **4G**：高速数据（LTE）
- **5G**：超高速、低延迟、大连接

## 7. 网络性能与优化

### 7.1 网络性能指标

#### 带宽（Bandwidth）
- **定义**：网络的最大传输能力
- **单位**：bps（比特每秒）

#### 延迟（Latency）
- **定义**：数据传输的时间延迟
- **组成**：传播延迟、传输延迟、处理延迟、排队延迟

#### 吞吐量（Throughput）
- **定义**：实际的数据传输速率
- **影响因素**：网络拥塞、协议开销

#### 丢包率（Packet Loss Rate）
- **定义**：丢失数据包的比例
- **原因**：网络拥塞、设备故障

### 7.2 网络优化技术

#### 流量控制
- **滑动窗口**：控制发送速率
- **令牌桶**：平滑突发流量

#### 拥塞控制
- **慢启动**：逐渐增加发送速率
- **拥塞避免**：线性增长
- **快重传**：快速检测丢包
- **快恢复**：快速恢复发送速率

#### QoS（Quality of Service）
- **流量分类**：区分不同类型流量
- **队列管理**：优先级调度
- **流量整形**：控制流量特性

## 9. 网络故障排除

### 9.1 网络诊断工具

#### ping
- **功能**：测试网络连通性
- **原理**：ICMP Echo请求/响应

#### traceroute/tracert
- **功能**：跟踪数据包路径
- **原理**：TTL递增

#### netstat
- **功能**：显示网络连接状态
- **参数**：-a（所有连接）、-n（数字格式）

#### nslookup/dig
- **功能**：DNS查询工具
- **用途**：域名解析测试

#### wireshark
- **功能**：网络协议分析器
- **用途**：数据包捕获和分析

### 9.2 常见网络问题

#### 连接问题
- **物理连接**：检查网线、端口
- **配置问题**：IP地址、子网掩码、网关
- **DNS问题**：域名解析失败

#### 性能问题
- **带宽不足**：网络拥塞
- **延迟过高**：路由问题
- **丢包严重**：设备故障

## 10. 新兴网络技术

### 10.1 软件定义网络（SDN）
- **概念**：控制平面与数据平面分离
- **优势**：集中控制、灵活配置
- **协议**：OpenFlow

### 10.2 网络功能虚拟化（NFV）
- **概念**：网络功能软件化
- **优势**：降低成本、提高灵活性
- **应用**：虚拟防火墙、负载均衡器

### 10.3 边缘计算
- **概念**：在网络边缘处理数据
- **优势**：降低延迟、减少带宽消耗
- **应用**：IoT、自动驾驶

### 10.4 物联网（IoT）
- **特点**：大量设备连接、低功耗
- **协议**：MQTT、CoAP、LoRaWAN
- **挑战**：安全性、可扩展性

## 总结

计算机网络是现代信息社会的基础设施，理解网络原理和协议对于IT从业者至关重要。从OSI模型到TCP/IP协议栈，从有线网络到无线网络，从网络安全到性能优化，每个方面都有其独特的技术特点和应用场景。

随着云计算、物联网、5G等新技术的发展，网络技术也在不断演进。掌握网络基础知识，跟上技术发展趋势，是每个技术人员必备的技能。在实际工作中，要根据具体需求选择合适的网络技术和解决方案，确保网络的可靠性、安全性和高性能。
