# 对比排课吸顶功能 - 最终正确方案

## 问题回顾

### 用户真实需求
在对比排课页面中：
1. ✅ **外部滚动**：可以上下滚动查看所有学员/老师
2. ✅ **表头吸顶**：滚动时，当前可见学员的头部和周几表头保持在顶部
3. ✅ **自动切换**：滚动到下一个学员时，新的表头自动吸顶

### 之前的错误尝试

**第一次尝试（失败）：**
- sticky 元素跨越了组件边界
- 找不到正确的滚动容器

**第二次尝试（不符合需求）：**
- 让每个课表内部独立滚动
- 虽然 sticky 能工作，但改变了交互方式
- 用户无法在外部滚动查看所有学员

## 最终正确方案

### 核心原理

**关键发现：** `position: sticky` 可以跨越**没有 overflow 限制的中间层**，直到找到滚动容器！

```
滚动容器: .section-content (overflow-y: auto)
    ↓ 滚动在这里发生
  .student-item-wrapper (无 overflow ✅)
    ↓ 不阻挡 sticky
  .student-item (无 overflow ✅)
    ↓ 不阻挡 sticky
  .timetable-container (无 overflow ✅)
    ↓ 不阻挡 sticky
  TimeCanlendarView 组件
    ├─ .student-header (sticky, top: 0) ✅ 成功找到滚动容器！
    └─ .schedule-container
        └─ .header (sticky, top: 57px) ✅ 成功找到滚动容器！
```

### 实现要点

#### 1. 移除所有 overflow 限制

**在 compareCanlendar.vue 中：**
```scss
.student-item-wrapper,
.teacher-item-wrapper {
    margin-bottom: 8px;
    // ❌ 不要设置 overflow: hidden
    // ❌ 不要设置固定高度
}

.student-item,
.teacher-item {
    background-color: #fff;
    // ❌ 不要设置 overflow
    // ❌ 不要设置固定高度
    // ✅ 让内容自然撑开
}
```

#### 2. 保持外部滚动容器

```scss
.section-content {
    flex: 1;
    overflow-y: auto; // ✅ 滚动容器
    min-height: 0;
}
```

#### 3. 设置 sticky 定位（在 TimeCanlendarView.vue）

```scss
.canlendar-container.is-compare {
  // 学员/老师信息头部
  .student-header,
  .teacher-header {
    position: sticky;
    top: 0; // 吸在滚动容器顶部
    z-index: 100;
    background-color: #fafafa;
  }

  // 周几表头
  .schedule-container .header {
    position: sticky;
    top: 57px; // 在学员/老师头部下方
    z-index: 99;
    background-color: #f5f7fa;
  }
}
```

### 为什么现在可以工作？

**Sticky 的查找规则：**
1. 从 sticky 元素开始，向上查找祖先元素
2. 跳过所有**没有创建滚动上下文**的元素（overflow: visible）
3. 找到第一个有滚动能力的祖先（overflow: auto/scroll）
4. 将该祖先作为参考容器

**我们的结构满足条件：**
- ✅ 中间层都没有 overflow 限制
- ✅ sticky 元素可以"看到" .section-content
- ✅ 吸顶效果正常工作

## 常见问题

### Q: 为什么不需要设置固定高度？
**A:** 因为滚动是在外部容器 `.section-content` 中发生的，每个课表的内容自然撑开即可。

### Q: 中间层可以有其他样式吗？
**A:** 可以！只要不设置 `overflow: hidden/auto/scroll` 就不会阻挡 sticky。

### Q: 如果需要限制单个课表高度怎么办？
**A:** 不建议限制。如果必须限制，考虑使用 `max-height` 而不是 `height`，并且不要设置 `overflow`。

### Q: 性能如何？
**A:** 非常好！sticky 是浏览器原生实现，比 JavaScript 方案性能高得多。

## 总结

这个方案通过**移除中间层的 overflow 限制**，让 sticky 元素可以正确找到外部滚动容器，从而实现了完美的吸顶效果。

### 核心要点
1. ✅ 滚动容器：`.section-content` (overflow-y: auto)
2. ✅ 中间层：无 overflow 限制
3. ✅ Sticky 元素：在组件内部正常设置
4. ✅ 效果：表头自动吸顶，自动切换

这就是**最简单、最优雅**的实现方案！🎉
