# 列表字段显示设置功能开发文档

## 一、功能概述

列表字段显示设置功能允许用户自定义表格列的显示/隐藏和排序，配置会保存到服务器，实现个性化定制。该功能已在排课列表（`arrangeTableList`）中实现，可作为其他列表页面的参考实现。

## 二、核心组件

### 1. 组件位置
- **组件文件**: `src/components/popup/setTableColumns.vue`
- **使用示例**: `src/pages/scheduleManage/arrangeTableList/arrangeTableList.vue`

### 2. 组件接口定义

#### IColumn 接口
```typescript
interface IColumn {
    Key: string,        // 字段唯一标识（用于匹配表格列）
    Label: string,      // 字段显示名称（中文）
    Visible: number,    // 是否显示：1=显示，0=隐藏
    Disabled?: boolean, // 是否禁用（禁用字段不可修改，始终显示）
    ColumnKey: string,  // 后端字段名（用于API查询和导出，支持逗号分隔多个字段）
    Tips?: string,      // 提示文案（支持HTML格式）
}
```

## 三、功能实现流程

### 1. 初始化列配置

#### 1.1 定义默认列配置（allColumns）
在页面组件中定义完整的列配置数组，包含所有可配置的列：

```typescript
const allColumns = [

        Label: '上课班级/学员',

    },
    {
        Key: 'ShiftName',
        Label: '上课课程',
        Visible: 1,

    },
    {
        Key: 'CampusName',
        Label: '上课校区',
        Visible: 1,
        ColumnKey: 'CampusName'
    },
    {
        Key: 'StartTime',
        Label: '上课时间',
        Visible: 1,
        ColumnKey: 'StartTime,EndTime'  // 多个字段用逗号分隔
    },
    {
        Key: 'CostNum',

        Tips: '取本节课每个学员扣费数量的合计'  // 提示信息
    },
    // ... 更多列配置

```

**注意事项**：
- `Key`: 必须与表格模板中的 `v-if="item.Key=='xxx'"` 完全匹配
- `ColumnKey`: 可以是单个字段或多个字段（逗号分隔），用于后端查询和导出
- `Visible`: 默认显示状态（1=显示，0=隐藏）
- `Disabled`: 设置为 `true` 的列不可修改，始终显示，通常用于关键列（如主键、操作列等）
- `Tips`: 可选，支持HTML格式，会在设置弹窗中显示提示图标

#### 1.2 获取用户自定义列配置
页面加载时，从服务器获取用户保存的列配置：

```typescript
const userDefinedColumns = ref([] as any)

// 在搜索或初始化时调用
function search(searchCondition: any) {
    Object.assign(condition, searchCondition)
    
    // 如果已初始化，直接使用
    if (userDefinedColumns.value.length > 0) {
        condition.ExportColumn = userDefinedColumns.value
            .map((item: any) => item.ColumnKey)
            .join(',')
            .split(',')
        doSearch()
    } else {
        // 首次加载，获取用户配置
        loading.value = true
        getUserColumns({
            moduleId: 'scheduleManage_table'  // 唯一标识，建议格式：页面名_table
        }).then((res: any) => {
            let data = res.Data
            if (data.AllColumns) {
                let obj = JSON.parse(data.AllColumns)
                // 使用列同步工具函数处理列的同步
                userDefinedColumns.value = syncUserColumns(obj, allColumns)
            } else {
                // 如果没有保存的配置，使用默认配置
                userDefinedColumns.value = cloneDeep(allColumns)
            }
            doSearch()
        }).catch(() => {
            loading.value = false
        })
    }
}
```

**关键点**：
- 使用 `syncUserColumns` 工具函数同步列配置（处理新增/删除的列）
- 如果用户没有保存过配置，使用默认配置
- 初始化完成后调用 `doSearch()` 查询数据

#### 1.3 计算可见列
使用 `computed` 计算实际显示的列：

```typescript
const definedColumns = computed(() => {
    // 过滤出可见且非禁用的列
    let columns = userDefinedColumns.value.filter(
        (item: any) => item.Visible && !item.Disabled
    )
    
    // 可以根据业务逻辑进一步过滤
    // 例如：某些条件下隐藏特定列
    if (!ShowClassProgressBar.value) {
        columns = columns.filter(
            (item: any) => item.Key !== 'ClassPlanNum_ClassPlanCount'
        )
    }
    
    return columns
})
```

**注意事项**：
- `definedColumns` 只包含可见且非禁用的列
- 禁用列（`Disabled: true`）不在 `definedColumns` 中，需要单独在模板中渲染

### 4. 保存列配置

`setTableColumns` 组件内部会调用 `postUserColumns` API 保存配置：

```typescript
// 组件内部实现
function submit() {
    if (moduleId.value == '') {
        ElMessage.error('moduleId不能为空')
        return
    }
    
    // 根据拖拽排序后的顺序重新构建columns数组
    const sortedColumns: IColumn[] = []
    
    // 先添加disabled的字段（保持原有顺序）
    columns.value.forEach(item => {
        if (item.Disabled) {
            sortedColumns.push(item)
        }
    })
    
    // 再添加拖拽排序后的字段
    dragableColumns.value.forEach(dragItem => {
        const originalItem = columns.value.find(col => col.Key === dragItem.Key)
        if (originalItem) {
            sortedColumns.push({
                ...originalItem,
                Visible: dragItem.Visible  // 使用拖拽后的可见性状态
            })
        }
    })
    
    let allColumns = sortedColumns.map((item: any) => {
        return {
            Key: item.Key,
            Visible: item.Visible
        }
    })
    
    postUserColumns({
        moduleId: moduleId.value,
        allColumns: JSON.stringify(allColumns),
        hideColumns: ''  // 新版不再使用
    }).then((res: any) => {
        _resolve({
            data: sortedColumns  // 返回排序后的结果
        })
        drawer.value = false
    })
}
```

**保存逻辑**：
- 禁用列始终在最前面，保持原有顺序
- 非禁用列按拖拽后的顺序排列
- 只保存 `Key` 和 `Visible` 字段

### 5. 查询和导出时使用列配置

#### 5.1 查询时传递列配置
```typescript
function funcQuery() {
    let params: any = {}
    let columns = userDefinedColumns.value.filter((item: any) => item.Visible)
    
    Object.assign(params, condition, sort, page.value, {
        ExportColumn: columns.map((item: any) => item.ColumnKey).join(',').split(',')
    })
    
    queryCourseNew(params).then((res: any) => {
        list.value = res.Data.List || []
        assignPage(page.value, res.Data)
    })
}
```

**注意事项**：
- `ExportColumn` 需要是数组格式
- 如果 `ColumnKey` 是多个字段（逗号分隔），需要先 `join(',')` 再 `split(',')`

## 四、setTableColumns 组件详解

### 1. 组件功能
- 显示所有可配置的列（包括禁用列）
- 支持勾选/取消勾选控制列的显示/隐藏
- 支持拖拽排序（使用 `vue-draggable-plus`）
- 支持"全部"快捷操作
- 禁用字段显示但不可操作

### 2. 组件 Props（通过 open 方法传入）
```typescript
{
    columns: IColumn[],  // 列配置数组
    moduleId: string     // 模块唯一标识
}
```

### 3. 组件交互

#### 3.1 全部勾选/取消
- 点击"全部"复选框，可批量显示/隐藏所有非禁用列
- 禁用列不受影响
- 计算逻辑：当所有列都显示时为选中，否则为未选中

#### 3.2 单个列操作
- 点击列项可切换显示/隐藏状态
- 禁用列不可点击（`cursor: not-allowed`）
- 点击时会同步更新 `columns` 和 `dragableColumns` 的状态

#### 3.3 拖拽排序
- 鼠标悬停时显示拖拽图标（`visibility: hidden` -> `visible`）
- 拖拽可调整列的顺序
- 禁用列不参与拖拽，始终在最前面
- 使用 `vue-draggable-plus` 的 `useDraggable` 实现

#### 3.4 提示信息
- 如果列配置中有 `Tips`，会显示提示图标
- 点击图标查看详细说明（支持HTML格式）
- 使用 `transToConfigDescript` 处理配置描述

### 4. 组件样式

#### 4.1 布局
- 抽屉从右侧滑出，宽度 260px
- 顶部显示"全部"复选框
- 中间是可滚动区域（`el-scrollbar`），显示所有列
- 底部是"取消"和"保存"按钮

#### 4.2 样式特点

## 六、工具函数

### 1. syncUserColumns
位置：`src/utils/column-sync.ts`

**功能**：同步用户自定义列与默认列，处理新增列和删除列的情况

**使用场景**：
- 当系统新增了列时，自动添加到用户配置中
- 当系统删除了列时，自动从用户配置中移除
- 保持用户自定义的显示状态和排序

**实现逻辑**：
1. 按照用户配置的顺序处理列，只保留在默认列中存在的列
2. 合并默认配置和用户配置（用户配置优先）
3. 添加用户没有配置但在默认列中存在的列（新增的列）

**示例**：
```typescript
import { syncUserColumns } from '@/utils'

const userColumns = [
    { Key: 'CampusName', Visible: 1 },
    { Key: 'StartTime', Visible: 0 }
]

const defaultColumns = [
    { Key: 'CampusName', Label: '上课校区', Visible: 1, ColumnKey: 'CampusName' },
    { Key: 'StartTime', Label: '上课时间', Visible: 1, ColumnKey: 'StartTime' },
    { Key: 'NewColumn', Label: '新列', Visible: 1, ColumnKey: 'NewColumn' }  // 新增的列
]

const syncedColumns = syncUserColumns(userColumns, defaultColumns)
// 结果：
// [
//     { Key: 'CampusName', Label: '上课校区', Visible: 1, ColumnKey: 'CampusName' },  // 保持用户配置
//     { Key: 'StartTime', Label: '上课时间', Visible: 0, ColumnKey: 'StartTime' },     // 保持用户配置
//     { Key: 'NewColumn', Label: '新列', Visible: 1, ColumnKey: 'NewColumn' }          // 新增的列使用默认配置
// ]
```

## 七、开发注意事项

### 1. moduleId 命名规范
- 格式：`页面名_table`
- 示例：`scheduleManage_table`、`examManage_table`
- 必须保证唯一性，用于区分不同页面的列配置
- 在 `getUserColumns` 和 `postUserColumns` 中必须一致

### 2. Key 命名规范
- 必须与表格模板中的 `v-if="item.Key=='xxx'"` 完全匹配
- 建议使用驼峰命名，如 `CampusName`、`StartTime`
- 区分大小写

### 3. ColumnKey 规范
- 单个字段：`'CampusName'`
- 多个字段（逗号分隔）：`'StartTime,EndTime'`
- 用于后端查询和导出，必须与后端接口字段名一致
- 在传递给后端时，需要先 `join(',')` 再 `split(',')` 转换为数组

### 4. 禁用列的使用
- 设置为 `Disabled: true` 的列始终显示，不可修改
- 通常用于关键列（如主键、操作列等）
- 禁用列不参与拖拽排序，始终在最前面
- 禁用列不在 `definedColumns` 中，需要在模板中单独渲染

### 5. 列配置的初始化时机
- 建议在首次查询数据前初始化
- 如果 `userDefinedColumns.length > 0`，说明已初始化，直接使用
- 如果为空，需要先调用 `getUserColumns` 获取配置
- 初始化完成后调用查询方法

### 6. 列配置的同步
- 使用 `syncUserColumns` 工具函数处理列配置的同步
- 确保新增列和删除列时，用户配置能正确更新
- 新增列使用默认配置，删除列自动移除

### 7. 样式前缀
- 项目使用 `wtwo-` 前缀（非 `el-`）
- 所有自定义样式类名应使用 `wtwo-` 前缀
- Element Plus 组件会自动转换为 `wtwo-` 前缀

### 8. 自动导入
- 项目使用 `unplugin-auto-import` 自动导入常用 API
- `ElMessage`、`ElMessageBox` 等无需手动导入
- 类型定义在 `auto-imports.d.ts` 中

### 9. 表格列渲染顺序
- 固定列（选择框、禁用列）在最前面
- 动态列（`definedColumns`）在中间
- 固定列（操作列）在最后面
- 禁用列虽然不在 `definedColumns` 中，但需要在模板中单独渲染

### 10. 拖拽排序
- 使用 `vue-draggable-plus` 的 `useDraggable` 实现
- 需要在抽屉打开后重新初始化拖拽功能
- 拖拽图标默认隐藏，悬停时显示

```vue
import { cloneDeep } from 'lodash'
import setTableColumns from '@/components/popup/setTableColumns.vue'
import { getUserColumns } from '@/api'
import { syncUserColumns } from '@/utils'

// 默认列配置
const allColumns = [
    { Key: 'CampusName', Label: '校区', Visible: 1, ColumnKey: 'CampusName' },
    // ... 更多列
]

// 用户自定义列配置
const userDefinedColumns = ref([] as any)

// 计算可见列
const definedColumns = computed(() => {
    return userDefinedColumns.value.filter(
        (item: any) => item.Visible && !item.Disabled
    )
})

// 初始化列配置
function initColumns() {
    if (userDefinedColumns.value.length > 0) {
        return
    }
    
    getUserColumns({
        moduleId: 'examManage_table'
    }).then((res: any) => {
        let data = res.Data
        if (data.AllColumns) {
            let obj = JSON.parse(data.AllColumns)
            userDefinedColumns.value = syncUserColumns(obj, allColumns)
        } else {
            userDefinedColumns.value = cloneDeep(allColumns)
        }
        // 初始化完成后查询数据
        loadData()
    })
}

// 打开设置弹窗
const setTableColumnsRef = ref<InstanceType<typeof setTableColumns> | null>(null)
function setTableColumnsOpen() {
    let columns = cloneDeep(userDefinedColumns.value)
    setTableColumnsRef.value?.open({
        columns: columns,
        moduleId: 'examManage_table'
    }).then((res: any) => {
        userDefinedColumns.value = res.data ? cloneDeep(res.data) : []
        loadData()
    })
}

// 查询数据
function loadData() {
    let columns = userDefinedColumns.value.filter((item: any) => item.Visible)
    let params = {
        ExportColumn: columns.map((item: any) => item.ColumnKey).join(',').split(',')
        // ... 其他查询参数
    }
    // 调用查询API
}

onMounted(() => {
    initColumns()
})
</script>
```

## 九、常见问题

### 1. 列配置不生效
- **检查 `moduleId`**：确保 `getUserColumns` 和 `postUserColumns` 中的 `moduleId` 一致
- **检查 `Key`**：确保 `Key` 与表格模板中的 `v-if` 条件完全匹配（区分大小写）
- **检查初始化**：确保 `userDefinedColumns` 已正确初始化
- **检查过滤条件**：确保 `definedColumns` 的过滤条件正确

### 2. 新增列不显示
- 确保在 `allColumns` 中添加了新列的配置
- 使用 `syncUserColumns` 同步列配置
- 检查表格模板中是否有对应的 `v-if` 判断
- 确保新列的 `Key` 与模板中的判断条件匹配

### 3. 拖拽排序不生效
- 确保 `vue-draggable-plus` 已正确安装
- 检查 `dragableRef` 是否正确绑定
- 确保在抽屉打开后重新初始化拖拽功能（`nextTick` 中调用 `initDraggable`）
- 检查 `dragableColumns` 是否正确过滤（只包含非禁用列）

### 4. 导出列不正确
- 检查 `ColumnKey` 是否正确设置
- 确保导出时传递了正确的 `ExportColumn` 参数（数组格式）
- 如果 `ColumnKey` 是多个字段，需要先 `join(',')` 再 `split(',')`
- 检查后端接口是否支持动态列导出

### 5. 禁用列显示问题
- 禁用列不在 `definedColumns` 中，需要在模板中单独渲染
- 确保禁用列的 `Disabled: true` 设置正确
- 禁用列在设置弹窗中显示在最前面，但不可操作

### 6. 提示信息不显示
- 确保列配置中有 `Tips` 字段
- 在设置弹窗中使用 `transToConfigDescript` 处理提示信息
- 在表格头部使用 `v-html` 渲染提示信息（支持HTML格式）

## 十二、考试管理特殊实现

### 1. 概述

考试管理模块使用了专用的列设置组件 `setExamTableColumns.vue`，与排课列表的公共组件 `setTableColumns.vue` 相比，有以下特殊需求：

- **原始顺序模式**：保持列的原始顺序，禁用列可以拖动但不可编辑
- **分组顺序模式**：与排课列表相同，禁用列在前，只有非禁用列可以拖动

### 2. 组件位置

- **考试管理专用组件**：`src/pages/examManage/popup/setExamTableColumns.vue`
- **公共组件（排课列表）**：`src/components/popup/setTableColumns.vue`
- **使用示例**：`src/pages/examManage/examManageList/examManageList.vue`

### 3. 核心区别

#### 3.1 显示模式

**排课列表（公共组件）**：
- 固定为分组模式：禁用列在前，可拖拽列在后
- 禁用列不能拖动，也不能编辑

**考试管理（专用组件）**：
- 支持两种模式：
  - `original`（原始顺序）：所有列按原始顺序显示，所有列都可以拖动
  - `grouped`（分组顺序）：与排课列表相同，禁用列在前，只有非禁用列可以拖动

#### 3.2 禁用列的交互

**排课列表**：
- 禁用列：不能拖动，不能编辑（checkbox disabled，无拖拽图标，`cursor: not-allowed`）

**考试管理（原始顺序模式）**：
- 禁用列：可以拖动，不能编辑（checkbox disabled，有拖拽图标，`cursor: grab`）

#### 3.3 数据结构

**排课列表**：
```typescript
const columns = ref<IColumn[]>([])
const dragableColumns = ref<IColumn[]>([])  // 只包含非禁用列
```

**考试管理**：
```typescript
const columns = ref<IColumn[]>([])
const dragableColumns = ref<IColumn[]>([])  // 只包含非禁用列
const allColumnsForDrag = ref<IColumn[]>([])  // 原始顺序模式下，包含所有列
const orderMode = ref<'original' | 'grouped'>('original')  // 顺序模式
```

#### 3.4 拖拽功能实现

**排课列表**：
```typescript
// 只绑定非禁用列
useDraggable(dragableRef, dragableColumns, {
    animation: 150,
    onUpdate() {
        // 同步更新columns中对应项的可见性状态
        dragableColumns.value.forEach(dragItem => {
            const originalItem = columns.value.find(col => col.Key === dragItem.Key)
            if (originalItem) {
                originalItem.Visible = dragItem.Visible
            }
        })
    }
})
```

**考试管理**：
```typescript
// 根据模式选择不同的数据源
if (orderMode.value === 'original') {
    // 原始顺序模式：绑定所有列（包括禁用列）
    useDraggable(dragableRef, allColumnsForDrag, {
        animation: 150,
        onUpdate() {
            // 拖拽更新后，同步更新columns的顺序
            columns.value = [...allColumnsForDrag.value]
            // 同步更新dragableColumns
            dragableColumns.value = columns.value.filter((item:any)=>!item.Disabled)
        }
    })
} else {
    // 分组顺序模式：只绑定非禁用列
    useDraggable(dragableRef, dragableColumns, {
        animation: 150,
        onUpdate() {
            // 同步更新columns中对应项的可见性状态
            dragableColumns.value.forEach(dragItem => {
                const originalItem = columns.value.find(col => col.Key === dragItem.Key)
                if (originalItem) {
                    originalItem.Visible = dragItem.Visible
                }
            })
        }
    })
}
```

#### 3.5 保存逻辑

**排课列表**：
```typescript
// 先添加禁用列（保持原有顺序），再添加拖拽后的非禁用列
columns.value.forEach(item => {
    if (item.Disabled) {
        sortedColumns.push(item)
    }
})
dragableColumns.value.forEach(dragItem => {
    const originalItem = columns.value.find(col => col.Key === dragItem.Key)
    if (originalItem) {
        sortedColumns.push({
            ...originalItem,
            Visible: dragItem.Visible
        })
    }
})
```

**考试管理**：
```typescript
if (orderMode.value === 'grouped') {
    // 分组模式：和排课列表一样
    columns.value.forEach(item => {
        if (item.Disabled) {
            sortedColumns.push(item)
        }
    })
    dragableColumns.value.forEach(dragItem => {
        const originalItem = columns.value.find(col => col.Key === dragItem.Key)
        if (originalItem) {
            sortedColumns.push({
                ...originalItem,
                Visible: dragItem.Visible
            })
        }
    })
} else {
    // 原始顺序模式：直接使用 allColumnsForDrag 的顺序（已包含拖拽后的顺序）
    sortedColumns = allColumnsForDrag.value.map(item => ({
        ...item,
        Visible: item.Visible
    }))
}
```

#### 4.2 打开设置弹窗

```typescript
function setTableColumnsOpen() {
    let columns = cloneDeep(userDefinedColumns.value)
    
    setTableColumnsRef.value?.open({
        columns: columns,
        moduleId: 'examManage_table',
```

**关键参数**：
- `orderMode: 'original'`：使用原始顺序模式，禁用列可以拖动但不可编辑
- `orderMode: 'grouped'`：使用分组顺序模式，与排课列表相同

### 5. 样式差异

**排课列表**：
```scss
&.columns-item-disabled{
    cursor: not-allowed;  // 禁用列显示"禁止"光标
}
```

**考试管理**：
```scss
&.columns-item-disabled{
    cursor: grab;  // 禁用列显示"抓取"光标，表示可以拖动
    &:active{
        cursor: grabbing;
    }
}
```

### 6. 适用场景

**使用公共组件（排课列表模式）**：
- 禁用列不需要拖动
- 禁用列始终显示在最前面
- 适用于大多数列表页面

**使用专用组件（考试管理模式）**：
- 需要保持列的原始顺序
- 禁用列需要可以拖动排序
- 禁用列不能编辑显示/隐藏状态
- 适用于有特殊顺序要求的列表页面

### 7. 注意事项

1. **数据同步**：在原始顺序模式下，需要同步更新 `columns`、`dragableColumns` 和 `allColumnsForDrag` 三个数据源
2. **全部勾选**：`allChecked` 的 setter 需要同步更新 `allColumnsForDrag` 中非禁用列的可见性状态
3. **点击事件**：禁用列的点击事件需要被阻止（`@click="!item.Disabled && handleColumnClick(item)"`）
4. **拖拽绑定**：根据 `orderMode` 选择不同的数据源进行拖拽绑定
