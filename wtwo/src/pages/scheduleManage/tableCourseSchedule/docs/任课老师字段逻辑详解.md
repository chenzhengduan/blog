# 任课老师字段逻辑详解

## 概述

任课老师字段采用三字段设计模式，确保前端操作、显示和后端传输的数据一致性：

- **`MainTeacherID`**: 表格绑定字段，存储教师ID，用于表格数据绑定和操作
- **`MainTeacherName`**: 显示字段，存储教师姓名，用于界面显示
- **`MainTeacherList`**: 传输字段，对象数组格式，用于后端API传输

beforeCut: ({ data, selectionRangeIndexes, selectionRangeKeys }) => {

// 通用数据收集函数
const collectDataForClipboard = (selectionRangeIndexes, isCut) => {
    const columnFields = columns.map(v => v.field).filter((_, index) => {
        return index >= selectionRangeIndexes.startColIndex && index <= selectionRangeIndexes.endColIndex
    });
    
    copyData.value = displayTableData.value.filter((_, index) => {
        return index >= selectionRangeIndexes.startRowIndex && index <= selectionRangeIndexes.endRowIndex

        const clipboardRow = { ID: row.ID }
        columnFields.filter(v => v).forEach(field => {
            // 收集ID字段和对应的Name字段
            clipboardRow[field] = row[field]
            clipboardRow[fieldMap[field]] = row[fieldMap[field]]
        })
        return clipboardRow
    })
}

```typescript
afterCut: ({ data, selectionRangeIndexes, selectionRangeKeys }) => {
    // 清理剪切后的原字段值，并记录变更
    if (selectionRangeIndexes && displayTableData.value) {
        // 获取剪切的行列范围
        const startRowIndex = selectionRangeIndexes.startRowIndex
        const endRowIndex = selectionRangeIndexes.endRowIndex

            const targetRow = displayTableData.value[rowIndex]
            if (targetRow && !targetRow.isGroupRow && !targetRow.isGroupFooter) {

                    if (column && column.field) {
                        // 清理被剪切字段的相关名称

                        // 记录字段变更到变更收集器
                        recordFieldChange(targetRow.ID, column.field, '', 'cut')

                        const nameField = fieldMap[column.field]
                        if (nameField) {
                            recordFieldChange(targetRow.ID, nameField, '', 'cut')
                        }
                        
                        // 特殊处理：如果是主教师字段，记录 MainTeacherList 的变更
                        if (column.field === 'MainTeacherID') {
                            recordFieldChange(targetRow.ID, 'MainTeacherList', [], 'cut')
                        }
                    }
                }
                
                // 同步到 tableData.value
                const tableDataIndex = tableData.value.findIndex(row => row.ID === targetRow.ID)
                if (tableDataIndex !== -1) {
                    tableData.value[tableDataIndex] = { ...targetRow }
                }
            }
        }
    }
}
```

#### 作用
- 收集剪切区域的所有字段数据（包括ID和Name）
- 清空原字段值
- 记录所有字段变更
- 支持多行多列剪切

#### 后续操作
- 数据存储在copyData.value中
- 原字段被清空并记录变更
- 可以粘贴到其他位置
- 触发自动保存

#### 与复制的区别
- **复制**：不清除原字段，只收集数据
- **剪切**：清除原字段，收集数据，记录变更
- **粘贴逻辑**：完全一致，都使用copyData.value中的数据

## 数据传输流程

### 前端 → 后端
1. **数据收集**: `collectAllChanges()` 收集所有变更
2. **字段构建**: 自动构建 `MainTeacherList` 传输字段
3. **UUID验证**: 验证 `MainTeacherID` 格式
4. **API调用**: 调用 `BatchSaveCourseDraft` 接口

### 后端 → 前端
1. **草稿查询**: `GetCourseDraftList` 返回 `MainTeacherList`
2. **数据同步**: 提取 `MainTeacherID` 和 `MainTeacherName`
3. **保存响应**: 处理 `SavedDraftList` 和 `SaveErrorList`
4. **错误处理**: 映射字段错误并标记

## 关键函数说明

### `recordFieldChange(rowId, field, value, type)`
- **作用**: 记录字段变更到变更收集器
- **参数**: 行ID、字段名、新值、变更类型
- **触发**: 所有字段修改操作

### `validateIDField(fieldName, value)`
- **作用**: 验证ID字段的UUID格式
- **处理**: 不符合格式的传null，数组类型过滤无效项
- **应用**: 所有ID字段的传输前验证

### `detectInvalidFields(draft)`
- **作用**: 检测无效的关联数据
- **逻辑**: 检查ID字段是否有对应的名称值
- **返回**: 无效字段名数组
