# 上课教室线上课逻辑改动方案

## 1. 功能概述

### 1.1 需求背景
当前表格排课中，上课教室字段对所有课程类型都是必填的。但实际业务中，线上课不需要教室，应该像上课科目字段一样，根据课程类型（CourseType）显示"无需填写"。

### 1.2 功能描述
- **线上课（CourseType == '1'）**：教室字段显示"无需填写"，不参与校验
- **线下课（CourseType == '2'）**：教室字段正常选择，参与校验
- **状态切换**：线上课  线下课切换时自动清空/恢复教室选择

### 1.3 参考实现
参考现有的上课科目字段逻辑（基于 EnableSubject）

## 2. 详细改动方案

#### 2.1.4 修改 empty 状态显示
`vue

#### 2.1.5 添加 CourseType 变化监听
`vue
watch(() => props.courseType, (newType, oldType) => {
  if (newType == '1') {
    // 切换到线上课，清空教室并停止拉取
    selectedValue.value = ''
    emit('update:modelValue', '')
    emit('change', '', null)
    classroomList.value = []
    return
  }
  if (props.campusId) {
    fetchClassroomList()
  }
})
`

#### 2.1.6 线上课不触发接口（关键修复）
- 在 `fetchClassroomList` 起始位置增加保护：`if (props.courseType == '1') return`。
- `watch(campusId)`、`watch(searchKeyword)` 加同样条件，线上课不请求。
- `filteredClassroomList` 在线上课返回 `[]`，面板无可选项。

### 2.6 第六步：预检查开关行为（新增）

## 5. 风险评估

### 5.1 兼容性风险
- **风险**：现有线上课数据可能有教室字段值
- **应对**：切换到线上课时自动清空，不影响历史数据

### 5.2 性能风险
- **风险**：动态判断可能影响渲染性能
- **应对**：使用计算属性缓存，避免重复计算；线上课直接不请求接口

### 5.3 用户体验风险
- **风险**：用户可能不理解状态切换逻辑
- **应对**：提供清晰的提示文案和操作反馈
