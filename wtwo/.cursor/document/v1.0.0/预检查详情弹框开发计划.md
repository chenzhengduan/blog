# 预检查详情弹框开发计划

## 项目概述
实现预检查冲突与限制的详情弹框功能，包括限制详情和冲突详情两个tab，以及主表格中对应字段的视觉标记。预检查采用渐进式检查策略，优先检查可视区域数据，滚动时动态检查新数据，已检查过的数据在滚动时不再重复检查。

## 功能需求

### 1. 预检查详情弹框
- **位置**: `@/popup/PreCheckDetailsDialog.vue`
- **结构**: 双tab设计（限制详情 + 冲突详情）
- **数据源**: 预检查API返回的 `CheckFieldList` 和 `ConflictFieldList`

### 2. 表格字段标记
- **限制字段**: 橙色边框标记（`CheckFieldList.FieldNameList`）
- **冲突字段**: 红色边框标记（`ConflictFieldList.FieldNameList`）
- **错误字段**: 红色边框标记（`ErrorFieldList`）

### 3. 交互功能
- **前往修改**: 复用现有的 `handleGoToRowFromDialog` 滚动功能
- **弹框关闭**: 保持表格字段标记状态

## 技术实现

### 渐进式预检查与虚拟滚动策略
- **目标**: 以“效率与体验优先”为原则，只对“当前屏幕可见 + 少量缓冲”的数据发起预检查；滚动时增量检查新出现的行。
- **适配**: 利用 `base-table` 的 `virtualScrollOption` 与 `scrolling({ startRowIndex })` 回调。
- **核心数据结构**:
  - `checkedIds: Set<string>` 已检查并有结果的行ID；滚动时不重复检查。
  - `pendingIds: Set<string>` 已提交检查、尚未返回的行ID；避免并发重复。
  - `preCheckedIds: Set<string>` 本轮“参与检查”的ID（仍用于列渲染控制）。
- **可视区范围**:
  - 从 `startRowIndex` 获取可视起点。
  - `visibleCount = 实际渲染的行数`，可通过 `tableRef.getTableInstance().$el.querySelectorAll('.fan-table-body-tr').length` 粗略获取。
  - 设定缓冲区：`PRE_BUFFER = 10`，`POST_BUFFER = 30`。
  - 范围计算：`start = max(0, startRowIndex - PRE_BUFFER)`；`end = min(total-1, startRowIndex + visibleCount + POST_BUFFER)`。
- **触发策略**:
  - 开启预检查时：立即对“当前可视区 + 缓冲”的未检查/未挂起行发起检查。
  - 滚动时：`scrolling` 回调内做 200–400ms 防抖，增量挑选新范围内未检查/未挂起的行发起检查。
  - 若用户“从中间开始”或“滚到底部再开始”，策略同上；第一次进入即触发一次当前范围检查。
- **行变更后的复检**:
  - 若某行 `row.ID ∈ checkedIds`，且该行被用户改动：
    1) 先保存该行（仅该行）。
    2) 保存成功且无错误后，将该行加入 `pendingIds`，调用检查接口仅检查该行；成功返回后：
       - 更新 `preCheckResults`；
       - 将 `row.ID` 重新加入 `checkedIds`；
       - 加入 `preCheckedIds`（用于本轮渲染）；
       - 从 `pendingIds` 删除。
    3) 保存失败或返回错误：不自动复检；保持 `checkedIds` 不变，提示用户。
- **并发与去重**: 所有待检查ID集合应做 `diff = candidates \ (checkedIds ∪ pendingIds)` 去重后再提交。
- **失败与重试**: 接口失败的ID可加入 `retryQueue`，在下次滚动或用户主动触发时重试。

- **伪代码**:
```ts
const checkedIds = new Set<string>()
const pendingIds = new Set<string>()
const preCheckedIds = new Set<string>()

function triggerCheckByRange(start: number, end: number) {
  const ids = displayTableData.value.slice(start, end + 1).map(r => r.ID)
  const toCheck = ids.filter(id => !checkedIds.has(id) && !pendingIds.has(id))
  if (!toCheck.length) return

  toCheck.forEach(id => pendingIds.add(id))
  callCheckAPI(toCheck).then(res => {
    // 保存本次参与检查的ID，仅用于渲染控制
    res.data.forEach(item => preCheckedIds.add(item.DraftId))
    // 更新结果与状态
    res.data.forEach(item => preCheckResults.set(item.DraftId, item))
    toCheck.forEach(id => {
      checkedIds.add(id)
      pendingIds.delete(id)
    })
  }).catch(() => {
    toCheck.forEach(id => pendingIds.delete(id))
  })
}

// 初次开启
function onPreCheckEnabled() {
  const { start, end } = calcCurrentRangeWithBuffer()
  triggerCheckByRange(start, end)
}

// 滚动增量（防抖）
const onScrollDebounced = debounce(() => {
  const { start, end } = calcCurrentRangeWithBuffer()
  triggerCheckByRange(start, end)
}, 300)

function onRowChanged(rowId: string) {
  if (!checkedIds.has(rowId)) return
  saveRow(rowId).then(ok => {
    if (!ok) return
    if (pendingIds.has(rowId)) return
    pendingIds.add(rowId)
    callCheckAPI([rowId]).then(res => {
      res.data.forEach(item => {
        preCheckedIds.add(item.DraftId)
        preCheckResults.set(item.DraftId, item)
      })
      checkedIds.add(rowId)
      pendingIds.delete(rowId)
    }).catch(() => pendingIds.delete(rowId))
  })
}
```

function handleViewPreCheckDetails(rowId, checkResult) {
  // 获取当前行数据
  const targetRow = tableData.value.find(r => r.ID === rowId)
  if (!targetRow) return

  // 获取预检查结果数据
  const preCheckData = preCheckResults.value.get(rowId)
  if (!preCheckData) return

  // 设置弹框数据
  currentPreCheckRow.value = targetRow
  currentPreCheckData.value = preCheckData
  preCheckDetailsVisible.value = true
}

### Phase 4: 表格字段标记

#### 4.1 扩展cellStyleOption.bodyCellClass
```typescript
bodyCellClass: ({ row, column, rowIndex }) => {
  const classes = []
  
  // ... 现有逻辑 ...
  
  // 预检查字段标记
  if (preCheckEnabled.value && !row.isGroupRow && !row.isGroupFooter) {
    const preCheckData = preCheckResults.value.get(row.ID)
    if (preCheckData) {
      // 错误字段标记
      if (preCheckData.ErrorFieldList.includes(column.field)) {
        classes.push('precheck-error-field')
      }
      
      // 限制字段标记
      const isRestrictionField = preCheckData.CheckFieldList.some(
        check => check.FieldNameList.includes(column.field)
      )
      if (isRestrictionField) {
        classes.push('precheck-restriction-field')
      }
      
      // 冲突字段标记
      const isConflictField = preCheckData.ConflictFieldList?.FieldNameList.includes(column.field)
      if (isConflictField) {
        classes.push('precheck-conflict-field')
      }
    }
  }
  
  return classes.join(' ')
}
```
