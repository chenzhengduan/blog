## 三、实现方案（草案）

### 3.1 流程态设计（单 Dialog 多态 + 异常 Dialog）
- 主 Dialog 三态：
  - 提示态：排课提示（展示统计、预检查摘要、复选框、操作按钮）
  - 进行中：排课中（隐藏操作，展示 Loading/进度）
  - 完成态：排课完成（汇总结果 + “清空表格中的数据”复选框 + 确定）
- 异常 Dialog：仅在后端返回限制/冲突/非法时弹出，提供“仅排通过检查的排课 / 暂不排课，返回修改”

### 3.2 关键交互与参数
- 本轮提交 ID：按“全表有效草稿”收集（与预检查有效行判定一致）
- `CheckConflfict`：由确认态复选框决定（预检查开启时默认勾选，否则默认不勾选）
- “仅排通过检查”：用失败 ID 从本轮提交集合中剔除并重入进行中态

### 3.3 表格联动
- 将后端失败结果映射为与预检查一致的字段级样式（红框 + 图标 + 气泡文案），复用当前 `errorField / errorMessages` 机制
- 过滤重试后，成功行需移除失败标记

### 3.4 状态变量（建议）
- `publishInProgress: boolean`
- `lastSubmittedIds: string[]`
- `failedIdsFromPublish: Set<string>`
- `isRetryForPassedOnly: boolean`
- `rememberCheckConflict: boolean`（复选框记忆）

### 3.5 接口与类型
- 提交：`CourseDraftPublishDraft`
- 删除草稿：`DeleteCourseDraft`
- 返回类型：`CourseDraftPublishDraft_ResFormModel`（`src/types/model/table-course-class.ts`）

### 3.6 错误与边界
- 无有效行：沿用预检查“无可检数据”提示，直接中断
- 网络异常：从进行中切回提示态，允许重试/取消
- 过滤后无可排：提示“无可排数据”，结束流程

## 四、任务拆解（实现层）

1) 主 Dialog 组件态管理（提示/进行中/完成）与交互联动
2) 异常 Dialog 组件：渲染文案、按钮行为、失败 ID 过滤
3) 有效行收集器：与预检查一致（抽取或复用）
4) 与表格联动：失败结果 -> 字段级标注映射与清理
5) API 对接：提交、删除草稿，类型对齐与异常处理
6) 复选框记忆：本地存储读写与默认值策略
7) 统计口径与完成态汇总展示
8) 测试用例覆盖：全成/部分失败/全失败、重试链路、清空数据
