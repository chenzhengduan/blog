# 给学员排课功能改造方案

## 1. 功能概述

### 1.1 需求背景
当前表格排课只支持"给班级排课"（selectedTableType = 10），需要新增"给学员排课"功能（selectedTableType = 20），实现基于学员的1对1课程排课。

### 1.2 功能描述
- **给班级排课**：选择班级，为整个班级安排课程
- **给学员排课**：选择具体学员，基于学员报读的1对1课程进行排课
- **字段差异**：班级排课用ClassID，学员排课用StudentUserID
- **课程逻辑**：学员排课的课程选择基于学员已报读的1对1课程

### 1.3 参考实现
参考现有的"给班级排课"逻辑，在现有表格基础上通过条件判断支持两种排课类型。

    if (config.nameField) {
      fieldMap[field] = config.nameField
    }
  })
  
  return fieldMap
}

    return [...baseFields, 'ClassID']

  } else if (tableType === 30) {

// 检测草稿数据中的非法字段
export const detectInvalidFields = (draft: any, fieldMap: Record<string, string>) => {
  const errorFields: string[] = []

  // 遍历字段映射，检查是否有对应的Name值
  Object.entries(fieldMap).forEach(([idField, nameField]) => {
    const idValue = draft[idField]
    const nameValue = draft[nameField]

    // 如果ID字段有值，但Name字段为空或无效，则标记为非法
    if (idValue && idValue !== '') {
      // 对于助教字段（逗号分隔字符串）
      if (idField === 'AssistantTeacherID') {
        if (typeof idValue === 'string' && idValue.trim() !== '' && (!nameValue || nameValue.trim() === '')) {
          errorFields.push(idField)
        }
      } else {
        // 对于单个ID字段
        if (nameValue === undefined || nameValue === null || nameValue === '') {
          errorFields.push(idField)
        }
      }
    }
  })

  return errorFields
}
`

// 验证必填字段
export const validateRequiredFields = (rows: any[], tableType: number) => {
  const requiredFields = getRequiredFieldsByType(tableType)
  
  // 验证逻辑...
  const errors = []
  
  rows.forEach((row, index) => {
    requiredFields.forEach(field => {
      if (!row[field]) {
        errors.push({
          rowIndex: index,
          field,
          message: 请填写
        })
      }
    })
  })
  
  return {
    isValid: errors.length === 0,
    errors
  }
}

## 7. 风险评估

### 7.1 技术风险
- **组件复杂度**：学员选择组件逻辑较复杂
- **依赖关系**：学员->课程的依赖关系需要正确处理
- **配置影响**：跨校区配置需要正确获取和应用
- **字段顺序**：需要确保字段顺序在切换时保持一致
- **文件修改范围**：涉及多个 composables 文件，需要仔细测试
- **配置抽取风险**：抽取ClassID逻辑可能影响现有功能

### 7.2 业务风险
- **数据一致性**：两种排课类型的数据结构差异
- **用户体验**：组件切换时的状态保持
- **性能影响**：新增组件对页面性能的影响

### 7.3 缓解措施
- **分阶段实施**：先抽取ClassID逻辑，再添加学员排课
- **充分测试**：每个阶段都进行充分测试
- **回滚方案**：保留原有逻辑，确保可以快速回滚
- **渐进式改造**：先保证现有功能正常，再扩展新功能
