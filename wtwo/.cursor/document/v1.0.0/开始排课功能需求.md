# 开始排课功能需求文档

## 1. 功能概述

### 1.1 功能名称
开始排课

### 1.2 功能描述
将当前表格中的排课草稿数据提交到后台，进行正式的排课操作。

### 1.3 功能位置
- 页面：排课表格页面 (`class-table-course.vue`)
- 触发按钮：操作栏右侧的"确认排课"按钮

## 2. 详细业务流程

### 2.1 步骤一：前置必填校验
- 复用"预检查开启时的必填弹框"（含"前往定位"功能）
- 校验表格中带星号必填列是否存在未填写
- 未通过则显示提示并中断流程

### 2.2 步骤二：排课确认 Dialog（提示态）
- **标题**：排课提示
- **内容**：
  - 本次共 xx 节排课，确认排课吗？
  - 若开启了预检查：追加"预检查结果：共 xx 冲突或限制，xx 通过"
- **交互元素**：
  - 左下角复选框"检查上课冲突"
    - 预检查开启时：默认勾选
    - 预检查未开启时：默认不勾选
  - 右侧按钮：取消、确认排课
- **确认后**：进入步骤三

### 2.3 步骤三：调用排课接口
- **API**：CourseDraftPublishDraft（src/api/arrange.ts）
- **入参**：
  - DraftIDList：全表有效草稿 ID 数组
  - CheckConflfict：0/1，取自复选框值
- **进入**：步骤四

### 2.4 步骤四：排课中（主 Dialog 进行态）
- 同一弹窗切换状态：
  - 标题改为"排课中"
  - 隐藏复选框、取消与确认按钮
  - 显示 loading/进度条

### 2.5 步骤五：处理接口返回
#### 5.1 全部成功
- 主 Dialog 切换到完成态：
  - 标题：排课完成
  - 内容：统计信息
  - 底部：复选框"清空表格中的数据" + "确定"按钮
- 确定后：
  - 若勾选则调用 DeleteCourseDraft（传表格所有草稿ID）
  - 关闭弹窗，页面可空着或重新查询

#### 5.2 存在限制/冲突/非法
- **表格标注**：按预检查标准给失败记录字段级红框与图标文案
- **异常 Dialog**：
  - 内容格式：
    - 被限制：xx 节排课
    - （这些行的错误字段的集合）
    - 有冲突：xx 节排课
    - （这些行冲突字段的集合）
  - 底部按钮：
    - "仅排通过检查的排课"：过滤失败ID，关闭所有弹窗，继续步骤四
    - "取消排课，返回修改"：关闭所有弹窗，保留表格标注

### 2.6 步骤六：最终完成（重试后的完成态）
- 展示汇总：共 xx，限制或冲突 xx，剩下的 xx 已完成
- 复选框"清空表格中的数据" + "确定"
- 勾选则调用 DeleteCourseDraft 清空后关闭

## 3. 技术规格

### 3.1 数据范围
- **有效行判定**：与预检查一致
  - 字段：CampusID / ClassID / ShiftID / Date / timeRange / ClassRoomID / MainTeacherID
  - 条件：任一字段有值即视为有效行

### 3.2 弹窗设计
- **主 Dialog**：Element Plus Dialog，三态切换（提示→进行中→完成）
- **异常 Dialog**：独立 Dialog，仅在有失败时出现
- **关闭控制**：都只能手动关闭，不能 ESC

### 3.3 统计口径
- 共 X = 本轮提交 ID 数
- 失败 X = 后端返回失败条数（限制/冲突/非法合计）
- 完成 X = 共 X - 失败 X
- 不单列非法数量

### 3.4 状态管理
- 新结果覆盖旧标注
- 成功行清除旧有错误标记与文案
- 不记忆复选框状态

### 3.5 接口与类型
- **提交**：CourseDraftPublishDraft
- **删除草稿**：DeleteCourseDraft
- **返回类型**：CourseDraftPublishDraft_ResFormModel（src/types/model/table-course-class.ts）

## 4. 边界与异常处理

### 4.1 数据边界
- 无有效行：沿用预检查"无可检数据"提示，直接中断
- 过滤后无可排：提示"无可排数据"，结束流程

### 4.2 网络异常
- 从进行中切回提示态，允许重试/取消

### 4.3 迭代处理
- 允许多次异常 Dialog
- 持续过滤失败 ID 并重试，直到取消或全部成功

## 5. 实现任务拆解

1. 主 Dialog 组件态管理（提示/进行中/完成）与交互联动
2. 异常 Dialog 组件：渲染文案、按钮行为、失败 ID 过滤
3. 有效行收集器：与预检查一致（抽取或复用）
4. 与表格联动：失败结果 → 字段级标注映射与清理
5. API 对接：提交、删除草稿，类型对齐与异常处理
6. 统计口径与完成态汇总展示
7. 测试用例覆盖：全成/部分失败/全失败、重试链路、清空数据
