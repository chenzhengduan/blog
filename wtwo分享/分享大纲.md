## 三、旧架构与新架构的对接方式（10分钟）

### 📍 1. 主应用 WonePC（Avalon）现状

- 页面结构复杂、维护成本高、协作效率低

### 📍 2. 子应用接入（Vue3 + Vite）

- 子应用作为独立项目，通过 `<micro-app>` 加载
- 使用 hash 路由避免刷新 404
- 页面级的渐进迁移策略（一个模块一个页面重构）

### 📍 3. 主子通信与状态共享

- 参数传递（data 属性）
- window 全局对象
- 自定义 eventBus 封装通信逻辑

## 四、常见问题与解决方案（7分钟）

| 问题场景         | 原因分析                   | 解决方案                          |
|------------------|----------------------------|-----------------------------------|
| 页面白屏         | 构建资源路径错误           | Vite 配置 base 为 `'./'`         |
| 样式污染         | 子应用样式影响主应用       | 开启 shadowDOM、scoped CSS       |
| 刷新 404         | history 模式刷新不兼容     | 使用 hash 路由 / nginx fallback  |
| 状态同步延迟     | 主子应用通信方式混乱       | 封装统一的通信 bus               |
| 加载缓慢         | 子应用首次进入卡顿         | 使用 `preload` 预加载资源        |

- 主应用部署到主站入口
- 子应用构建上传到 CDN 或静态资源目录
