1. 微前端概述

1.1 核心理念

微前端是一种将单体前端应用分解为多个可独立开发、部署和运行的前端应用的架构模式。

三大核心原则：

  • 独立开发：团队可使用不同技术栈独立开发
  • 独立部署：应用可独立构建、测试和部署
  • 独立运行：应用在主框架中能够独立运行

1.2 业务价值

  • ✅ 技术栈无关：Avalon与Vue3共存，渐进式迁移
  • ✅ 团队自主：不同团队独立开发，提高并行效率
  • ✅ 代码隔离：减少耦合，提高可维护性
  • ✅ 性能优化：按需加载，改善首屏性能
  • ✅ 风险控制：单个应用故障不影响整体

2. 为什么选择micro-app

2.1 技术对比

micro-appWeb Components低低完善我们的场景

single-spa路由劫持高高需配置大型项目

2.2 micro-app优势

  • 🎯 使用简单：类似iframe的HTML标签方式
  • 🎯 轻量级：仅≈10kb (gzip)，零依赖
  • 🎯 完善沙箱：JS沙箱 + 样式隔离 + 元素隔离
  • 🎯 技术兼容：适配Avalon等传统框架
  • 🎯 功能丰富：数据通信、预加载、生命周期等

3. micro-app核心特性

3.1 基础使用

3.1.1 初始化配置

// UMD方式 - 适合Avalon项目
<!-- micro-app 微前端框架 -->
<script src="js/libs/micro-app/index.umd.js"></script>
document.addEventListener('DOMContentLoaded', function() {
  if (window.microApp) {
    window.microApp.default.start({
      shadowDOM: false,
      disableScopecss: false,
      disableSandbox: false,
      destroy: true
    });
  }
});

3.1.2 应用接入

<!-- 基础模式 -->
<micro-app 
  name="vue3-app-appoint" 
  url="http://localhost:8001" 
  baseroute="/">
</micro-app>

<!-- iframe模式 - 解决兼容性问题 -->
<micro-app 
  name="vue3-app-appoint" 
  url="http://localhost:8001" 
  iframe
  disable-memory-router="true"
  disable-patch-request="true">
</micro-app>

5. 样式隔离方案

5.1 样式隔离策略

  1. 自动隔离：自动添加应用前缀，避免样式冲突
  2. 沙箱隔离：子应用样式无法影响主应用
  3. 自定义命名空间：为组件库添加自定义前缀

5.2 Element Plus命名空间配置

SCSS主题配置 `sub-element-theme.scss`

@forward 'element-plus/theme-chalk/src/mixins/config.scss' with (
  $namespace: 'sub'
);
@import 'element-plus/theme-chalk/src/index.scss';

Vue组件配置 `App.vue`

<template>
  <el-config-provider namespace="sub">
    <router-view />
  </el-config-provider>
</template>

效果: `.el-button` → `.sub-button`

6. 路由方案设计

6.1 路由模式选择

6.1.1 Hash路由模式（推荐）

import { createRouter, createWebHashHistory } from 'vue-router';
const router = createRouter({
  history: createWebHashHistory(),
  routes: [...]
});
  • ✅ 不与主应用路由冲突
  • ✅ 无需特殊配置，支持iframe模式

6.2 iframe模式下的路由

当使用iframe模式时，推荐关闭虚拟路由系统，让子应用路由完全独立。

<micro-app
  name="vue3-app-appoint"
  url="http://localhost:8001" 
  iframe
  disable-memory-router="true">
</micro-app>

7. 通信机制详解

7.1 通信方式概览

通信方向方法应用场景
主→子setData / setGlobalData传递配置、用户信息等
子→主dispatch通知状态变化、用户操作等
全局通信GlobalData跨应用数据共享

7.2 主应用发送数据

window.microApp.setData('vue3-app-appoint', { campusList: [...] });

7.3 子应用接收数据

// onMounted
window.microApp.addDataListener((data) => {
  console.log('接收到主应用数据:', data);
});
// onUnmounted
window.microApp.clearDataListener();

8. iframe模式应用

8.1 iframe模式优势

  • 🔒 完全隔离：样式、JS、DOM完全隔离
  • 🛡️ 安全性高：沙箱级别的安全隔离
  • 🔧 兼容性好：解决复杂应用的兼容性问题
  • 🎯 稳定性强：不受主应用环境影响

8.2 原生 iframe vs micro-app iframe 模式

维度 原生 iframe micro-app iframe 模式
使用方式 手动编写 <iframe src="xxx"> 使用 <micro-app iframe> 标签
通信机制 需要手动封装繁琐的 postMessage 内置 dispatch, setData 等API，无需关心底层
应用管理 完全失控，无法感知内部状态 拥有统一的生命周期、keep-alive、预加载等机制
集成体验 与主应用割裂，像独立网页 作为受控组件，能与主应用及其它子应用顺畅协作

10. 最佳实践

10.1 开发规范

  • 应用命名: kebab-case (vue3-app-appoint)
  • 事件命名: camelCase (showHelpDrawer)
  • 样式命名: BEM + 应用前缀 (.v3-appoint__item)

10.2 性能优化

  • 📦 **代码分割**: 按页面拆分chunk
  • 🔄 **缓存策略**: 合理设置缓存头
  • 🚀 **预加载**: microApp.preFetch([...])

10.3 错误处理

监听error事件进行降级处理，对通信失败进行try...catch。

11. 常见问题与解决方案

Q1: Element Plus样式冲突?

A: 使用自定义命名空间$namespace: 'sub'。

Q2: 子应用弹窗无法覆盖主应用?

A: 通过通信机制，让主应用显示弹窗。

Q3: 子应用路由冲突?

A: 使用Hash路由，或为history模式配置baseroute。

Q4: 通信数据丢失?

A: 使用autoTrigger=true确保初次数据能被接收。

Q5: 首屏加载慢?

A: 采用预加载 + 懒加载策略。

总结

WOnePC微前端架构方案基于micro-app框架，实现了Avalon与Vue3的完美融合。通过完善的样式隔离、灵活的路由方案、可靠的通信机制和iframe模式支持，为团队提供了一个稳定、高效的微前端解决方案。

  • 🎯 技术栈融合：Avalon + Vue3 无缝集成

  • 🚀 开发效率：团队并行开发，独立部署
  • 🛡️ 稳定可靠：iframe模式提供最高级别隔离
  • 📈 渐进升级：支持版本切换，降低迁移风险
